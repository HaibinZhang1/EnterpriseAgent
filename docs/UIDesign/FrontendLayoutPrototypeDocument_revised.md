> 版本：v1.1  
> 来源：`NewRequirementDocument_revised.zip` 中修订后需求文档  
> 用途：用于桌面客户端与 Web 管理端原型界面生产、前端页面拆分和交互验收。  
> 范围：仅定义页面结构、信息层级、组件字段、交互逻辑、状态分支、权限分支和错误/空状态。暂不定义视觉风格、主题配色、品牌图形、动效和具体 UI 皮肤。

## 0. 需求基线一致性说明

本版本已按修订后需求基线同步以下关键规则，原型生产时必须以本节作为跨页面一致性约束。

| 规则 | 前端影响 |
|---|---|
| Extension ID 全局唯一 | 发布向导、审核详情、扩展管理均展示全局唯一校验；归档后 ID 不复用 |
| 授权范围不控制本地清理 | 未授权或授权收缩后，安装、下载、写入、新增启用、更新置灰；停用、卸载、清理托管配置、本地敏感变量清理仍可执行 |
| MCP transport 按主流 MCP 设计收敛 | `remote-http -> Streamable HTTP`，`remote-sse -> SSE（兼容）`，`local-command -> stdio`；用户不可自由组合 |
| MCP 使用量固定口径 | MCP 使用量 = 去重成功配置写入用户数 |
| Plugin manual-download 支持企业内部下载地址 | 必须展示下载地址、SHA-256、文件大小、来源系统、有效期，并在下载后做 Hash 校验 |
| 发布申请先建 submission 与 revision | 提交后进入 validating，保存当前 revision，再执行规则校验和 AI 系统预审 |
| 退回修改保留 revision 历史 | 我的提交和审核详情必须展示多次提交版本、差异、校验结果和审核意见 |
| 管理员可转移扩展维护人或归属部门 | 扩展管理详情必须提供转移入口、影响摘要和审计记录 |
| 部门暂不允许移动 | 组织结构页面不展示“移动部门”操作 |
| 部门停用后用户不可登录、扩展失效 | 登录页、用户列表、组织结构、扩展详情均需体现部门停用状态 |
| 客户端设备入口对部门管理员可见 | 部门管理员只能查看管理范围内用户设备；系统管理员查看全部设备 |
| 企业内网地址强校验预留 | MCP endpoint、AI 服务地址当前仅做必填、URL 格式、协议等基础校验；企业内网地址强校验作为预留能力 |

## 1. 原型生产目标

本文件把需求转化为前端可落地的界面蓝图，目标是让原型设计人员和前端开发人员可以直接按页面编号生产界面。

### 1.1 产品端划分

系统前端分为两个独立产品端：

| 产品端 | 技术栈 | 目标用户 | 主要任务 |
|---|---|---|---|
| Windows 桌面客户端 | Electron + React | 普通用户、发布者 | 使用 Agent、发现扩展、安装/接入/管理本地扩展、提交发布申请 |
| Web 管理端 | React | 部门管理员、系统管理员 | 审核发布申请、治理扩展、管理部门用户、查看审计、管理客户端设备和更新 |

桌面客户端不得放入部门、用户、审核、审计、客户端更新发布等管理功能。Web 管理端不得承载普通用户的本地扩展使用功能。

### 1.2 角色与可访问范围

| 角色 | 桌面客户端 | Web 管理端 | 说明 |
|---|---|---|---|
| 普通用户 | 可访问全部桌面一级入口 | 不可访问 | 可以浏览可见扩展、Star、安装授权范围内扩展、提交发布申请 |
| 部门管理员 | 与普通用户一致，不增加桌面管理页签 | 可访问概览、审核、扩展管理、部门与用户、审计日志、客户端设备 | 只能处理管理范围内数据、管理范围内用户设备和下级部门管理员 |
| 系统管理员 | 与普通用户一致，不增加桌面管理页签 | 可访问全部管理入口 | 管理全局审核、扩展、用户、设备、更新和系统设置 |

### 1.3 原型画布建议

| 画布 | 建议尺寸 | 说明 |
|---|---:|---|
| 桌面客户端主窗口 | 1440 × 900 | Electron 主窗口；最小宽度建议不低于 1200 |
| 桌面客户端弹窗 | 720 × 520 / 960 × 680 | 普通弹窗 / 大型向导 |
| 桌面客户端详情抽屉 | 560 - 720 宽 | 从右侧滑出或覆盖式抽屉 |
| Web 管理端 | 1440 × 900 | 左侧导航 + 顶部栏 + 内容区 |
| Web 管理端详情页 | 右侧抽屉或全屏详情 | 审核详情可使用全屏详情页 |

## 2. 全局前端原则

### 2.1 页面类型

系统页面分为以下类型，原型应复用同类模板：

| 页面类型 | 用途 | 典型页面 |
|---|---|---|
| 首页型 | 聚焦入口、榜单、待办，不展示复杂列表 | 桌面社区首页、Web 管理概览 |
| 结果型 | 搜索、筛选、排序、卡片或表格 | 社区搜索结果、扩展管理列表、用户列表 |
| 详情型 | 展示完整信息和主操作 | 扩展详情、审核详情、用户详情、设备详情 |
| 向导型 | 分步骤提交复杂表单 | 发布向导、客户端更新发布向导 |
| 弹窗型 | 完成单一局部任务 | 登录、确认、MCP 接入、Plugin 安装、启用范围 |
| 设置型 | 分组表单配置 | 桌面设置、系统设置 |

### 2.2 页面密度规则

1. 页面标题必须描述任务，不写解释型标题。
2. 首屏说明文字最多两行。
3. 卡片描述最多两行。
4. 卡片操作只保留 `Star/取消 Star` 这类轻操作，安装、更新、接入、卸载等主操作进入详情页。
5. 表格行操作最多展示 2 个常用操作，其余放入“更多”。
6. 复杂说明放入 tooltip、帮助文本、空状态、确认弹窗或详情页。
7. 所有危险操作必须二次确认，并展示影响摘要。

### 2.3 通用组件清单

| 组件 | 适用位置 | 必备内容 |
|---|---|---|
| 顶部栏 | 桌面客户端、Web 管理端 | 当前用户、通知、账号菜单、设置入口或退出入口 |
| 一级导航 | 桌面客户端、Web 管理端 | 当前选中态、禁用态、权限隐藏 |
| 搜索框 | 社区、管理列表 | 输入框、搜索按钮、清空按钮、回车提交 |
| 筛选栏 | 结果页、管理列表 | 筛选项、重置、应用、已选筛选摘要 |
| 卡片 | 社区结果、类型入口 | 类型标识、名称、两行描述、关键指标、状态、轻操作 |
| 榜单 | 社区首页 | 榜单标题、周期切换、前 5 条、查看更多 |
| 表格 | 管理端、本地列表 | 列头、排序、分页、批量操作、空状态 |
| 详情抽屉 | 桌面扩展详情、管理端轻详情 | 头部摘要、分区内容、底部操作区 |
| 全屏详情 | 审核详情 | 顶部摘要、锚点导航、内容分区、审核操作 |
| 步骤条 | 发布向导、更新发布 | 当前步骤、上一步、下一步、提交 |
| 确认弹窗 | 危险操作 | 影响摘要、原因字段、确认/取消 |
| 错误提示 | 全局 | 动作语义、错误原因、下一步动作 |
| 空状态 | 列表页 | 一句话描述、推荐动作 |

### 2.4 通用状态展示

| 状态类型 | 展示规则 |
|---|---|
| 加载中 | 列表和详情使用骨架屏或局部 Loading，不遮挡整屏，除非首次加载页面 |
| 空状态 | 保留筛选区，内容区展示空状态和推荐动作 |
| 错误状态 | 展示错误原因和下一步动作，支持重试 |
| 无权限 | 隐藏不可访问入口；对可见但不可操作对象，按钮置灰并展示一句原因 |
| 离线 | 桌面顶部显示连接状态；禁用社区搜索、安装、发布、更新等依赖服务端的动作 |
| 会话失效 | 当前操作中断，弹出登录框；登录后回到原任务入口 |
| 并发变化 | 提示“数据已变化，请刷新”，避免重复提交 |

### 2.5 可见选项、授权范围与本地清理的界面规则

系统同时存在“授权范围”和“可见选项”。原型必须分开展示，不得合并为一个概念。授权范围只控制服务端侧新操作和新增本地写入，不控制用户对本机已托管内容的清理。

| 概念 | 控制内容 | 前端表现 |
|---|---|---|
| 授权范围 | 安装、下载、复制配置、配置写入、新增启用、更新 | 详情页主操作可用/置灰；发布向导第三步必填 |
| 本地清理操作 | 停用、卸载、清理本地托管配置、清理本地敏感变量 | 即使授权收缩、扩展下架或用户不再授权，只要本机存在托管项，仍允许执行 |
| 可见选项 | 无权限用户是否可以看到扩展，是否进入榜单 | 发布向导复选框；详情页头部展示；管理端可治理 |

发布向导中的可见选项文案必须固定为：

```text
仅授权范围内展示
说明：勾选表示无权限用户不可见、不进入榜单统计。
```

交互规则：

| 场景 | 前端处理 |
|---|---|
| 默认未勾选 | 所有登录用户可看卡片和详情；未授权用户可 Star；安装、下载、复制配置、配置写入、新增启用、更新置灰 |
| 未授权但本机已有托管项 | 社区详情主操作置灰；本地页仍允许停用、卸载、清理托管配置、清理敏感变量 |
| 勾选“仅授权范围内展示” | 未授权用户搜索不到、榜单不展示、直接链接提示“该扩展不可见或不存在” |
| 从默认展示改为仅授权范围内展示 | 展示收缩，可直接生效，但必须弹确认并记录审计 |
| 从仅授权范围内展示改为默认展示 | 展示扩大，必须进入审核流程 |
| 授权收缩 | 服务端新安装、新下载、新写入、新增启用和更新不可用；本地已安装或已接入内容保留，允许用户主动清理 |

## 4. 桌面客户端全局布局

### 4.1 D-00 桌面应用外壳

#### 布局结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部栏：Logo/产品名     Agent | 社区 | 本地     通知 账号 设置 │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│                         当前页面内容区                         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

#### 导航规则

| 区域 | 内容 | 规则 |
|---|---|---|
| 一级导航 | Agent、社区、本地 | 固定三个入口，不因管理员角色增加后台入口 |
| 右上角 | 通知、账号、设置 | 设置从账号菜单或图标打开，不作为一级导航 |
| 连接状态 | 服务端连接、离线状态 | 离线时顶部显示轻提示，禁用依赖服务端动作 |

#### 启动逻辑

1. 应用启动后检查本地会话。
2. 未登录时可展示登录弹窗，也可进入只读空壳，但访问社区、本地需要登录。
3. 已登录时默认进入 Agent 首页。
4. 首次启动生成并持久化设备 ID，随后登记设备信息。
5. 后台检测服务端连接、客户端版本、本地环境能力。

### 4.2 D-01 登录弹窗

#### 布局

```text
┌────────────────────────────┐
│ 登录 Enterprise Agent Hub   │
│ 手机号                      │
│ 密码                        │
│ [登录]                      │
│ 错误提示区                  │
└────────────────────────────┘
```

#### 字段

| 字段 | 类型 | 规则 |
|---|---|---|
| 手机号 | 文本输入 | 必填、格式校验；手机号是用户属性唯一键，不是系统技术主键 |
| 密码 | 密码输入 | 必填，默认隐藏 |
| 登录按钮 | 主按钮 | 表单合法后可点击；失败过多进入限频或临时锁定状态 |

#### 交互

- 登录成功：关闭弹窗，返回用户原本要执行的动作。
- 登录成功但需要改密：打开 D-26 强制改密弹窗，改密完成后再返回原任务。
- 登录失败：展示明确原因，不暴露账户枚举信息。
- 会话失效触发登录：登录成功后回到原页面或原任务入口，但不自动重复危险操作。
- 修改密码、管理员重置密码或账号冻结后，其他端已登录会话失效。

#### 登录错误状态

| 场景 | 文案 | 后续动作 |
|---|---|---|
| 手机号或密码错误 | 登录失败：手机号或密码错误。 | 保留输入，允许重试 |
| 失败次数过多 | 登录失败：失败次数过多，账号已临时锁定。 | 禁用登录按钮或展示稍后重试提示 |
| 账号冻结 | 登录失败：账号已被冻结，请联系管理员。 | 仅允许关闭弹窗 |
| 所属部门停用 | 登录失败：所属部门已停用，请联系管理员。 | 仅允许关闭弹窗 |
| 普通用户访问管理端 | 登录失败：当前账号无权访问管理端。 | 管理端保留登录页 |
| 服务不可用 | 登录失败：企业服务暂不可用。请检查网络或服务地址。 | 提供重试或打开设置入口 |

#### 按模式交互

| 模式 | 主流程 |
|---|---|
| managed-package | 获取下载凭证 → 下载包 → Hash 校验 → 检查兼容版本 → 安装到目标工具 → 记录状态 |
| config-plugin | 检查目标工具配置 → 备份 → 写入配置 → 启用 → 记录状态 |
| manual-download | 获取下载凭证或受控下载地址 → 下载文件 → Hash 校验 → 打开安装说明 → 用户可标记已安装/已卸载 |

manual-download 使用企业内部下载地址时，仍必须登记 SHA-256、文件大小、来源系统、有效期；客户端下载后必须做 Hash 校验，校验失败不得标记为已安装。下载事件仍受授权、统计和审计控制。

#### 禁止行为

- 未授权时不得下载、安装、写入配置或重新下载。
- 目标工具不兼容时不得执行自动安装。

| 安全下架 | 安全原因、影响范围、处置建议、安全下架策略 | 仅系统管理员；通知存量用户；本地页按策略展示安全风险和处置状态 |
| 重新上架 | 重新上架原因、是否涉及安全风险、修复说明 | 可能直接生效或创建审核申请 |
| 归档 | 原因、二次确认 | 终态，不可重新上架；Extension ID 不再复用 |
| 修改授权范围 | 当前范围、目标范围、影响摘要 | 授权扩大走审核；收缩直接生效并审计 |
| 修改可见选项 | 当前选项、目标选项、影响摘要 | 展示扩大走审核；展示收缩直接生效并审计 |
| 转移维护人 | 当前维护人、目标维护人、原因、通知选项 | 目标用户必须为可用用户；写审计；通知原维护人和新维护人 |
| 转移归属部门 | 当前归属部门、目标归属部门、影响摘要、原因 | 影响审核归属、管理范围、统计归属；写审计；通知相关管理员和维护人 |

#### 安全下架策略选择

```text
安全下架策略
( ) 仅提醒：本地页标记安全风险，用户可继续使用
( ) 阻止新增启用/接入：已启用保留，但不能新增目标或更新配置
( ) 强制停用托管项：客户端在线同步后自动停用托管 Skill/MCP/Plugin 配置
```

选择不同策略后，弹窗必须展示预计影响用户数、预计影响设备数、是否通知存量用户、是否需要客户端在线同步。

## 19. 关键交互流程

### 19.1 社区发现流程

```text
进入社区首页
  -> 查看榜单或输入搜索
  -> 搜索结果页按类型分组展示
  -> 点击卡片打开详情
  -> 判断可见选项和授权范围
      -> 授权：展示主操作
      -> 未授权但可见且本机未安装/未接入：主操作置灰，Star 可用
      -> 未授权但可见且本机已安装/已接入：社区主操作置灰，本地页允许停用/卸载/清理
      -> 不可见：提示该扩展不可见或不存在
```

### 19.2 Skill 安装与启用流程

```text
Skill 详情点击安装
  -> 服务端校验授权
  -> 获取下载凭证
  -> 下载包
  -> Hash 校验
  -> 写入 Central Store
  -> 记录本地事件
  -> 安装成功后可打开启用范围
  -> 选择工具/项目
  -> 生成执行计划
  -> 应用变更
  -> 成功/失败/部分成功反馈
```

授权收缩、下架或归档后，新增启用和更新不可用，但本地页仍允许停用、卸载和清理。

### 19.3 MCP Server 接入流程

```text
MCP 详情点击安装到工具
  -> 校验授权
  -> 打开接入弹窗
  -> 展示固定接入方式与 transport
      -> remote-http：Streamable HTTP
      -> remote-sse：SSE（兼容）
      -> local-command：stdio
  -> 选择目标工具
  -> 检测工具 MCP 写入能力
  -> 填写变量，敏感变量本机保存
  -> 预览脱敏配置摘要
  -> 备份原配置
  -> 写入托管配置项
  -> 执行连接检测
  -> 记录本地接入状态和事件
  -> 失败时展示原因并尽量回滚
```

MCP 使用量统计口径为：去重成功配置写入用户数。授权收缩后，只允许检测或清理本机已有托管配置，不允许新增写入、复制配置或更新配置。

### 19.4 Plugin 安装流程

```text

  -> 校验授权
  -> 打开安装弹窗
  -> 根据安装模式展示不同操作
      -> managed-package：下载、Hash 校验、安装、回滚
      -> config-plugin：备份配置、写入、启用、回滚
      -> manual-download：受控下载、Hash 校验、查看说明、标记状态
  -> 记录本地事件
  -> 同步服务端统计
```

manual-download 使用企业内部下载地址时，必须展示下载地址、SHA-256、文件大小、来源系统、有效期；下载后校验失败不得标记已安装。

### 19.5 发布申请流程

```text
点击 + 发布
  -> 登录校验
  -> 选择类型并提供内容
  -> 自动识别并补全信息
  -> 选择授权范围和可见选项
  -> 填写风险声明
  -> 确认提交
  -> 服务端创建 submission 和当前 revision
  -> 状态进入 validating
  -> 执行服务端最终校验
  -> 保存系统规则预审结果
  -> 执行 AI 系统预审或标记 AI 不可用
  -> 状态进入 pending_review
  -> 管理员审核
  -> 发布者在我的提交查看当前状态和 revision 历史
```

退回修改后重新提交必须生成新的 revision，并保留历史提交快照、规则校验、AI 预审结果和管理员意见。

### 19.6 管理员审核流程

```text
管理员进入审核列表
  -> 筛选待处理申请
  -> 打开审核详情
  -> 查看申请内容、revision 历史、系统校验、AI 预审、类型化预览、授权和可见影响、组织与权限判断
  -> 填写审核意见
  -> 通过 / 退回修改 / 拒绝
  -> 服务端重新校验权限、状态和当前 revision
  -> 写审核记录和审计日志
  -> 通知发布者
```

### 19.7 可见选项变更流程

```text
作者或管理员修改可见选项
  -> 默认展示改为仅授权范围内展示
      -> 展示收缩
      -> 确认影响摘要
      -> 直接生效并写审计
  -> 仅授权范围内展示改为默认展示
      -> 展示扩大
      -> 创建审核申请
      -> 管理员审核通过后生效
```

### 19.8 扩展下架与安全下架流程

```text
扩展详情点击下架
  -> 选择下架原因并填写说明
  -> 确认影响摘要
  -> 下架生效
  -> 社区不再展示
  -> 禁止新安装/接入/下载/更新
  -> 存量本地不自动删除
  -> 写审计日志并通知相关用户
```

安全下架只允许系统管理员执行，必须选择安全下架策略：

| 策略 | 服务端行为 | 桌面端表现 |
|---|---|---|
| 仅提醒 | 禁止新安装/接入/下载/更新，通知存量用户 | 本地页标记安全风险，用户可停用或卸载 |
| 阻止新增启用/接入 | 禁止新安装、接入、新增启用和更新 | 已启用保留，但新增目标置灰 |
| 强制停用托管项 | 下发强制停用策略，客户端在线同步后处理 | 展示等待停用、已自动停用或停用失败 |

### 19.9 授权收缩与本地清理流程

```text
管理员收缩授权范围
  -> 展示影响摘要
  -> 确认后直接生效并写审计
  -> 服务端禁止失权用户新安装、新下载、新写入、新增启用和更新
  -> 桌面端同步后标记“授权收缩”
  -> 本地页仍允许用户停用、卸载、清理托管配置和敏感变量
```

### 19.10 部门与用户管理流程

```text
进入部门与用户
  -> 组织结构维护部门树，不支持移动部门
  -> 用户账号维护用户
  -> 所有写操作按角色和部门范围校验
  -> 用户部门或角色变更后会话失效
  -> 部门停用后，该部门及下级用户无法登录，已有会话失效，归属该部门的活跃扩展视为失效
  -> 写审计日志
```

### 19.11 扩展维护人或归属部门转移流程

```text
管理员在扩展管理详情点击转移维护人/归属部门
  -> 选择目标维护人或目标归属部门
  -> 展示影响摘要
  -> 填写原因
  -> 确认后生效
  -> 写审计日志
  -> 通知原维护人、新维护人、相关管理员
```

### 19.12 客户端更新流程

```text
系统管理员创建新版本
  -> 填写版本和发布说明
  -> 上传或登记安装包
  -> 校验 SHA-256 和签名
  -> 设置发布策略
  -> 确认发布
  -> 客户端检查到新版本
  -> 下载并校验 Hash
  -> 校验签名
  -> 启动安装程序
  -> 更新结果上报设备事件和审计日志
```

## 20. 表单与校验规则汇总

### 20.1 发布向导客户端校验

| 校验项 | 规则 |
|---|---|
| Extension ID | 必填，格式合法，全局唯一，不得与任何类型已有 ID 冲突；归档后 ID 不再复用 |
| 显示名称 | 必填 |
| 描述 | 必填，卡片摘要最多两行 |
| 版本号 | 必须符合 SemVer，版本更新需递增 |
| 变更说明 | 必填 |
| 分类 | 必须选择 1 个 |
| 标签 | 选择 1 到 5 个 |
| 授权范围 | 必须选择 |
| 可见选项 | 必须明确，默认不勾选 |
| Skill 包 | 必须包含 SKILL.md，不超过 5MB，不超过 100 个文件 |
| MCP Server | 必须声明接入方式、transport 固定映射、连接方式、配置模板、变量 schema、权限声明、数据访问说明；合法组合为 remote-http/streamable-http、remote-sse/sse、local-command/stdio；endpoint 当前仅做格式、协议和必填校验 |
| Plugin | 必须声明目标工具、安装模式、兼容版本、安装/下载方式、权限声明、风险说明；manual-download 企业内部下载地址必须登记 SHA-256、文件大小、来源系统、有效期 |

### 20.2 管理端写操作校验

| 动作 | 校验 |
|---|---|
| 审核通过/退回/拒绝 | 管理员权限、禁止自审、申请最新状态 |
| 删除部门 | 无用户、无下级、无活跃扩展、无待审核申请、有管理权限；部门移动不支持 |
| 删除/冻结用户 | 不得操作管理范围外用户，不得删除或冻结最后系统管理员 |
| 修改角色 | 不得越权设置系统管理员；角色变更后会话失效 |
| 修改可见选项 | 展示扩大进入审核，展示收缩确认后直接生效并审计 |
| 发布客户端更新 | 版本递增、Hash、签名、包大小、强制更新原因 |
| 修改系统设置 | 权限、变更摘要、审计日志；密码策略、会话策略、安全策略变更需展示影响摘要 |

## 21. 错误、成功、空状态文案规范

### 21.1 错误文案格式

```text
{动作}失败：{原因}。{下一步动作}。
```

示例：

| 场景 | 文案 |
|---|---|
| Skill 安装失败 | 安装 Skill 失败：下载包 Hash 校验不通过。请重新安装或联系管理员。 |

| 登录失败：账号锁定 | 登录失败：失败次数过多，账号已临时锁定。 |
| 审核并发 | 审核失败：该申请已被其他管理员处理。请刷新列表。 |
| MCP 写入失败 | MCP 配置写入失败：目标工具配置文件不可写。请检查工具路径或联系管理员。 |
| Plugin 安装失败 | Plugin 安装失败：目标工具版本不兼容。请升级目标工具或选择手动安装。 |
| 不可见访问 | 查看详情失败：该扩展不可见或不存在。 |

### 21.2 成功提示

登录成功、强制改密成功、发布申请提交成功、审核通过、退回修改、拒绝、安装成功、MCP 配置写入成功、MCP 连接检测成功、Plugin 安装成功、更新成功、启用成功、停用成功、卸载成功、用户创建成功、用户冻结成功、部门停用成功、扩展下架成功、安全下架策略保存成功、维护人转移成功、归属部门转移成功、客户端更新发布成功。

### 21.3 空状态

| 页面 | 文案 | 推荐动作 |
|---|---|---|
| 社区搜索无结果 | 没有找到符合条件的扩展 | 清空筛选、返回社区首页 |
| 本地已安装为空 | 本机还没有安装或接入扩展 | 去社区探索 |
| 工具未检测到 | 未检测到可用工具 | 扫描工具、添加自定义工具 |
| 项目为空 | 还没有添加项目 | 添加项目 |
| 我的提交为空 | 你还没有提交发布申请 | 去发布扩展 |
| 审核待办为空 | 当前没有待处理申请 | 返回概览 |
| 扩展管理为空 | 当前范围内没有扩展 | 清空筛选 |
| 用户列表为空 | 当前筛选下没有用户 | 清空筛选、新增用户 |
| 审计日志为空 | 当前筛选下没有审计记录 | 调整筛选条件 |

## 22. 原型验收清单

### 22.1 桌面客户端

- [ ] 一级导航只有 Agent、社区、本地。
- [ ] 社区首页先展示搜索、发布按钮、三类榜单和类型入口，不展示卡片网格。
- [ ] 搜索后才进入结果页。
- [ ] 全部搜索结果按 Skill、MCP Server、Plugin 分组，不混排。
- [ ] Skill、MCP、Plugin 卡片字段符合模板，卡片不堆叠安装/更新/接入按钮。
- [ ] 详情抽屉按类型展示完整信息和主操作。
- [ ] 未授权但默认可见时，详情可读，主操作置灰，Star 可用。
- [ ] 勾选“仅授权范围内展示”的扩展对未授权用户不可见且不进入榜单。
- [ ] 发布入口是按钮，不是页签。
- [ ] 发布向导四步完整，包含授权范围和可见选项。
- [ ] MCP Server 可以完成安装到工具、变量填写、配置预览、写入、连接检测、更新和卸载的原型闭环，并按 Streamable HTTP / SSE（兼容）/ stdio 展示 transport。
- [ ] Plugin 可以按 managed-package、config-plugin、manual-download 三种模式完成安装/下载/记录状态的原型闭环；manual-download 展示下载源、SHA-256、文件大小、来源系统、有效期。
- [ ] 本地页只有已安装扩展、工具、项目三个子入口。
- [ ] 通知和设置不作为一级页面。
- [ ] 危险操作都有确认弹窗和影响摘要。
- [ ] 授权收缩后，安装/下载/写入/新增启用/更新置灰，但本地停用、卸载、清理仍可用。
- [ ] 安全下架三档策略在本地页有不同状态展示。
- [ ] 登录、首次登录和管理员重置密码后强制改密状态可原型化。

### 22.2 Web 管理端

- [ ] 普通用户无法进入管理端。
- [ ] 部门管理员和系统管理员看到不同导航。
- [ ] 概览页只展示待办、风险、最近审核、最近审计和快捷入口，不执行具体审核。
- [ ] 审核列表和详情包含系统校验、AI 系统预审、类型化预览、授权范围、可见选项、revision 历史、组织与权限判断和审核操作。
- [ ] 审核详情禁止自审，提交决定时处理并发状态。
- [ ] 扩展管理支持 Skill、MCP Server、Plugin 类型切换、维护人/归属部门转移和治理动作。
- [ ] 组织结构与用户账号拆为两个任务型子页；组织结构不展示移动部门入口，部门停用影响摘要完整。
- [ ] 部门管理员可以管理下级部门管理员，但不能管理同级、上级、横向管理员或系统管理员。
- [ ] 审计日志可筛选、查看详情、复制 requestID、导出 CSV。
- [ ] 客户端设备页面展示设备列表、版本分布、在线和事件；部门管理员可见但仅展示管理范围内设备。
- [ ] 客户端更新只对系统管理员开放，包含版本列表、创建版本、发布、暂停、撤回、更新事件。
- [ ] 系统设置只对系统管理员开放，包含 AI 系统预审配置和安全策略。
- [ ] 管理端所有危险操作都有确认弹窗。
- [ ] 批量导入体现手机号属性唯一、用户 ID 技术主键、已存在用户不允许批量变更角色、允许新增部门管理员。
- [ ] 系统设置中 MCP endpoint 和 AI 服务地址的企业内网地址强校验均标记为预留，只做基础格式校验。

## 23. 给原型设计的交付说明

1. 原型优先按页面编号创建页面和组件，确保页面之间跳转闭环完整。
2. 原型不需要设计主题色、品牌插画和动效，但必须体现信息层级、操作位置和状态分支。
3. 卡片、详情抽屉、榜单、表格、确认弹窗、错误提示、空状态应作为可复用组件生产。
4. 三类扩展必须在所有页面中保持类型清晰，不得混成一种对象。
5. 桌面端原型重点验证普通用户任务：发现、安装、接入、发布、本地管理。
6. 管理端原型重点验证管理员任务：审核、治理、组织用户、审计、设备和更新。
7. Agent 首页仅保留入口边界，不在本文件中生产具体 Agent 执行界面。
8. 原型生产前必须检查是否仍出现旧规则：MCP `http` transport、移动部门、卸载受授权控制、无 revision 的发布流程、未拆分的 Plugin manual-download 指标。
