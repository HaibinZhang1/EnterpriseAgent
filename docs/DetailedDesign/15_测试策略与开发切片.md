# 15. 测试策略与开发切片

## 15.1 测试目标

测试目标是证明系统满足：

- 高内聚、低耦合；
- 权限和状态一致；
- 本地事务可回滚；
- 审计可追踪；
- 包安全可防护；
- 离线与同步可恢复；
- Docker 离线部署可运行。

测试不覆盖 UI 视觉和布局。

## 15.2 测试分层

```text
单元测试
  -> 领域规则、状态机、权限、校验器、执行计划生成

集成测试
  -> Spring Boot + PostgreSQL Testcontainers
  -> 文件存储临时卷
  -> Flyway
  -> 审计和事务

契约测试
  -> OpenAPI request/response
  -> Desktop API Client
  -> 管理端 API Client

端到端测试
  -> 服务端 + 客户端后端临时目录
  -> 安装/接入/卸载流程

安全测试
  -> 包安全、脱敏、权限绕过、路径越界

部署测试
  -> Docker Compose、备份、恢复、健康检查
```

## 15.3 服务端单元测试

### Auth

| 用例 | 断言 |
|---|---|
| 密码正确登录 | 返回 session，写登录审计 |
| 密码错误 | 登录失败，记录 login_attempt |
| 连续失败 | 账号锁定 |
| 用户冻结 | 登录拒绝 |
| 部门停用 | 登录拒绝 |
| 密码修改 | 旧 session 失效 |
| Token 不进日志 | 日志脱敏 |

### Org

| 用例 | 断言 |
|---|---|
| 部门管理员管理下级普通用户 | 允许 |
| 部门管理员管理同级管理员 | 拒绝 |

| 停用部门前影响摘要 | 返回用户数/管理员数/扩展数/待审核数 |

### Extension/Scope/Visibility

| 用例 | 断言 |
|---|---|
| public 可见但未授权 | 搜索返回，主操作不可用 |
| authorized_only 未授权 | 搜索不返回，详情不可见 |

| 展示收缩 | 直接生效并审计 |
| 展示扩大 | 创建 submission |
| 下架后主操作 | 拒绝 |
| 归档后重新上架 | 拒绝 |

### Review

| 用例 | 断言 |
|---|---|
| 禁止自审 | permission_denied |
| 部门管理员审核范围内申请 | 允许 |
| 部门管理员审核跨部门申请 | 拒绝 |
| 系统管理员审核全员申请 | 允许 |
| 并发重复审核 | 第二次 review_already_processed |
| 退回修改重新提交 | revision_no 增加，历史保留 |
| 审核通过 | version/scope/visibility 生效，审计存在 |

## 15.4 服务端集成测试

### 数据库与事务

- Flyway 从空库迁移成功。
- 每个关键写操作审计同事务存在。
- 审计失败时业务失败。
- 包存储失败时不创建发布结果。
- 审核通过事务回滚后扩展状态不改变。

### 文件存储

- temp 上传成功。
- 校验通过移动正式目录。
- 同 hash 同版本不覆盖历史。
- temp 清理不会删正式包。
- 预览失败不影响规则预审主体。

### AI 预审

- AI disabled：submission 进入 pending_review。
- AI timeout + continue：ai_status=UNAVAILABLE，进入人工审核。
- AI timeout + block：状态阻断。
- AI 输入不含 secret 明文。
- AI 返回异常 JSON 时可处理并降级。

## 15.5 包安全测试

| 攻击 | 期望 |
|---|---|
| `../evil.txt` | package_path_traversal |
| `/etc/passwd` | package_path_traversal |
| `C:\evil.txt` | package_path_traversal |
| zip symlink 指向包外 | package_path_traversal |
| 压缩炸弹 | package_uncompressed_size_exceeded |
| 文件数超限 | package_file_count_exceeded |

## 15.6 客户端后端单元测试

### LocalExecutor

| 用例 | 断言 |
|---|---|
| write-file-atomic 成功 | temp 被替换，原文件备份 |
| 写入失败 | 原文件保留 |
| json-upsert 成功 | JSON 修改正确 |
| json-upsert parse 失败 | 不覆盖原文件 |
| copy-tree 失败 | 回滚 |
| symlink 失败 copy 成功 | 结果成功，事件记录 fallback |
| remove-path 根目录 | PlanValidator 拒绝 |
| secret step 序列化 | plan_json 不含 secret value |
| 部分成功 | result=PARTIAL_SUCCESS |

### ToolAdapter

每个内置适配器测试：

- detect 在临时目录或 mock 环境返回正确工具；
- capabilities 正确；
- build plan 不越界；
- dry-run 不写文件；
- uninstall 只删除托管项。

### SecureVariableStore

- 保存/读取/删除 secret；
- 密钥不存在返回 null；
- 日志不包含 value；
- export local state 不包含 value。

## 15.7 客户端后端集成测试

使用临时目录模拟：

4. MCP 配置写入 JSON 文件。

6. MCP 卸载只删托管项。
7. Plugin managed-package 安装和卸载。
8. Plugin config-plugin 写入和清理。
9. manual-download 校验 Hash 并标记。
10. 离线禁止新安装。
11. 本地事件断网排队、联网同步。

## 15.8 契约测试

使用 OpenAPI 生成测试：

- 所有 API 成功响应含 requestId。
- 所有错误响应含 code/message/retryable。
- 分页接口含 page/pageSize/total/hasNext。
- 写操作声明 Idempotency-Key。
- 敏感字段不出现在响应 schema 中。
- Desktop API Client 能正确反序列化关键 DTO。

## 15.9 端到端验收用例

### 用例 A：Skill 首发到安装

1. 普通用户提交 Skill 包。
2. 服务端校验 `SKILL.md`。
3. AI 预审进入人工审核。
4. 部门管理员审核通过。
5. 另一个授权用户搜索到。
6. 请求下载凭证。
7. 客户端安装成功。
8. 本地事件同步。
9. 下载量统计增加。
10. 审计链路可通过 requestID 查询。

### 用例 B：MCP 接入失败回滚

1. 用户打开授权 MCP。
2. 获取定义。
3. 填写变量，敏感变量保存本机。
4. 写配置前备份。
5. 模拟目标配置不可写。
6. 写入失败。
7. 回滚成功。
8. 上报 `MCP_CONFIG_WRITE_FAILED`。
9. 服务端写审计。
10. 日志不含敏感变量值。

### 用例 C：展示收缩

1. 扩展为所有登录用户可见。
2. 未授权用户可搜索但不可操作。
3. 作者执行展示收缩。
4. 服务端直接生效并审计。
5. 未授权用户搜索不到。
6. 榜单不统计该扩展。

### 用例 D：安全下架

1. 系统管理员安全下架 Plugin。
2. 服务端状态 `SECURITY_DELISTED`。
3. 通知存量安装用户。
4. 客户端同步后标记风险。
5. 若策略为强制停用，执行托管项清理计划。
6. 清理事件同步。
7. 回滚失败时审计可查。

### 用例 E：客户端更新

1. 系统管理员创建更新版本。
2. 上传安装包和签名摘要。
3. 发布更新。
4. 客户端检查更新。
5. 下载凭证签发。
6. 客户端下载并校验 Hash。
7. 客户端校验签名。
8. 启动安装程序。
9. 更新后首次启动上报新版本。
10. 版本分布更新。

## 15.10 性能测试

目标规模几百人，重点验证：

| 场景 | 目标 |
|---|---|
| 社区首页 | 2 秒内返回 |
| 搜索第一页 | 2 秒内返回 |
| 扩展详情基础信息 | 1 秒内返回 |
| 审核列表第一页 | 2 秒内返回 |
| 审计日志第一页 | 3 秒内返回 |
| 客户端设备列表 | 3 秒内返回 |
| 包校验 | 受大小限制，避免阻塞 HTTP 线程池 |
| AI 预审 | 不阻塞主线程池，超时降级 |

性能实现建议：

- 搜索字段建索引或使用 PostgreSQL full text；
- 榜单使用定时聚合表；
- 审计查询索引 request_id、created_at、actor、object、device；
- 大文件下载走流式响应；
- 包校验在后台任务执行。

## 15.12 自动开发提示

为保证 Agent 自动开发可控，建议每个 Slice 的代码生成顺序：

1. 生成数据模型和 migration。
2. 生成 DTO 和 OpenAPI。
3. 生成 Repository。
4. 生成 Domain Service 单元测试。
5. 生成 Domain Service。
6. 生成 Application Service。
7. 生成 Controller。
8. 生成集成测试。
9. 生成客户端 API client 或本地服务。
10. 执行测试并修复。

不要先生成大而全 Controller，再回补业务逻辑。

## 15.14 Agent 自动开发准入检查

每个开发 Slice 完成后必须满足：

1. DTO 与 `05_服务端接口契约.md` 一致。
2. 表结构与 `04_服务端数据模型设计.md` 一致，Flyway 可从空库启动。
3. 权限主操作枚举与 `06_权限_可见性_审核与状态机.md` 一致。
4. 本地执行代码只通过 `ExecutionPlan` 写文件，不允许 UI 或业务服务直接写目标工具配置。
5. 新增接口必须有错误码、幂等策略、审计策略和契约测试。
6. 新增统计事件必须说明是否进入榜单、去重键和 period 聚合规则。
