# 05. 服务端接口契约

## 5.1 通用约定

### Base URL

```text
/api
```

### Header

| Header | 方向 | 必填 | 说明 |
|---|---|---:|---|
| Authorization | client -> server | 登录后必填 | `Bearer <token>` |
| X-Request-ID | client -> server | 可选 | 客户端生成 requestID，服务端缺失时生成 |
| Idempotency-Key | client -> server | 写操作建议必填 | 避免重复提交 |
| X-Client-Version | desktop -> server | 建议 | 客户端版本 |
| X-Device-ID | desktop -> server | 设备接口必填 | 设备 ID |

### 响应结构

```json
{
  "requestId": "req_01HX9N9Y8QS2K7W6QZX2TQ3X0D",
  "success": true,
  "data": {},
  "error": null
}
```

### 分页结构

请求：

```text
?page=1&pageSize=20&sort=createdAt,desc
```

响应：

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "total": 100,
  "hasNext": true
}
```

### 错误结构

```json
{
  "requestId": "req_...",
  "success": false,
  "data": null,
  "error": {
    "code": "validation_failed",
    "message": "提交参数不合法",
    "details": {
      "field": "extensionId",
      "reason": "格式必须为小写字母、数字和连字符"
    },
    "retryable": false
  }
}
```

### 稳定枚举和接口幂等约定

所有接口 DTO 必须使用下列服务端稳定枚举，前端只做展示映射，不得自造枚举值。

```text
Role = NORMAL_USER | DEPARTMENT_ADMIN | SYSTEM_ADMIN
ExtensionType = SKILL | MCP_SERVER | PLUGIN
ExtensionStatus = PUBLISHED | DELISTED | SECURITY_DELISTED | ARCHIVED
VisibilityMode = PUBLIC_TO_ALL_LOGGED_IN | AUTHORIZED_ONLY
ScopeType = DEPARTMENT | DEPARTMENT_TREE | SELECTED_DEPARTMENTS | ALL_EMPLOYEES
SubmissionType = FIRST_PUBLISH | VERSION_UPDATE | METADATA_CHANGE | SCOPE_EXPANSION | SCOPE_REDUCTION | VISIBILITY_EXPANSION | VISIBILITY_REDUCTION | RELIST | ARCHIVE
SubmissionStatus = CREATED | VALIDATING | AI_PRECHECKING | PENDING_REVIEW | IN_REVIEW | CHANGES_REQUESTED | REJECTED | APPROVED | WITHDRAWN
DownloadPurpose = INSTALL | UPDATE | MANUAL_DOWNLOAD | REVIEW_PREVIEW | CLIENT_UPDATE | ADMIN_EXPORT
MainOperation = INSTALL | DOWNLOAD_PACKAGE | CONTROLLED_DOWNLOAD | COPY_CONFIG | WRITE_CONFIG | ENABLE_NEW_TARGET | UPDATE | CONNECTION_TEST
```

写接口幂等规则：

- `POST /submissions`、`POST /submissions/{id}/revisions`、审核决定、治理动作、下载凭证签发、客户端更新发布等必须携带 `Idempotency-Key`。
- 服务端将 `actorId + operation + Idempotency-Key + requestHash` 写入 `idempotency_records`。
- 相同幂等键和相同请求体返回首次成功响应；相同幂等键但请求体不同返回 `idempotency_conflict`。
- `POST /local-events/sync` 使用事件内 `idempotencyKey` 和 `local_events(device_id, idempotency_key)` 去重，不复用通用业务幂等表。

## 5.2 Auth API

### POST /auth/login

请求：

```json
{
  "phone": "13800000000",
  "password": "******",
  "clientType": "DESKTOP",
  "deviceId": "dev_abc",
  "clientVersion": "1.0.0"
}
```

响应：

```json
{
  "token": "only-return-once",
  "expiresAt": "2026-05-07T10:00:00Z",
  "user": {
    "id": "uuid",
    "name": "张三",
    "phoneMasked": "138****0000",
    "role": "NORMAL_USER",
    "departmentId": "uuid",
    "departmentName": "研发部",
    "mustChangePassword": false
  },
  "permissionSummary": {
    "canUseDesktop": true,
    "canUseAdminWeb": false
  }
}
```

服务端动作：

1. 校验账号状态、部门状态、锁定状态。
2. 校验密码。
3. 创建 session。
4. 写登录审计。
5. 若是桌面客户端，登记或更新设备摘要。

### POST /auth/logout

撤销当前 session，写审计。

### GET /auth/me

返回当前用户摘要。不得返回完整权限决策。

### POST /auth/change-password

请求：

```json
{
  "oldPassword": "******",
  "newPassword": "******"
}
```

动作：校验密码策略，更新密码 Hash，撤销其他 session，写审计。

### POST /auth/reset-password/complete

用户使用管理员生成的一次性重置 token 完成密码设置。该接口不要求既有登录态，但必须提供有效 token。

请求：

```json
{
  "resetToken": "only-user-knows",
  "newPassword": "******"
}
```

服务端动作：

1. Hash 后查询 `password_reset_tokens.token_hash`。
2. 校验未过期、未使用、未撤销。
3. 校验新密码策略。
4. 更新 `users.password_hash`、`password_changed_at`，清除或保留 `must_change_password` 由策略决定。
5. 写 `password_reset_tokens.used_at`，撤销目标用户既有 session。
6. 写审计，审计中不得包含 resetToken 或新密码。

### GET /extensions/{extensionId}

返回扩展详情。若扩展为 `AUTHORIZED_ONLY` 且当前用户不可见，返回：

```json
{
  "code": "visibility_restricted",
  "message": "该扩展不可见或不存在"
}
```

MCP 详情响应必须包含脱敏配置模板和变量 schema，不包含用户敏感变量值。

请求：

```json
{"starred": true}
```

### GET /extensions/{extensionId}/versions

返回版本历史。普通用户不可选择安装旧版本；该接口用于详情展示。

## 5.4 下载凭证 API

### POST /download-tickets

请求：

```json
{
  "objectType": "EXTENSION_PACKAGE",
  "extensionId": "code-review-skill",
  "version": "1.2.0",
  "purpose": "INSTALL"
}
```

DownloadPurpose 取值和统计关系：

| purpose | 授权校验 | 统计口径 |
|---|---|---|
| INSTALL | 安装或接入前校验 | Skill 安装下载去重计入下载量；Plugin managed-package 不直接等于安装量 |
| UPDATE | 更新前校验 | 不计入下载量或安装量 |
| MANUAL_DOWNLOAD | Plugin manual-download 受控下载前校验 | 计入 manual-download 受控下载用户数 |
| REVIEW_PREVIEW | 管理员审核预览下载 | 不计入扩展榜单 |
| CLIENT_UPDATE | 客户端更新包下载 | 不计入扩展榜单 |
| ADMIN_EXPORT | 管理导出 | 不计入扩展榜单，必须审计 |

服务端校验：

- 会话；
- 用户状态；
- 部门状态；
- 扩展状态；
- 版本状态；
- 授权范围；
- 可见选项；
- 下载对象存在。

响应：

```json
{
  "ticket": "dt_only_return_once",
  "downloadUrl": "/api/packages/download?ticket=dt_only_return_once",
  "expiresAt": "2026-05-06T10:15:00Z",
  "sha256": "64hex",
  "size": 102400
}
```

### GET /packages/download?ticket=...

下载包。服务端只记录 ticket hash，不在日志输出 ticket 明文。

## 5.5 发布申请 API

### POST /submissions

用于首次发布、版本更新、元信息修改、授权扩大、展示扩大、重新上架等需要审核的提交。

请求：

```json
{
  "type": "FIRST_PUBLISH",
  "extensionType": "MCP_SERVER",
  "extensionId": "finance-mcp-server",
  "baseExtensionId": null,
  "version": "1.0.0",
  "metadata": {
    "name": "财务 MCP Server",
    "description": "财务数据查询接入",
    "category": "数据",
    "tags": ["数据", "分析"],
    "changeLog": "首次发布"
  },
  "authorizationScope": {
    "scopeType": "SELECTED_DEPARTMENTS",
    "departments": [
      {"departmentId": "uuid", "includeChildren": true}
    ]
  },
  "visibilityMode": "AUTHORIZED_ONLY",
  "riskStatement": {
    "summary": "访问财务只读数据",
    "containsScript": false
  },
  "typePayload": {
    "mcpDefinition": {}
  },
  "uploadRefs": [
    {
      "kind": "PACKAGE",
      "tempUploadId": "tmp_uuid"
    }
  ]
}
```

响应：

```json
{
  "submissionId": "uuid",
  "revisionNo": 1,
  "status": "VALIDATING",
  "displayStatus": "待审核"
}
```

服务端动作：

1. 校验提交者状态。
2. 创建 submission。
3. 创建 revision。
4. 写提交审计。
5. 触发规则预审和可选 AI 预审。
6. 预审完成后进入 `PENDING_REVIEW`。

### POST /submissions/{id}/revisions

退回修改后重新提交，创建新 revision。

### POST /submissions/{id}/withdraw

发布者撤回待审核或需修改申请。服务端校验申请人和状态。

### GET /submissions/mine

发布者查询自己的提交记录。

### GET /submissions/{id}

查询提交详情。发布者只能看自己的；管理员按管理范围看。

## 5.6 包上传 API

### POST /uploads/package

Multipart 上传，返回临时上传 ID。该接口写入 `temp_uploads`，仅做前置校验和临时存储，不代表发布通过。

请求参数：

| 字段 | 位置 | 必填 | 说明 |
|---|---|---:|---|
| uploadType | form field | 是 | SKILL_PACKAGE / MCP_MANIFEST / PLUGIN_PACKAGE / PLUGIN_MANIFEST / CLIENT_UPDATE_PACKAGE |
| file | multipart file | 是 | 上传文件 |

响应：

```json
{
  "tempUploadId": "tmp_uuid",
  "uploadType": "SKILL_PACKAGE",
  "sha256": "64hex",
  "size": 204800,
  "fileCount": 12,
  "expiresAt": "2026-05-07T10:00:00Z",
  "precheck": {
    "status": "WARNING",
    "warnings": [
      {"code": "package_unsafe_file_detected", "message": "包含脚本文件"}
    ],
    "fileManifestSummary": {
      "previewableCount": 3,
      "riskFileCount": 1
    }
  }
}
```

规则：

- `tempUploadId` 必须能在 `temp_uploads` 表查到，且归属当前用户。
- submission 创建或 revision 重新提交时，服务端必须重新执行最终校验，并把 temp 上传标记为 `CONSUMED`。
- 过期或已消费的 `tempUploadId` 返回 `upload_expired` 或 `validation_failed`。
- 上传失败不得创建 submission。

### DELETE /uploads/{tempUploadId}

用户主动放弃临时上传。服务端校验所有者后将状态置为 `EXPIRED` 或删除临时文件，不影响已创建的 revision。

## 5.7 审核 API

### GET /reviews/tasks

参数：

```text
status=PENDING|IN_REVIEW|PROCESSED
type=SKILL|MCP_SERVER|PLUGIN
submissionType=FIRST_PUBLISH
riskLevel=HIGH
aiStatus=WARNING
departmentId=uuid
```

服务端按管理员管理范围过滤。

### GET /reviews/tasks/{submissionId}

打开审核详情。服务端重新校验管理员是否有权查看和处理。

响应包含：

- 顶部摘要；
- 当前 revision；
- 系统校验；
- AI 预审；
- 类型化预览；
- 授权和可见影响；
- 历史 revision；
- 可执行审核动作。

### POST /reviews/tasks/{submissionId}/decision

请求：

```json
{
  "revisionId": "uuid",
  "decision": "APPROVE",
  "comment": "通过",
  "reasonCodes": []
}
```

服务端必须：

1. 校验管理员会话和角色；
2. 禁止自审；
3. 校验当前状态未被处理；
4. 校验 revision 为当前 revision；
5. 校验管理范围；
6. 生效决定；
7. 写 review、审计、通知。

并发失败返回：

```json
{
  "code": "review_already_processed",
  "message": "该申请已被其他管理员处理，请刷新"
}
```

## 5.8 管理端用户与部门 API

本节列核心接口，完整请求体、响应体、权限和错误码补充见 `19_接口契约_OpenAPI补充.md`。所有接口都按当前管理员范围过滤，并且所有写操作必须携带 `Idempotency-Key`。

```text
GET    /admin/departments/tree
POST   /admin/departments
PATCH  /admin/departments/{id}
POST   /admin/departments/{id}/disable
POST   /admin/departments/{id}/enable
DELETE /admin/departments/{id}

GET    /admin/users
POST   /admin/users
PATCH  /admin/users/{id}
POST   /admin/users/{id}/freeze
POST   /admin/users/{id}/unfreeze
DELETE /admin/users/{id}
POST   /admin/users/{id}/reset-password
POST   /admin/users/import/preview
POST   /admin/users/import/commit
```

用户修改请求：

```json
{
  "name": "李四",
  "phone": "13900000000",
  "departmentId": "uuid",
  "role": "DEPARTMENT_ADMIN",
  "status": "ACTIVE"
}
```

所有角色/部门/状态变更必须撤销目标用户已有 session，并写审计。

重置密码响应只返回一次性 reset token 或一次性链接，不记录明文：

```json
{
  "resetToken": "only-return-once",
  "expiresAt": "2026-05-07T10:00:00Z",
  "mustChangePassword": true
}
```

部门删除阻塞响应必须列出阻塞原因，方便管理端跳转但不定义 UI：

```json
{
  "code": "department_delete_blocked",
  "message": "部门下仍存在用户或活跃扩展",
  "details": {
    "userCount": 12,
    "childDepartmentCount": 1,
    "activeExtensionCount": 3,
    "pendingSubmissionCount": 2
  }
}
```

## 5.9 作者与维护人治理 API

作者和当前维护人在桌面客户端可执行部分治理动作；这些接口不要求管理员角色，但服务端必须校验“当前用户是作者或维护人”“扩展状态允许”“动作类型是否需要审核”。

```text
GET  /me/extensions
GET  /me/extensions/{extensionId}
POST /me/extensions/{extensionId}/delist
POST /me/extensions/{extensionId}/scope/reduce
POST /me/extensions/{extensionId}/visibility/reduce
POST /me/extensions/{extensionId}/relist-submission
POST /me/extensions/{extensionId}/archive-request
POST /me/extensions/{extensionId}/version-submission
POST /me/extensions/{extensionId}/metadata-submission
POST /me/extensions/{extensionId}/scope-expansion-submission
POST /me/extensions/{extensionId}/visibility-expansion-submission
```

作者侧可直接生效的动作：

| 动作 | 接口 | 结果 |
|---|---|---|
| 普通下架自己的活跃扩展 | `POST /me/extensions/{id}/delist` | 状态变更为 DELISTED，写审计 |
| 授权收缩 | `POST /me/extensions/{id}/scope/reduce` | 新 scope 生效，写审计 |
| 展示收缩 | `POST /me/extensions/{id}/visibility/reduce` | visibilityMode 改为 AUTHORIZED_ONLY，写审计 |

作者侧必须创建 submission 的动作：

| 动作 | 接口 | submissionType |
|---|---|---|
| 发布新版本 | `/version-submission` | VERSION_UPDATE |
| 元信息修改 | `/metadata-submission` | METADATA_CHANGE |
| 授权扩大 | `/scope-expansion-submission` | SCOPE_EXPANSION |
| 展示扩大 | `/visibility-expansion-submission` | VISIBILITY_EXPANSION |
| 重新上架申请 | `/relist-submission` | RELIST |
| 归档申请 | `/archive-request` | ARCHIVE 或直接治理，按权限和状态判断 |

作者侧下架请求：

```json
{
  "reasonType": "MAINTENANCE",
  "reasonDetail": "作者主动维护下架"
}
```

授权收缩请求：

```json
{
  "targetScope": {
    "scopeType": "DEPARTMENT",
    "departments": [{"departmentId": "uuid", "includeChildren": false}]
  },
  "reason": "缩小试用范围"
}
```

展示收缩请求：

```json
{
  "targetVisibilityMode": "AUTHORIZED_ONLY",
  "reason": "避免未授权用户看到配置说明"
}
```

普通用户不得调用 `/admin/*` 路由。管理端接口和作者侧接口可复用同一个 ApplicationService，但 Controller 层路由必须区分，避免桌面客户端误依赖管理端权限。

## 5.12 Plugin 安装定义 API

### GET /extensions/{extensionId}/plugin-definition

返回安装模式、安装清单、包信息、手动说明。下载仍需单独申请 Download Ticket。

manual-download 且使用企业内网下载地址时，响应只返回脱敏来源摘要和 Hash，不直接绕过授权：

```json
{
  "installMode": "MANUAL_DOWNLOAD",
  "manualInstallDoc": "脱敏后的安装说明",
  "manualUninstallDoc": "脱敏后的卸载说明",
  "externalDownload": {
    "sourceSystem": "internal-artifact-repo",
    "urlPreview": "https://artifact.internal/...",
    "sha256": "64hex",
    "size": 123456,
    "expiresAt": "2026-06-01T00:00:00Z"
  },
  "requiresDownloadTicketPurpose": "MANUAL_DOWNLOAD"
}
```

客户端必须先调用 `POST /download-tickets`，purpose 使用 `MANUAL_DOWNLOAD`。服务端授权通过后可以返回 Hub 代理下载 URL 或授权后的内网 URL；客户端下载后仍必须校验登记的 SHA-256，并通过本地事件同步下载结果。

## 5.13 本地事件同步 API

### POST /local-events/sync

请求：

```json
{
  "deviceId": "dev_abc",
  "events": [
    {
      "idempotencyKey": "dev_abc:skill.install:code-review-skill:1.2.0:001",
      "requestId": "req_origin",
      "extensionId": "code-review-skill",
      "version": "1.2.0",
      "type": "SKILL_INSTALL",
      "result": "SUCCESS",
      "errorCode": null,
      "occurredAt": "2026-05-06T10:00:00Z",
      "payloadSummary": {
        "targetCount": 1,
        "mode": "central-store"
      }
    }
  ]
}
```

响应：

```json
{
  "results": [
    {
      "idempotencyKey": "dev_abc:...",
      "status": "ACCEPTED",
      "serverEventId": "uuid"
    }
  ],
  "serverStateHints": [
    {
      "extensionId": "code-review-skill",
      "state": "SCOPE_REDUCED",
      "message": "授权范围已收缩，不可更新或新增启用"
    }
  ]
}
```

结果含义：

| 状态 | 含义 |
|---|---|
| ACCEPTED | 已接收并处理 |
| IGNORED | 幂等重复或无需处理 |
| REJECTED | 事件结构非法或用户/设备不匹配 |

## 5.18 最小 DTO 与契约补充

后续生成 OpenAPI 时，至少补齐以下 schema，避免 Controller 与客户端 SDK 各自推断。

### Permission DTO

```json
{
  "canViewDetail": true,
  "canInstall": false,
  "canDownload": false,
  "canCopyConfig": false,
  "canWriteConfig": false,
  "canEnableNewTarget": false,
  "canUpdate": false,
  "canConnectionTest": false,
  "deniedOperations": [
    {"operation": "INSTALL", "reasonCode": "scope_restricted", "reasonText": "不在授权范围内"}
  ]
}
```

### Review Decision DTO

```json
{
  "revisionId": "uuid",
  "expectedRevisionNo": 2,
  "expectedStatus": "PENDING_REVIEW",
  "decision": "REQUEST_CHANGES",
  "comment": "请补充卸载步骤",
  "reasonCodes": ["PLUGIN_UNINSTALL_INCOMPLETE"]
}
```

### Client Update Create DTO

```json
{
  "version": "1.1.0",
  "buildNo": "2026050701",
  "platform": "WINDOWS",
  "arch": "X64",
  "channel": "STABLE",
  "forceUpdate": false,
  "minSupportedVersion": "1.0.0",
  "releaseNotes": "修复本地安装失败问题",
  "packageTempUploadId": "tmp_uuid",
  "signatureStatus": "VALID",
  "certificateSummary": {"subject": "CN=EnterpriseAgentHub"}
}
```

### Settings Patch DTO

```json
{
  "expectedVersion": 3,
  "value": {},
  "reason": "调整 AI 预审超时"
}
```

所有 PATCH 设置必须使用 `expectedVersion` 或等价乐观锁，避免多个系统管理员并发覆盖。

## 5.19 接口生成建议

后续自动开发时建议先生成 OpenAPI 3.0 文件，再生成：

- Java Controller interface；
- TypeScript desktop API client；
- Web 管理端 API client；
- 契约测试。

OpenAPI 中每个接口必须声明：

- `requestId` 响应头或响应字段；
- 错误码枚举；
- 权限要求；
- 是否需要 `Idempotency-Key`；
- 敏感字段脱敏说明。
