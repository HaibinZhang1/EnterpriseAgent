# 04. 服务端数据模型设计

## 4.1 建模原则

1. PostgreSQL 是业务主库。
2. ID 使用不可变 UUID 或雪花 ID，外部展示可使用字符串。
3. Extension ID 全局唯一，归档后不复用。
4. 历史展示需要保存快照，当前权限判断使用实时用户、部门、部门树和授权范围。
5. Submission Revision、MCP 定义、Plugin 安装清单、包快照必须按版本保存。
6. 高频统计事件与审计日志分表，避免污染审计。
7. JSONB snapshot 必须包含 `schemaVersion` 字段。
8. 所有软删除使用 status 字段，不物理删除核心历史对象。

## 4.2 用户与部门

### users

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | uuid | pk | 不可变 userID |
| name | varchar(100) | not null | 姓名 |
| phone | varchar(32) | unique not null | 手机号属性唯一键 |
| password_hash | varchar(255) | not null | 密码 Hash |
| password_algo | varchar(32) | not null | bcrypt/argon2 等 |
| password_changed_at | timestamptz | not null | 密码更新时间 |
| must_change_password | boolean | not null default false | 下次登录修改密码 |
| department_id | uuid | fk departments | 所属部门 |
| role | varchar(32) | not null | NORMAL_USER / DEPARTMENT_ADMIN / SYSTEM_ADMIN |
| status | varchar(32) | not null | ACTIVE / FROZEN / DELETED |
| locked_until | timestamptz | nullable | 登录失败锁定 |
| created_at | timestamptz | not null | 创建时间 |
| updated_at | timestamptz | not null | 更新时间 |

索引：

```sql
create unique index uk_users_phone on users(phone) where status <> 'DELETED';
create index idx_users_department on users(department_id);
create index idx_users_role_status on users(role, status);
```

说明：若企业要求已删除用户手机号不可复用，则 `uk_users_phone` 不使用 partial index，而是全局 unique。

### password_reset_tokens

用于管理员重置密码、一次性临时重置链接或强制改密流程。只保存 token hash，不保存明文 token。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | uuid | pk | 主键 |
| user_id | uuid | fk users not null | 被重置用户 |
| token_hash | varchar(255) | unique not null | 一次性 token hash |
| purpose | varchar(32) | not null | ADMIN_RESET / USER_CHANGE_REQUIRED |
| expires_at | timestamptz | not null | 过期时间 |
| used_at | timestamptz | nullable | 使用时间 |
| revoked_at | timestamptz | nullable | 撤销时间 |
| created_by | uuid | fk users nullable | 创建人，系统初始化可空 |
| created_at | timestamptz | not null | 创建时间 |

索引：

```sql
create unique index uk_password_reset_token_hash on password_reset_tokens(token_hash);
create index idx_password_reset_user_active on password_reset_tokens(user_id, expires_at)
where used_at is null and revoked_at is null;
```

规则：

- 明文 token 只在接口响应中返回一次，不进入审计、运行日志或错误信息。
- 使用成功后必须写 `used_at`，并撤销目标用户既有 session。
- 管理员重置密码后可把 `users.must_change_password` 置为 true。

### metric_period_aggregates

用于社区榜单、管理端趋势和去重统计的周期聚合。该表可由定时任务或事件消费器从 `activity_events`、`local_events`、`download_tickets` 增量生成。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid | 主键 |
| extension_id | uuid | 扩展 |
| extension_type | varchar(32) | SKILL / MCP_SERVER / PLUGIN |
| metric_type | varchar(64) | STAR_NEW / DOWNLOAD_UNIQUE / INSTALL_UNIQUE / CONFIG_WRITE_UNIQUE / MANUAL_MARK_INSTALLED_UNIQUE / HOT_SCORE |
| period_type | varchar(16) | DAY / WEEK / MONTH / ALL |
| period_key | varchar(32) | 例如 2026-05-07、2026-W19、2026-05、ALL |
| value | bigint | 指标值 |
| calculated_at | timestamptz | 计算时间 |

约束：

```sql
create unique index uk_metric_period_aggregate
on metric_period_aggregates(extension_id, metric_type, period_type, period_key);
create index idx_metric_period_query
on metric_period_aggregates(extension_type, metric_type, period_type, period_key, value desc);
```

统计口径：

- Skill 下载量：`download_tickets.purpose = INSTALL` 且同一用户、同一扩展去重；`UPDATE` 不计入下载量。
- MCP Server 使用量：`local_events.event_type = MCP_CONFIG_WRITE` 且 `result in (SUCCESS, PARTIAL_SUCCESS)` 时按用户去重。
- Plugin managed-package：`PLUGIN_INSTALL` 成功用户去重；受控下载可单独展示但不得混入安装量。
- Plugin config-plugin：`PLUGIN_CONFIG_WRITE` 成功用户去重。
- Plugin manual-download：`download_tickets.purpose = MANUAL_DOWNLOAD` 去重受控下载用户数，`PLUGIN_MANUAL_MARK_INSTALLED` 去重用户标记已安装数，两者分开展示。
- `visibility_mode = AUTHORIZED_ONLY` 的扩展不进入社区榜单聚合，但管理端仍可按权限查看统计。

## 4.12 数据不变量

必须通过服务层事务、数据库约束或两者共同保证：

1. `extensions.extension_id` 全局唯一，归档不复用。
2. 同一扩展同一版本只存在一条 `extension_versions`。
3. `extensions.current_version_id` 必须指向该扩展的 `PUBLISHED` version。
4. `MCP access_type/transport` 必须合法组合。
5. 同一用户对同一扩展只有一条 Star 记录。
6. 同一设备同一幂等键的本地事件只处理一次。
7. 已归档扩展不能重新上架、发布新版本、修改授权范围或可见选项。
8. 已安全下架扩展重新上架只能由系统管理员处理。
9. 最后一个可用系统管理员不能被冻结、删除或降权。
10. 审核通过必须绑定当前 revision，不能对历史 revision 生效。
11. `settings.version` 必须与 `PATCH /admin/settings/{key}` 的 `expectedVersion` 匹配；更新成功后版本递增，冲突返回 `setting_version_conflict`。


## 4.13 数据模型与接口一致性约束

为保证 Agent 自动开发时表结构、接口和统计口径一致，以下字段为强制约定：

1. `POST /uploads/package` 返回的 `tempUploadId` 必须落到 `temp_uploads.id`，submission revision 只能引用当前用户未过期且未消费的临时上传。
2. `POST /download-tickets` 请求体中的 `purpose` 必须落到 `download_tickets.purpose`，并直接参与下载量、manual-download 受控下载量和客户端更新下载排除规则。
3. 所有管理端和作者侧治理写接口都必须使用 `idempotency_records`，本地事件同步使用 `local_events` 自身去重索引。
4. 密码重置、临时密码或一次性改密链接必须使用 `password_reset_tokens`，不得把临时密码明文写入数据库、审计或日志。
5. MCP 定义、Plugin 安装清单和 submission revision 必须保存快照，审核通过时记录生效 revision。
6. 指标聚合不得从审计日志计算，必须从 `activity_events`、`download_tickets`、`local_events` 或本地状态投影计算。
7. 设置 API 的响应 `version` 必须来自 `settings.version`，设置历史中的 `before_version/after_version` 必须与更新前后版本一致。


## 4.14 关键索引与约束补充

以下索引和约束是自动生成迁移时必须包含的最低集合：

```sql
create unique index uk_users_phone on users(phone) where status <> 'DELETED';
create unique index uk_extensions_extension_id on extensions(extension_id);
create unique index uk_extension_versions_ext_version on extension_versions(extension_id, version);
create unique index uk_stars_user_extension on stars(user_id, extension_id);
create unique index uk_local_events_device_idempotency on local_events(device_id, idempotency_key);
create unique index uk_idempotency_actor_operation_key
  on idempotency_records(coalesce(actor_id, '00000000-0000-0000-0000-000000000000'::uuid), operation, idempotency_key);
create index idx_audit_request_id on audit_logs(request_id);
create index idx_metric_period on metric_period_aggregates(metric_name, period_type, period_key);
create index idx_settings_key_version on settings(key, version);
```

`settings` 更新必须使用条件更新：

```sql
update settings
set value = :value,
    version = version + 1,
    updated_by = :actorId,
    updated_at = now()
where key = :key
  and version = :expectedVersion;
```

影响行数为 0 时，不得静默覆盖，必须返回 `setting_version_conflict`。
