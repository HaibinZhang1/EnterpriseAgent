# 11. 包存储、安全校验与 AI 系统预审

## 11.1 服务端文件存储结构

```text
/storage/
├── packages/
│   ├── skill/
│   │   └── <extensionId>/<version>/<sha256>.zip
│   ├── mcp/
│   │   └── <extensionId>/<version>/manifest.yaml
│   └── plugin/
│       └── <extensionId>/<version>/<sha256>.zip
├── manifests/
│   ├── mcp/<extensionId>/<version>/manifest.yaml
│   └── plugin/<extensionId>/<version>/install-manifest.yaml
├── previews/
│   └── <extensionId>/<version>/
├── client-updates/
│   └── windows-x64/<version>/<sha256>.exe
├── temp/
│   └── uploads/<tempUploadId>/
└── backups/
```

规则：

- 正式路径包含 extensionId、version、hash。
- 不允许同一正式路径被覆盖。
- 上传先到 temp，最终校验通过后 move 到正式目录。
- 数据库保存正式路径、hash、大小、文件数。
- 删除或归档扩展不直接删除历史包。

## 11.2 上传处理流程

```text
POST /uploads/package
  -> 生成 tempUploadId
  -> 保存原始文件到 temp
  -> 计算 SHA-256
  -> 初步体积校验
  -> 安全解压扫描，不执行文件
  -> 生成 file_manifest 和 preview candidates
  -> 返回临时校验摘要

POST /submissions
  -> 引用 tempUploadId
  -> 服务端最终校验
  -> 校验通过后正式入库或继续待审核
```

临时文件清理：

- 过期 tempUpload 默认 24 小时清理；
- 已绑定 submission 的 tempUpload 不清理，直到正式存储完成；
- 清理任务记录运行日志，不写审计，除非清理失败影响业务。

### temp_uploads 状态流转

```text
AVAILABLE -> CONSUMED    submission revision 成功引用
AVAILABLE -> EXPIRED     超过 expires_at 或用户主动删除
AVAILABLE -> REJECTED    前置校验失败且不允许继续
CONSUMED  -> *           禁止再次消费
EXPIRED   -> *           禁止引用
```

引用 temp 上传创建 revision 时，服务端必须校验：

1. `created_by` 等于当前提交人。
2. `upload_type` 与扩展类型和 submission type 匹配。
3. `status = AVAILABLE` 且未过期。
4. `sha256`、`size`、`file_count` 与临时文件再次计算结果一致。
5. `precheck_status != FAILED`，除非系统设置允许 WARNING 继续提交。

正式入库失败时，submission 不得进入 `PENDING_REVIEW`；必须返回 `storage_write_failed`，并保留临时上传供用户重试或自动清理。

## 11.3 SafeZipExtractor

### 输入

```java
SafeZipExtractRequest {
  Path archivePath;
  long maxCompressedSize;
  long maxUncompressedSize;
  long maxSingleFileSize;
  int maxFileCount;
  Set<String> allowedPreviewExtensions;
}
```

### 输出

```java
SafeZipExtractResult {
  String sha256;
  long compressedSize;
  long uncompressedSize;
  int fileCount;
  List<FileManifestItem> files;
  List<RiskFinding> findings;
  boolean rejected;
  String rejectCode;
}
```

### 拒绝项

必须拒绝：

- `../` 路径穿越；
- 绝对路径；
- Windows 盘符绝对路径，如 `C:\`；
- 解压后目标逃逸；
- zip 内软链接指向包外；
- 文件名为空；
- 文件名含控制字符；
- 系统保留名称；
- 压缩炸弹风险；
- 包大小、解压后大小、单文件大小、文件数超限。

### 风险提示项

不一定拒绝，但必须进入审核详情：

- 可执行文件；
- 脚本文件；
- 二进制文件；
- 系统配置文件；
- 证书、私钥、Token 样式内容；
- 外部或未识别网络地址；
- MCP local-command；
- Plugin 安装清单缺卸载或回滚；
- 全员可用、跨部门可用、所有登录用户可见。

## 11.5 预览规则

支持预览：

- `.md`
- `.markdown`
- `.txt`
- `.json`
- `.yaml`
- `.yml`

图片只生成缩略图，不执行内容。二进制和未知文件不预览，只提供受控下载入口。

单文件预览超过 256 KB 时截断：

```json
{
  "path": "README.md",
  "content": "...",
  "truncated": true,
  "originalSize": 500000
}
```

## 11.6 Skill 校验

### 必填

- `SKILL.md`。

### 校验项

| 校验 | 失败错误码 |
|---|---|
| 包内存在 `SKILL.md` | skill_manifest_missing |
| 包大小 <= 5MB | package_too_large |
| 文件数 <= 100 | package_file_count_exceeded |
| 版本 SemVer | validation_failed |
| 无路径穿越 | package_path_traversal |
| 无解压逃逸 | package_path_traversal |
| Hash 可计算 | hash_calculation_failed |

### frontmatter 提取

`SKILL.md` 支持 YAML frontmatter。服务端可提取：

```yaml
---
name: code-review-skill
description: 用于代码审查
allowed-tools: Read Grep
disable-model-invocation: false
---
```

提取字段只用于补全建议，最终 metadata 以 submission payload 为准。

## 11.7 MCP 定义校验

### 必填字段

```yaml
serverName: string
version: string
accessType: remote-http | remote-sse | local-command
transport: stdio | streamable-http | sse
command: string        # local-command 必填
endpoint: string       # remote-http / remote-sse 必填
variables: []
configTemplate: object
supportedTools: []
connectionTest: object
permissions: []
dataAccess: string
riskStatement: string
description: string
```

### 校验项

| 校验 | 错误码 |
|---|---|
| accessType/transport 合法组合 | mcp_transport_invalid |
| remote endpoint URL 格式合法 | mcp_endpoint_invalid |
| local-command command 必填 | mcp_transport_invalid |
| variables schema 合法 | validation_failed |
| 敏感变量标记 sensitive | validation_failed |
| configTemplate 可渲染 | mcp_config_template_invalid |
| connectionTest 合法 | validation_failed |
| env 不含真实密钥 | package_unsafe_file_detected |
| command 高风险标记 | warning，不拒绝 |
| http 兼容输入规范化 | 无错误，入库 streamable-http |

### configTemplate 渲染测试

服务端使用变量默认值或占位值执行一次渲染：

```java
RenderedConfig result = templateRenderer.render(template, placeholderVariables);
validateJsonSerializable(result);
```

不得保存用户真实敏感变量值。

## 11.8 Plugin 校验

### 必填字段

```yaml
pluginName: string
version: string
targetTools: string[]
installMode: managed-package | config-plugin | manual-download
compatibleVersions: string[]
permissions: string[]
riskStatement: string
changeLog: string
```

### managed-package

必须校验：

- 插件包存在；
- SHA-256；
- 文件清单；
- 安装步骤；
- 更新步骤；
- 卸载步骤；
- 回滚说明或明确不可回滚；
- 目标工具；
- 兼容版本。

### config-plugin

必须校验：

- 配置清单；
- 目标配置路径规则；
- 启用步骤；
- 禁用步骤；
- 卸载步骤；
- 回滚说明。

### manual-download

必须校验：

- 手动安装说明；
- 手动卸载说明；
- 受控文件或企业内网下载地址；
- SHA-256；
- 文件大小；
- 来源系统；
- 有效期；
- 用户如何确认安装完成。

manual-download 受控下载安全规则：

| 场景 | 服务端行为 | 客户端行为 |
|---|---|---|
| Hub 托管文件 | 签发 `purpose=MANUAL_DOWNLOAD` 的下载凭证 | 下载后校验 SHA-256，记录受控下载事件 |
| 企业内网下载地址 | 校验 URL 格式、来源系统、登记 Hash、有效期和授权范围；可返回代理下载 URL 或授权后的内网 URL | 下载后校验登记 SHA-256 和大小，失败上报 `hash_mismatch` 或 `plugin_download_failed` |
| 下载来源过期 | 返回 `plugin_download_source_expired` | 不展示下载成功，不允许标记下载完成 |
| 未授权用户 | 返回 `scope_restricted` | 不得复制真实下载地址 |

manual-download 不允许：

- 绕过服务端授权直接暴露真实下载地址；
- 将受控下载量和用户标记已安装数合并为一个安装量；
- 在没有手动卸载说明时发布；
- 伪装为自动安装。

## 11.9 敏感信息扫描

使用规则匹配扫描文本内容：

```text
(?i)api[_-]?key\s*[:=]\s*[A-Za-z0-9_\-]{16,}
(?i)token\s*[:=]\s*[A-Za-z0-9_\-\.]{20,}
-----BEGIN (RSA |EC |OPENSSH )?PRIVATE KEY-----
(?i)password\s*[:=]\s*[^,\s]{8,}
```

输出只记录摘要：

```json
{
  "findingType": "POSSIBLE_SECRET",
  "file": "README.md",
  "line": 12,
  "sampleRedacted": "apiKey=sk_***",
  "severity": "HIGH"
}
```

不得把疑似密钥明文写入审计、运行日志、AI 预审输入。

## 11.10 内容外显风险

若 `visibilityMode=PUBLIC_TO_ALL_LOGGED_IN`，系统预审必须额外检查：

- 名称；
- 描述；
- README 摘要；
- 配置示例；
- 安装说明；
- MCP endpoint 摘要；
- Plugin 内网下载地址摘要。

发现敏感信息时：

- 规则预审返回 WARNING 或 FAILED，具体由设置控制；
- 审核详情突出显示；
- AI 预审输入使用脱敏文本。

## 11.15 AI 预审失败降级

设置项：

```yaml
aiPrecheck:
  enabled: true
  baseUrl: http://ai.internal
  model: risk-model
  timeoutMs: 30000
  failurePolicy: CONTINUE_WITH_UNAVAILABLE # or BLOCK_UNTIL_MANUAL_OVERRIDE
```

策略：

| 策略 | 行为 |
|---|---|
| CONTINUE_WITH_UNAVAILABLE | 申请进入人工审核，AI 状态为 UNAVAILABLE |
| BLOCK_UNTIL_MANUAL_OVERRIDE | 申请停留在 AI_PRECHECKING 或 PRECHECK_BLOCKED，需要系统管理员处理 |
| DISABLED | 不调用 AI，AI 状态 DISABLED |

MVP 建议默认 `CONTINUE_WITH_UNAVAILABLE`。

## 11.16 包存储异常处理

| 场景 | 处理 |
|---|---|
| temp 写入失败 | 返回 `server_unavailable` 或 `storage_write_failed` |
| Hash 计算失败 | 返回 `hash_calculation_failed` |
| 路径穿越 | 返回 `package_path_traversal` |
| 包过大 | 返回 `package_too_large` |
| 文件数超限 | 返回 `package_file_count_exceeded` |
| 正式移动失败 | submission 不得进入可审核成功状态，返回 `storage_write_failed` |
| 预览生成失败 | 不阻断发布，但审核详情显示预览不可用 |
| AI 不可用 | 按设置降级 |

## 11.17 安全测试要求

必须覆盖：

1. zip slip：`../evil.txt`。
2. Windows 绝对路径：`C:\evil.txt`。
3. Unix 绝对路径：`/etc/passwd`。
4. 软链接指向包外。
5. 解压炸弹。
6. 单文件超限。
7. 文件数超限。
8. 疑似密钥脱敏。
9. MCP env 真实密钥警告。
10. local-command 高风险执行器识别。
11. Plugin manifest 缺卸载步骤。
12. AI 输入不包含 secret 明文。


## 11.18 包与下载安全自检清单

开发实现必须为以下场景提供自动化测试：

1. `tempUploadId` 不是当前用户创建时，创建 submission 失败。
2. `tempUploadId` 已过期、已消费或文件 Hash 改变时，创建 revision 失败。
3. `download_tickets.purpose = UPDATE` 不增加 Skill 下载量。
4. `download_tickets.purpose = MANUAL_DOWNLOAD` 只增加 Plugin manual-download 受控下载用户数。
5. 审核预览和客户端更新下载不进入扩展榜单。
6. 企业内网下载地址过期或 Hash 不匹配时，客户端不得记录成功下载。
7. AI 预审输入不包含 ticket、password、token、API key 或本机 MCP secret。
