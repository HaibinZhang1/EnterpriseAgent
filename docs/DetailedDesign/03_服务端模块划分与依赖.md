## 3.3 Auth 模块

### 职责

- 手机号 + 密码登录；
- 密码 Hash 与算法版本；
- 登录失败限频或锁定；
- 会话创建、刷新、撤销；
- 密码重置；
- 用户状态/部门状态/角色变化后的会话失效；
- 管理端会话有效期短于桌面端。

### 主要服务

```java
AuthApplicationService
SessionService
PasswordService
LoginAttemptService
PasswordResetService
SessionInvalidationService
```

### 关键规则

1. 密码使用 bcrypt、Argon2 或企业配置算法，保存 `password_algo`。
2. 登录成功和失败均写审计日志。
3. 用户被冻结、删除、部门停用、角色变化、部门变化、密码变更后，撤销所有未过期会话。
4. Token 只保存 hash，日志和审计不记录明文 Token。
5. 登录接口不得返回完整权限矩阵，只返回摘要，最终权限由服务端每次接口判断。

### 核心服务

```java

```

### 管理范围算法

```java
boolean canManageDepartment(UserActor actor, DepartmentId targetDept) {
    if (actor.isSystemAdmin()) return true;
    if (!actor.isDepartmentAdmin()) return false;
    return departmentTreeService.isSelfOrDescendant(actor.departmentId(), targetDept);
}

boolean canManageUser(UserActor actor, User target) {
    if (actor.isSystemAdmin()) return true;
    if (!actor.isDepartmentAdmin()) return false;
    if (!canManageDepartment(actor, target.departmentId())) return false;
    if (target.isSystemAdmin()) return false;
    if (target.isDepartmentAdmin()) {
        return departmentTreeService.isStrictDescendant(actor.departmentId(), target.departmentId());
    }
    return true;
}
```

## 3.7 Extension Version 模块

### 职责

- 版本号校验；
- SemVer 递增；
- 当前版本切换；
- 历史版本废弃；
- version metadata snapshot；
- 包 Hash、包路径、风险摘要；
- 版本发布事件。

### 核心服务

```java
ExtensionVersionService
VersionComparator

```

### 版本规则

- 只支持 `MAJOR.MINOR.PATCH`。
- 不支持 prerelease 和 build metadata。
- 新版本必须大于当前发布版本。
- 同一 Extension 只有一个当前发布版本。
- 历史版本保留，不允许普通用户选择安装旧版本。

## 3.20 Statistics 模块

### 职责

- 统一维护社区热榜、Star 榜、下载量榜、安装量榜和使用量榜的统计口径；
- 消费 `download_tickets`、`local_events`、`stars`、`activity_events` 和 outbox 事件；
- 写入 `metric_period_aggregates`，提供周榜、月榜、总榜查询；
- 严格排除 `visibility_mode = AUTHORIZED_ONLY` 的扩展进入社区榜单；
- 区分审计日志、普通统计事件和本地事件，不从 `audit_logs` 直接聚合榜单；
- 支持管理端按权限范围查看统计摘要和异常趋势。

### 核心服务

```java
StatisticsAggregationService
MetricEventConsumer
MetricPeriodAggregateRepository
CommunityRankingQueryService
ExtensionMetricSnapshotService
MetricDeduplicationService
```

### 聚合边界

1. `DownloadTicketService` 只记录下载凭证和下载目的，不直接更新榜单。
2. `LocalEventIngestService` 只完成本地事件接收和本地状态投影，不直接计算榜单。
3. `Statistics` 模块异步消费事件并按幂等键去重，写入 `metric_period_aggregates`。
4. 审核、下架、展示收缩、授权收缩等治理事件只影响统计可见性和榜单过滤，不修改历史审计。
5. 榜单查询优先读取聚合表；小规模 MVP 可同步刷新，但代码结构仍保持独立模块，便于后续改为异步任务。

## 3.22 模块依赖规则

### 允许依赖

```text
auth -> org, audit
org -> audit
extension -> scope, version, audit, notification
submission -> extension, scope, packagefile, precheck, audit, notification
review -> submission, extension, version, scope, audit, notification
packagefile -> settings, audit
download -> extension, version, scope, audit
localevent -> extension, device, audit
statistics -> extension, download, localevent, audit
device -> audit
clientupdate -> device, download, audit, notification
settings -> audit
```

### 禁止依赖

- `audit` 不依赖任何业务模块 Repository。
- `settings` 不依赖业务模块。
- `packagefile` 不直接修改 extension 状态。
- `precheck` 不直接审核通过或拒绝 submission。
- `device` 不影响用户权限。
- `notification` 不影响主业务状态。
- `statistics` 不参与审核、授权和下载凭证生效事务；它只消费已落库事实，不能反向修改业务主状态。

## 3.23 事务策略

| 用例 | 事务要求 |
|---|---|
| 登录成功 | 创建 session、登录审计同事务 |
| 用户角色/部门变化 | 修改用户、撤销 session、审计同事务 |
| 创建 submission | submission、revision、审计同事务；预审可异步 |
| 规则预审完成 | system_prechecks、状态推进、审计同事务 |
| 审核通过 | review、extension/version/scope/visibility 生效、审计同事务 |
| 授权/展示收缩 | 生效、审计同事务 |
| 下架/归档 | 状态变更、审计同事务 |
| 本地事件同步 | 去重、local state projection、异常审计同事务；统计事件写 outbox |
| 统计聚合 | 消费 outbox 或定时扫描，按幂等键去重，可独立事务重试 |
| 客户端更新发布 | 版本状态、审计同事务 |

Outbox 事件应在业务事务内插入，消费在事务后异步执行。

## 3.24 自动开发注意事项

1. 每个模块先生成接口和 DTO，再生成实现。
2. 领域规则集中在 Domain Service，不要散落在 Controller。
3. Repository 只处理持久化，不承载权限逻辑。
4. 跨模块读取使用 QueryService，避免直接查别的模块表。
5. 写操作必须携带 `ActorContext` 和 `RequestContext`。
6. 审计写入不可吞异常；若审计失败，关键写操作应失败，避免无审计的状态变更。
