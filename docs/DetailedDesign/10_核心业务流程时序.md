# 10. 核心业务流程时序

## 10.1 登录与设备登记

```text
Desktop/Web
  -> POST /auth/login
Server Auth
  -> 校验用户、密码、角色、部门、锁定状态
  -> 创建 session
  -> 写 audit auth.login.success 或 auth.login.failure
Desktop
  -> POST /client-devices/register
Server Device
  -> upsert client_devices
  -> 写 device event FIRST_REGISTER 或 HEARTBEAT
Desktop
  -> 定时 POST /client-devices/heartbeat
```

失败处理：

- 密码错误：`unauthenticated`，写失败审计。
- 账号锁定：`account_locked`。
- 部门停用：`department_disabled`。
- 用户冻结/删除：`permission_denied` 或 `unauthenticated`，避免透露过多账号信息。

## 10.2 社区搜索

```text
Desktop
  -> GET /extensions/search?q=...
Server ExtensionQuery
  -> 校验登录
  -> 按 extension.status=PUBLISHED
  -> 排除 ownerDepartment DISABLED
  -> 按 VisibilityPolicy 过滤
  -> 计算 Permission flags
  -> 返回分页
```

注意：

- 前端不能自己隐藏 `AUTHORIZED_ONLY` 扩展；服务端不返回。
- 可见但未授权的扩展返回权限 flags，用于主操作不可用。

## 10.3 Skill 安装

```text
Desktop Renderer
  -> ipc skill.install(extensionId)
Desktop SkillInstallService
  -> api.getExtensionDetail
  -> api.createDownloadTicket purpose=INSTALL
Server DownloadTicketService
  -> 校验扩展状态、版本、授权范围、可见选项
  -> 签发 ticket
  -> 写 audit download.ticket.issue
Desktop DownloadManager
  -> 下载到 temp
  -> HashVerifier.verify
  -> CentralStore.buildInstallPlan
  -> LocalExecutor.execute
  -> LocalEventQueue.enqueue SKILL_INSTALL
  -> LocalEventSyncService.sync
Server LocalEventIngest
  -> 幂等去重
  -> 更新 local state projection
  -> 更新统计
```

事务点：

- 服务端 ticket 签发与审计同事务。
- 客户端本地执行与事件入队本地事务或同一 execution journal。
- 事件同步服务端去重与统计同事务。

## 10.4 Skill 启用

```text
Desktop
  -> 获取本地已安装版本
  -> 检查离线策略
  -> ToolAdapterRegistry.detectTargets
  -> SkillEnableService.buildPlan
  -> LocalExecutor.execute
      -> path writable
      -> symlink or copy
      -> record managed target
  -> enqueue SKILL_ENABLE / SYMLINK_FALLBACK_COPY / PARTIAL_SUCCESS
```

在线时若是新增启用目标，建议调用服务端轻量授权校验接口或刷新详情；离线时只能基于有效授权缓存且未收到授权收缩/安全下架标记执行。

## 10.5 Skill 更新

```text
Desktop
  -> 本地状态发现 latestVersion > localVersion
  -> api.createDownloadTicket purpose=UPDATE
Server
  -> 重新校验授权范围和状态
Desktop
  -> 下载新包
  -> Hash 校验
  -> 安装新版本到 Central Store
  -> 备份 current pointer 和启用目标
  -> 切换 current
  -> 重新分发到启用目标
  -> 失败则回滚可回滚目标
  -> enqueue SKILL_UPDATE
```

若服务端返回：

- `scope_restricted`：标记授权收缩；
- `extension_delisted`：标记已下架；
- `extension_security_delisted`：标记安全风险；
- `hash_mismatch`：不得覆盖旧版本。

## 10.6 MCP 接入

```text
Desktop
  -> extension detail
  -> ipc mcp.getDefinition(extensionId)
Server
  -> 校验授权和状态
  -> 返回 MCP 定义、变量 schema、模板
Desktop
  -> ToolAdapterRegistry.list supportsMcpConfigWrite
  -> 用户变量输入由 UI 完成，本设计只定义后端接收变量
  -> Sensitive variables -> SecureVariableStore.save
  -> McpTemplateRenderer.render
  -> ToolAdapter.buildMcpWritePlan
  -> LocalExecutor.execute
      -> 备份配置
      -> 写入 managedConfigId
  -> McpConnectionTester.run
  -> enqueue MCP_CONFIG_WRITE and MCP_CONNECTION_TEST
```

失败策略：

- 配置写入失败：回滚配置文件。
- 连接检测失败：若配置写入成功，可保留配置但状态为异常，也可按计划回滚；默认由 plan 指定。建议首次接入检测失败时回滚，已接入更新检测失败时回滚。
- 敏感变量保存失败：阻断写入。

## 10.7 MCP 更新

```text
Desktop
  -> 检查新版本
  -> api.getMcpDefinition(newVersion)
  -> 对比 variables schema
  -> 保留已有 SecretRef
  -> 若新增必填变量则要求补充
  -> ToolAdapter.buildMcpUpdatePlan
  -> LocalExecutor.execute
  -> ConnectionTest
  -> enqueue MCP_UPDATE
```

删除变量后，旧 SecretRef 默认不立即删除，可在更新成功后标记可清理，由用户或清理任务删除。

## 10.8 MCP 卸载

```text
Desktop
  -> 读取 managedConfigId 列表
  -> ToolAdapter.buildMcpRemovePlan
  -> LocalExecutor.execute
      -> 备份配置
      -> 只移除 managed item
      -> 清理 local state
      -> 可删除相关 secret refs
  -> enqueue MCP_UNINSTALL
```

不得删除用户手写配置。

## 10.9 Plugin managed-package 安装

```text
Desktop
  -> getPluginDefinition
  -> 校验目标工具兼容性
  -> createDownloadTicket
  -> 下载包
  -> Hash 校验
  -> PluginManifestExecutor.parse
  -> ToolAdapter.buildPluginInstallPlan
  -> LocalExecutor.execute
  -> enqueue PLUGIN_INSTALL
```

安装清单缺少卸载或回滚说明时，服务端审核阶段应给警告；客户端执行时若清单无卸载步骤，应拒绝自动安装或标记不可回滚风险，按系统策略处理。

## 10.10 Plugin config-plugin 安装

```text
Desktop
  -> getPluginDefinition
  -> ToolAdapter 检测配置路径
  -> buildPluginInstallPlan
  -> 备份配置
  -> 写入托管配置项
  -> enqueue PLUGIN_CONFIG_WRITE
```

卸载只删除托管配置项。

## 10.11 Plugin manual-download

```text
Desktop
  -> getPluginDefinition
  -> 用户查看手动说明
  -> 授权用户 createDownloadTicket 或下载登记内网地址
  -> 下载到受控目录
  -> 校验 SHA-256
  -> enqueue PLUGIN_CONTROLLED_DOWNLOAD
  -> 用户标记已安装
  -> enqueue PLUGIN_MANUAL_MARK_INSTALLED
```

标记不代表真实安装成功，只作为本地状态。

## 10.12 发布申请

```text
Desktop Publish Backend
  -> 上传包或定义，获取 tempUploadId（落库 temp_uploads，状态 AVAILABLE）
  -> POST /submissions
Server SubmissionService
  -> 创建 submission status=VALIDATING
  -> 创建 revision no=1
  -> 写 audit submission.create
  -> Outbox SubmissionCreated
Precheck Worker
  -> Package final validation
  -> RulePrecheck
  -> AI Precheck if enabled
  -> 写 system_prechecks
  -> 计算 review owner
  -> status=PENDING_REVIEW
  -> audit precheck.completed
  -> notification admin
```

规则：

- 服务端最终校验权威。
- AI 预审不可用时按设置降级。
- 发布者看到展示状态“待审核”。

## 10.13 退回修改与重新提交

```text
Admin
  -> POST review decision REQUEST_CHANGES
Server
  -> 校验权限和状态
  -> 写 review
  -> status=CHANGES_REQUESTED
  -> audit + notify submitter

Submitter
  -> POST /submissions/{id}/revisions
Server
  -> current_revision_no + 1
  -> status=VALIDATING
  -> 重新预审
```

历史 revision 不得覆盖。最终通过记录生效 revision。

## 10.14 审核通过

```text
Admin
  -> POST /reviews/tasks/{submissionId}/decision APPROVE revisionId
Server ReviewService transaction
  -> lock/read submission
  -> 校验状态 pending/in_review
  -> 校验 revisionId == current revision
  -> 禁止自审
  -> 校验管理员范围
  -> 写 reviews
  -> ReviewEffectService.apply
      -> 首发：创建/更新 extension、version、scope
      -> 版本更新：发布新 version，废弃旧 version，切 current
      -> 元信息：更新 metadata
      -> 授权扩大：写新 scope
      -> 展示扩大：更新 visibility
      -> 重新上架：更新 status
  -> submission.status=APPROVED
  -> audit review.approve and extension.publish/scope/visibility event
  -> outbox notification
```

若状态已变化，返回 `review_already_processed`。

## 10.15 授权收缩

```text
Author/Admin
  -> POST /admin/extensions/{id}/scope/reduce
Server
  -> 校验权限
  -> ScopeChangeClassifier == REDUCTION
  -> 写新 scope 生效
  -> audit extension.scope.reduce
  -> notify affected installed users
Desktop next sync
  -> receive serverStateHints SCOPE_REDUCED
  -> local state mark SCOPE_REDUCED
```

超出新授权范围用户不能新增主操作和更新，但可停用、卸载、清理。

## 10.16 展示收缩

```text
Author/Admin
  -> POST visibility/reduce
Server
  -> PUBLIC_TO_ALL_LOGGED_IN -> AUTHORIZED_ONLY
  -> audit extension.visibility.reduce
  -> 从社区搜索和榜单排除
```

展示扩大必须创建 submission，不直接生效。

## 10.17 普通下架

```text
Actor
  -> POST /admin/extensions/{id}/delist
Server
  -> 校验作者/部门管理员/系统管理员权限
  -> status=DELISTED
  -> lifecycle event
  -> audit extension.delist
  -> notify installed users optionally
Desktop sync
  -> mark DELISTED
  -> forbid update/reinstall/new connect
```

存量本地内容不自动删除。

## 10.18 安全下架

```text
SystemAdmin
  -> POST /admin/extensions/{id}/security-delist
Server
  -> 校验系统管理员
  -> status=SECURITY_DELISTED
  -> 保存安全原因和处置建议
  -> audit extension.security_delist
  -> notify installed/connected/downloaded users
Desktop sync
  -> mark SECURITY_RISK
  -> apply configured existing usage policy
```

存量策略：

| 策略 | 客户端行为 |
|---|---|
| WARN_ONLY | 标记安全风险 |
| BLOCK_NEW_ENABLE_CONNECT | 禁止新增启用/接入/复制/更新 |
| FORCE_DISABLE_MANAGED_ITEMS | 停用或移除 Hub 托管项，保留备份和事件 |

离线客户端下一次在线同步后处理，不能承诺实时阻断。

## 10.19 归档

```text
Actor
  -> archive with reason and confirmation
Server
  -> 校验权限
  -> status=ARCHIVED
  -> audit
```

归档后不能重新上架、发布新版本、改授权或可见选项。

## 10.20 离线同步

```text
Desktop offline
  -> 允许本地查看/停用/卸载/清理
  -> LocalEventQueue 保存事件

Network restored
  -> POST /local-events/sync
Server
  -> 幂等处理
  -> 返回 accepted/rejected/ignored
  -> 返回 serverStateHints
Desktop
  -> 更新本地状态
  -> 对安全下架策略执行本地计划
```

## 10.21 客户端更新

```text
Desktop startup/login
  -> GET /client-updates/check
Server
  -> 根据当前版本、渠道、最低支持版本返回更新
Desktop
  -> create update download ticket
  -> 下载 installer
  -> verify SHA-256
  -> verify signature
  -> enqueue update event
  -> launch installer
  -> next start register new version
```

Hash 或签名失败：

- 不启动安装；
- 上报 update failed；
- 写设备事件和审计。

## 10.22 审计查询

```text
Admin
  -> GET /audit-logs?requestId=...
Server
  -> 按管理员范围过滤
  -> 返回审计列表
Admin
  -> GET /audit-logs/{id}
Server
  -> 返回详情和关联对象入口
```

审计导出 CSV 必须按范围过滤并写审计。


## 10.23 作者/维护人治理流程

### 作者普通下架

```text
Desktop
  -> POST /me/extensions/{extensionId}/delist
Server
  -> 校验当前用户是 author 或 maintainer
  -> 校验扩展状态可下架
  -> 状态改为 DELISTED
  -> 写 extension_lifecycle_events
  -> 写 audit extension.delist
  -> 通知已安装用户下架提示
```

### 作者授权收缩 / 展示收缩

```text
Desktop
  -> POST /me/extensions/{extensionId}/scope/reduce 或 visibility/reduce
Server
  -> 校验作者或维护人对象级权限
  -> 判断确实为 REDUCTION，不允许扩大伪装成收缩
  -> 直接生效
  -> 写审计
  -> 返回最新 scope / visibility
Desktop
  -> 刷新本地权限提示
```

### 作者授权扩大 / 展示扩大 / 新版本

```text
Desktop
  -> POST /me/extensions/{extensionId}/scope-expansion-submission
  -> 或 /visibility-expansion-submission
  -> 或 /version-submission
Server
  -> 创建 submission + revision
  -> 进入规则预审、AI 预审、人工审核
```

普通用户不得调用 `/admin/extensions/*`；管理员不得通过作者接口绕过管理端审计，所有领域服务最终都写同一类审计动作。

## 10.24 manual-download 受控下载流程

```text
Desktop
  -> GET /extensions/{extensionId}/plugin-definition
  -> 展示 manual-download 模式和说明摘要
  -> POST /download-tickets purpose=MANUAL_DOWNLOAD
Server
  -> 校验授权、版本、状态、可见选项和下载来源有效期
  -> 返回 Hub 代理 URL 或授权后内网 URL + sha256 + size
Desktop
  -> 下载到受控目录
  -> 校验 sha256 和 size
  -> 记录 PLUGIN_MANUAL_CONTROLLED_DOWNLOAD 本地事件
  -> 用户手动安装
  -> 用户标记已安装/已卸载
  -> 记录 PLUGIN_MANUAL_MARK_INSTALLED / PLUGIN_MANUAL_MARK_UNINSTALLED
Server
  -> 本地事件同步后更新 manual-download 受控下载用户数和用户标记已安装数，两者分开统计
```

## 10.25 指标聚合流程

```text
DownloadTicket / LocalEvent / ActivityEvent
  -> 事件落库并按幂等键去重
  -> MetricAggregationJob 按 DAY/WEEK/MONTH/ALL 聚合
  -> 排除 visibility_mode = AUTHORIZED_ONLY 的社区榜单项
  -> 生成 metric_period_aggregates
Community Home
  -> 读取类型化榜单
Admin Extension Detail
  -> 可读取全部权限范围内统计，包含 AUTHORIZED_ONLY 扩展
```

硬性规则：更新下载、审核预览下载、客户端更新下载、详情查看和连接检测次数不进入扩展下载量/安装量/使用量。
