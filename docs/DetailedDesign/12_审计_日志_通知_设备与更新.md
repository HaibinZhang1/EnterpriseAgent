# 12. 审计、日志、通知、设备与客户端更新

## 12.1 requestID 链路

每个请求必须有 requestID：

```text
Desktop/Web X-Request-ID
  -> RequestIdFilter
  -> Controller
  -> ApplicationService
  -> AuditLog
  -> RuntimeLog
  -> LocalEvent requestId
  -> DeviceEvent requestId
```

若客户端未提供，服务端生成：

```text
req_<ULID>
```

客户端本地执行产生的事件可使用原服务端 requestID 或新本地 requestID，并通过 `relatedRequestId` 关联。

## 12.2 服务端运行日志

运行日志输出到容器日志，至少包含：

```json
{
  "requestId": "req_01HX",
  "method": "POST",
  "path": "/api/submissions",
  "status": 200,
  "durationMs": 123,
  "actorId": "uuid",
  "errorCode": null
}
```

不得记录：

- Authorization；
- Token；
- 下载凭证；
- 密码；
- API Key；
- MCP 敏感变量值；
- 上传包完整内容。

## 12.3 审计写入接口

```java
record AuditCommand(
    String requestId,
    ActorSnapshot actor,
    ObjectRef object,
    String action,
    AuditResult result,
    String reason,
    JsonNode beforeSummary,
    JsonNode afterSummary,
    RequestMetadata requestMetadata
) {}
```

`AuditWriter.write(command)` 必须由业务事务调用。关键写操作若审计失败，建议业务失败，避免状态变更无法追踪。

## 12.4 审计摘要脱敏

`beforeSummary` 和 `afterSummary` 必须使用 `AuditRedactor`：

```java
JsonNode redact(JsonNode input) {
  remove("password");
  remove("token");
  remove("apiKey");
  remove("downloadTicket");
  maskPhone();
  maskSecretLikeValues();
}
```

示例：

```json
{
  "visibilityMode": {
    "before": "AUTHORIZED_ONLY",
    "after": "PUBLIC_TO_ALL_LOGGED_IN"
  },
  "mcpVariables": [
    {
      "name": "API_TOKEN",
      "sensitive": true,
      "value": "***REDACTED***"
    }
  ]
}
```

- 修改部门；
- 批量导入；

### 扩展和审核

- 提交发布申请；
- 撤回；
- 重新提交 revision；
- 规则预审完成；
- AI 预审完成/失败/不可用；
- 审核通过；
- 退回修改；
- 拒绝；
- 扩展上架；
- 下架；
- 安全下架；
- 重新上架；
- 归档；
- 授权收缩；
- 展示收缩；
- 维护人或归属部门转移。

### 客户端与本地事件

- 下载凭证发放；
- 权限拒绝；
- 不可见访问拒绝；
- 本地事件同步；
- MCP 配置写入失败上报；

- Plugin 安装失败上报；

- 更新 Hash 或签名校验失败上报。

## 12.6 审计查询权限

```java
AuditPredicate buildAuditPredicate(ActorContext actor) {
  if (actor.role == SYSTEM_ADMIN) return all();

  if (actor.role == DEPARTMENT_ADMIN) {
    Set<UUID> managedDepartments = departmentTree.selfAndDescendants(actor.departmentId);
    return objectDepartmentIn(managedDepartments)
        .or(actorIdEquals(actor.userId))
        .or(actorDepartmentIn(managedDepartments));
  }

  throw permission_denied;
}
```

普通用户不能进入管理端审计查询，但用户可在“我的提交”看到与自己相关的审核意见摘要，不等同于审计日志查询。

## 12.7 本地事件与审计关系

本地事件是客户端行为同步日志，审计是关键治理日志。

| 本地事件 | 是否进入审计 |
|---|---|
| Skill 安装成功 | 默认统计，不强制审计 |
| Skill 安装失败 | 可进入审计，至少可排障 |
| Hash 校验失败 | 必须审计 |
| MCP 配置写入失败 | 必须审计 |
| MCP 连接检测失败 | 必须审计 |
| Plugin 安装失败 | 必须审计 |
| 回滚失败 | 必须审计 |
| 停用/卸载成功 | 默认事件，不强制审计 |
| 权限拒绝 | 必须审计 |

## 12.8 通知模型

通知记录：

```json
{
  "userId": "uuid",
  "type": "SUBMISSION_APPROVED",
  "title": "发布申请已通过",
  "summary": "代码审查 Skill 1.0.0 已发布",
  "objectType": "SUBMISSION",
  "objectId": "uuid",
  "payload": {
    "extensionId": "code-review-skill"
  }
}
```

通知来源：

| 事件 | 接收人 |
|---|---|
| 提交成功 | 提交人 |
| 待审核 | 对应管理员 |
| 退回修改/拒绝/通过 | 提交人 |
| 新版本 | 已安装/已接入/已安装 Plugin 用户 |
| 授权收缩 | 受影响本地用户 |
| 安全下架 | 存量安装/接入/下载用户 |
| 客户端新版本 | 所有或策略目标设备用户 |
| AI 预审不可用 | 系统管理员 |
| 设备异常/更新失败 | 系统管理员或部门管理员范围内 |

通知本身不是审计，重要动作仍必须写审计。

## 12.9 客户端设备登记

### 设备 ID

客户端首次启动生成：

```json
{
  "deviceId": "dev_01HX...",
  "createdAt": "2026-05-06T10:00:00Z"
}
```

设备 ID 存储在本地，不随登录用户变化。用户登录后服务端更新当前用户关联。

### 登记请求

```json
{
  "deviceId": "dev_01HX",
  "hostnameHash": "sha256(hostname + salt)",
  "osVersion": "Windows 11 23H2",
  "arch": "x64",
  "clientVersion": "1.0.0"
}
```

主机名只存摘要。salt 可由客户端本地配置或服务端返回企业盐策略，避免直接反查。

## 12.10 设备事件

设备事件包括：

- 首次登记；
- 心跳；
- 版本变化；
- 本地事件同步失败；
- 更新检查；
- 更新包下载成功/失败；
- Hash 校验失败；
- 签名校验失败；
- 安装程序启动；
- 用户取消更新；
- 更新后首次启动。

心跳不进入审计；失败类关键事件进入审计。

## 12.11 客户端更新服务端设计

### 创建版本

系统管理员创建：

```json
{
  "version": "1.1.0",
  "buildNo": "20260506.1",
  "platform": "WINDOWS",
  "arch": "X64",
  "channel": "STABLE",
  "forceUpdate": false,
  "minSupportedVersion": "1.0.0",
  "releaseNotes": "修复本地安装问题"
}
```

### 上传或登记安装包

服务端校验：

- 文件存在；
- 包大小；
- SHA-256；
- 签名状态；
- 版本递增。

签名状态可由上传者提供构建系统摘要，也可由服务端调用系统工具校验。若服务端无法校验签名，必须标记 `UNKNOWN`，发布时按策略拒绝或要求人工确认。

### 发布状态机

```text
DRAFT -> PUBLISHED
PUBLISHED -> PAUSED
PUBLISHED -> WITHDRAWN
PAUSED -> PUBLISHED
PAUSED -> WITHDRAWN
WITHDRAWN -> * 禁止
```

已撤回版本不得重新发布。

## 12.12 客户端更新客户端流程

```ts
async function checkAndApplyUpdate() {
  const update = await api.checkUpdate(currentVersion, deviceId);
  if (!update.updateAvailable) return;

  const ticket = await api.createClientUpdateDownloadTicket(update.versionId);
  const installer = await downloadManager.download(ticket.downloadUrl);

  await hashVerifier.verify(installer, update.sha256);
  await signatureVerifier.verifyWindowsSignature(installer, update.signaturePolicy);

  await localEventQueue.enqueue(updateDownloadedEvent());
  await installerLauncher.launch(installer);
}
```

失败上报：

```json
{
  "eventType": "CLIENT_UPDATE_SIGNATURE_VERIFY_FAILED",
  "result": "FAILURE",
  "errorCode": "signature_invalid",
  "payloadSummary": {
    "targetVersion": "1.1.0",
    "packageHash": "64hex"
  }
}
```

## 12.13 安全下架存量策略

设置项：

```yaml
securityDelist:
  existingUsagePolicy: WARN_ONLY # BLOCK_NEW_ENABLE_CONNECT | FORCE_DISABLE_MANAGED_ITEMS
```

客户端收到安全下架提示：

```json
{
  "extensionId": "finance-mcp-server",
  "status": "SECURITY_DELISTED",
  "policy": "FORCE_DISABLE_MANAGED_ITEMS",
  "reasonSummary": "疑似泄露敏感配置"
}
```

执行：

- `WARN_ONLY`：只标记风险和通知。
- `BLOCK_NEW_ENABLE_CONNECT`：禁止新增启用/接入/更新。
- `FORCE_DISABLE_MANAGED_ITEMS`：生成停用/卸载托管项计划，执行并上报事件。

只能处理 Hub 托管项，不能承诺阻断目标工具直接加载的非托管内容。

## 12.14 审计导出

导出 CSV 规则：

1. 只导出管理员可见范围。
2. 导出参数写审计。
3. 默认脱敏手机号中间位。
4. IP 是否脱敏由系统设置控制。
5. 导出文件不包含密码、Token、API Key、下载凭证明文。

## 12.15 监控与排障建议

内网场景可先使用容器日志 + 管理端审计，不引入外部监控服务。建议提供：

- `/actuator/health`；
- `/actuator/info`；
- 数据库连接健康；
- 文件存储可写健康；
- AI 预审服务连通性健康；
- 最近失败本地事件统计；
- 最近更新失败统计。

所有健康接口不得泄露配置敏感值。


## 12.16 补充审计动作与统计边界

新增或明确以下审计动作：

| 动作 | 审计动作名 | 说明 |
|---|---|---|
| 临时上传被 submission revision 消费失败 | `upload.consume.failure` | 仅业务失败写审计；定时清理只写运行日志 |
| 作者普通下架 | `extension.delist` | actor 为作者/维护人 |
| 作者授权收缩 | `extension.scope.reduce` | 写 before/after scope 摘要 |
| 作者展示收缩 | `extension.visibility.reduce` | 写 before/after visibility 摘要 |
| 管理端安全下架 | `extension.security_delist` | 仅系统管理员 |
| 下载凭证签发 | `download.ticket.issue` | after_summary 包含 purpose，不含 ticket 明文 |
| manual-download Hash 失败上报 | `local_event.sync` | 本地事件 payload 只含脱敏路径摘要 |
| 设置版本冲突 | 可写运行日志，不强制审计 | 成功修改设置才必须审计 |

统计边界：

- `download.ticket.issue` 是审计，不直接等于榜单下载量；榜单由 `download_tickets.purpose` 和聚合任务计算。
- `local_event.sync` 是审计入口，真实统计由 `local_events` 和 `metric_period_aggregates` 计算。
- Star、搜索、详情查看默认进入普通统计事件，不进入审计，除非企业策略开启。

## 12.17 requestID 与幂等关联

关键写操作应在审计中记录：

```json
{
  "requestId": "req_01HX",
  "idempotencyKeyHash": "sha256:abcd1234",
  "action": "download.ticket.issue",
  "afterSummary": {
    "purpose": "MANUAL_DOWNLOAD",
    "objectType": "EXTERNAL_PLUGIN_FILE",
    "extensionId": "code-helper-plugin"
  }
}
```

不得记录：

- 原始 `Idempotency-Key`；
- 下载 ticket 明文；
- reset token 明文；
- API Key、密码、MCP secret；
- 未脱敏本地路径。
