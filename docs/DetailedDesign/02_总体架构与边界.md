## 2.2 部署拓扑

```text
企业内网
├─ Linux Server
│  ├─ Docker Compose
│  │  ├─ enterprise-agent-hub-api
│  │  ├─ enterprise-agent-hub-admin-static 或 Nginx
│  │  └─ postgres

│  ├─ EnterpriseAgentHub.exe
```

运行时不访问互联网。外网开发构建产物后，通过离线安装包、镜像 tar 和部署脚本复制到内网。

## 2.3 权威来源划分

| 数据/判断 | 权威来源 | 客户端可缓存 | 缓存用途 |
|---|---|---:|---|
| 用户角色 | 服务端 | 是 | 展示账号摘要，不能用于最终鉴权 |
| 部门树 | 服务端 | 是 | 离线展示历史信息 |
| 扩展状态 | 服务端 | 是 | 离线展示、本地提示 |
| 可见选项 | 服务端 | 是 | 缓存不可决定在线搜索结果 |
| 授权范围 | 服务端 | 是 | 仅支持离线存量启用判断，联网后刷新 |
| 下载凭证 | 服务端 | 否 | 短期一次性使用 |
| 包 Hash | 服务端 | 是 | 下载后校验 |
| MCP 敏感变量值 | 客户端 Secure Store | 否，不上服务端 | 本机配置渲染 |
| 本地安装状态 | 客户端，服务端接收事件副本 | 是 | 本地页、统计同步 |
| 审计日志 | 服务端 | 否 | 管理端排障 |
| 设备 ID | 客户端生成，服务端登记 | 是 | 设备关联与事件同步 |
| 客户端更新版本 | 服务端 | 是 | 检查更新和提示 |

## 2.4 信任边界

### 服务端信任边界

服务端永远不信任：

- 客户端传来的角色；
- 客户端传来的部门归属；
- 客户端声明的授权结果；
- 客户端声明的可见判断；
- 客户端声明的包 Hash；
- 客户端上报的敏感变量明文；
- 客户端 UI 是否禁用了按钮。

服务端必须在每个主操作接口重新计算：

```text
用户状态 + 部门状态 + 角色 + 扩展状态 + 版本状态 + 授权范围 + 可见选项 + 操作类型
```

### 客户端信任边界

客户端后端不信任 Renderer：

- Renderer 只能通过白名单 IPC 调用；
- IPC 入参必须使用 schema 校验；
- 文件路径必须规范化并限制在允许根目录或 Tool Adapter 声明路径内；
- Renderer 不能传入任意执行命令；
- Renderer 不能读取 Secure Store 中的敏感变量明文；
- Renderer 不能直接触发 shell 脚本执行。

## 2.5 运行时通信

### 桌面客户端到服务端

- 协议：HTTPS，内网证书由企业配置。
- 认证：登录后获取会话 Token，客户端保存到系统安全存储或受保护本地存储。
- requestID：客户端可生成 `X-Request-ID`，服务端缺失时生成并返回。
- 幂等：写接口使用 `Idempotency-Key`，本地事件同步使用事件级幂等键。
- 错误：稳定错误码 + 本地化可读 message + details。

### Web 管理端到服务端

- 协议：HTTPS。
- 鉴权：同 Auth Session，但管理端会话有效期和空闲超时应短于桌面端。
- 权限路由由前端辅助，服务端写操作最终校验。
- 管理端不调用客户端本地能力。

## 2.6 服务端运行时分层

```text
controller
  负责 HTTP 入参校验、分页参数、requestID 返回，不写业务状态

application
  编排用例、事务边界、调用权限、领域服务、审计、通知事件

domain
  规则判断、状态迁移、授权计算、可见过滤、包校验结果解释

infrastructure
  Repository、文件存储、AI 服务适配、邮件/通知、时钟、Hash、签名校验
```

依赖方向：

```text
controller -> application -> domain
application -> infrastructure interface
infrastructure implementation -> interface
```

禁止：

- Controller 直接调用 Repository 改状态。
- Domain 直接依赖 Spring Web 或数据库实现。
- Infrastructure 反向调用 Controller。
- 跨模块直接访问其他模块 Repository，必须通过应用服务或领域服务接口。

## 2.7 客户端后端运行时分层

```text
ipc
  白名单 IPC handler，schema 校验，错误统一包装

application
  业务用例：安装 Skill、写入 MCP、安装 Plugin、同步事件、检查更新

domain
  本地状态、执行计划、适配器能力、包校验、离线规则

infrastructure
  fs/path/crypto/sqlite/secure-store/http/update-installer/log
```

UI Renderer 只依赖 IPC facade，不得 import 本地执行层模块。

## 2.8 数据流示例：安装 Skill

```text
Renderer 调用 ipc.installSkill(extensionId)
  -> Desktop SkillService 校验本地环境
  -> Server DownloadTicket API 校验授权并签发凭证
  -> DownloadManager 下载到 temp
  -> HashVerifier 校验 SHA-256
  -> CentralStore 生成安装计划
  -> LocalExecutor 执行 temp -> version dir -> current pointer
  -> LocalEventQueue 写入 skill.install.success 或 failure
  -> LocalEventSync 在线同步
  -> Server LocalEvent 模块去重、统计、审计异常
```

## 2.9 数据流示例：发布审核

```text
Desktop Publish Backend 提交 payload + package
  -> Server SubmissionService 创建 submission + revision
  -> PackageService 存 temp 并校验
  -> SystemPrecheckService 规则预审
  -> AiPrecheckAdapter 可选调用内网 AI
  -> ReviewAssignmentService 计算审核归属
  -> 状态 pending_review
  -> Admin Review API 打开详情
  -> ReviewService 二次校验管理员权限和 submission 状态
  -> 生效版本/元信息/授权/可见变更
  -> AuditLog + Notification
```

## 2.10 模块间事件

建议使用单体内 Outbox 表避免事务后丢事件。

| 事件 | 触发模块 | 消费模块 | 说明 |
|---|---|---|---|
| `SubmissionCreated` | Submission | Precheck、Notification、Audit | 创建 revision 后触发 |
| `PrecheckCompleted` | Precheck | Review、Notification、Audit | 进入待审核 |
| `ReviewApproved` | Review | Extension、Notification、Audit、Statistics | 生效版本或治理变更 |
| `ExtensionDelisted` | Extension | Notification、LocalEvent、Audit | 通知存量用户 |
| `LocalEventAccepted` | Local Event | Statistics、Audit | 更新统计或记录异常 |
| `ClientUpdatePublished` | Client Update | Notification、Audit | 客户端检查更新可见 |
| `UserChanged` | Org/Auth | Auth Session、Audit | 失效会话 |

Outbox 只用于异步副作用，核心状态变更仍在同步事务中完成。
