# 07. 客户端后端架构

## 7.1 目标

客户端后端运行在 Electron Main Process 或受控 Node 子进程中，为 Renderer 提供安全、稳定、可测试的本地能力。它必须隔离 UI 与本机文件系统、工具配置、敏感变量和安装执行。

客户端后端负责：

- 会话和服务端 API 调用；
- 本地 Central Store；
- 包下载、Hash 校验；
- 本地工具和项目扫描；
- Skill 启用、停用、更新、卸载；
- MCP 配置写入、连接检测、更新、卸载；
- Plugin 受控安装、受控下载、更新、卸载、本地标记；
- 本地执行计划、备份、回滚、部分成功；
- 本地事件队列和同步；
- 设备登记、心跳、客户端更新检查；
- 本地日志和敏感信息脱敏。

客户端后端不负责：

- UI 布局、视觉、组件状态；
- Web 管理端功能；
- 服务端权限最终判断；
- Agent 执行内核；
- 自动部署 MCP Server。

## 7.3 本地数据目录

默认目录：

```text
%APPDATA%/EnterpriseAgentHub/
├── config.json
├── device.json
├── local.db
├── central-store/
│   ├── skills/
│   │   └── <extensionId>/<version>/
│   └── plugins/
├── mcp/
│   ├── configs/
│   └── variables/
├── temp/
├── backups/
├── events/
├── cache/
└── logs/
```

规则：

- `central-store` 保存受控副本。
- `temp` 保存下载中间文件，启动时清理过期临时文件。
- `backups` 保存回滚所需快照，按策略清理。
- `local.db` 保存本地状态、工具、项目、执行记录、事件队列。
- 敏感变量不写 `local.db` 明文；只保存 `secureRef`。

## 7.4 IPC 边界

Renderer 只能调用白名单 IPC。每个 IPC handler 必须：

1. 使用 schema 校验入参；
2. 校验会话或离线允许性；
3. 调用 Application Service；
4. 捕获异常并转换为统一错误；
5. 返回不含敏感明文的结果。

### IPC 命名建议

```text
auth.login
auth.logout
community.search
extension.getDetail
skill.install
skill.enablePlan
skill.applyEnablePlan
skill.update
skill.uninstall
mcp.getDefinition
mcp.renderConfigPreview
mcp.applyConfig
mcp.connectionTest
mcp.uninstallConfig
plugin.getDefinition
plugin.buildInstallPlan
plugin.applyInstallPlan
plugin.markManualInstalled
local.scanTools
local.scanProjects
local.syncEvents
device.register
update.check
update.downloadAndVerify
settings.getLocal
settings.updateLocal
```

### IPC 响应

```ts
type IpcResult<T> =
  | { success: true; requestId?: string; data: T }
  | { success: false; requestId?: string; error: DesktopError };
```

`DesktopError`：

```ts
interface DesktopError {
  code: string;
  message: string;
  details?: Record<string, unknown>;
  retryable: boolean;
}
```

## 7.5 API Client

`HubApiClient` 封装所有服务端调用。

职责：

- 自动附加 Authorization；
- 自动附加 X-Request-ID；
- 自动附加 X-Device-ID 和 X-Client-Version；
- 错误响应转换；
- Token 失效处理；
- 网络异常重试策略；
- 下载流处理；
- 不在日志记录 Authorization、ticket、敏感变量。

接口示例：

```ts
interface HubApiClient {
  login(req: LoginRequest): Promise<LoginResponse>;
  searchExtensions(req: SearchExtensionsRequest): Promise<PageResult<ExtensionSummary>>;
  getExtensionDetail(extensionId: string): Promise<ExtensionDetail>;
  createDownloadTicket(req: DownloadTicketRequest): Promise<DownloadTicketResponse>;
  getMcpDefinition(extensionId: string): Promise<McpDefinition>;
  getPluginDefinition(extensionId: string): Promise<PluginDefinition>;
  syncLocalEvents(req: LocalEventSyncRequest): Promise<LocalEventSyncResponse>;
  registerDevice(req: DeviceRegisterRequest): Promise<void>;
  checkUpdate(req: UpdateCheckRequest): Promise<UpdateCheckResponse>;
}
```

## 7.7 SessionStore

会话保存策略：

- Token 使用 Electron safeStorage、Windows Credential Manager 或受保护文件保存；
- 只在内存中短期持有明文；
- 登出清理 Token；
- Token 失效后清理本地会话，但不删除 Central Store；
- 日志不得输出 Token。

```ts
interface SessionStore {
  save(session: AuthSession): Promise<void>;
  get(): Promise<AuthSession | null>;
  clear(reason: string): Promise<void>;
}
```

## 7.8 DeviceIdStore

首次启动生成稳定 `deviceID`。

```json
{
  "deviceId": "dev_01HX...",
  "createdAt": "2026-05-06T10:00:00Z"
}
```

设备 ID 存在 `device.json`，不跟账号绑定。用户登录后服务端将 deviceID 关联当前用户。

## 7.9 CentralStore

### 目录结构

```text
central-store/skills/<extensionId>/
├── versions/
│   ├── 1.0.0/
│   └── 1.1.0/
└── current.json

central-store/plugins/<extensionId>/
├── versions/
└── current.json
```

`current.json`：

```json
{
  "extensionId": "code-review-skill",
  "version": "1.1.0",
  "sha256": "64hex",
  "installedAt": "2026-05-06T10:00:00Z"
}
```

### 安装原则

1. 下载到 `temp/downloads/<requestId>`。
2. 校验 Hash。
3. 解压或复制到 `versions/<version>.staging`。
4. 校验 `SKILL.md` 或安装清单。
5. 原子重命名到 `versions/<version>`。
6. 写 `current.json.tmp` 后替换。
7. 记录本地事件。

## 7.10 SecureVariableStore

MCP 敏感变量只保存在本机安全存储。

接口：

```ts
interface SecureVariableStore {
  saveSecret(scope: SecretScope, name: string, value: string): Promise<SecretRef>;
  getSecret(ref: SecretRef): Promise<string | null>;
  deleteSecret(ref: SecretRef): Promise<void>;
  listRefs(extensionId: string): Promise<SecretRef[]>;
}
```

`SecretRef` 示例：

```json
{
  "provider": "windows-credential-manager",
  "key": "EnterpriseAgentHub/mcp/finance-mcp/API_TOKEN",
  "displayName": "API_TOKEN"
}
```

日志和本地事件只能记录 `displayName` 和是否已设置，不记录 value。

## 7.11 离线能力控制

### 离线允许

- 查看本地缓存扩展；
- 查看工具和项目；
- 停用 Skill；
- 卸载 Skill；
- 停用或卸载托管 MCP 配置；
- 禁用或卸载托管 Plugin；
- 修改本地工具和项目路径；
- 若有有效授权缓存且未收到授权收缩/安全下架标记，可启用已安装 Skill 到已有允许范围。

### 离线禁止

- 社区搜索；
- 新安装 Skill；
- 新接入 MCP；
- 新下载 Plugin；
- 更新扩展；
- 提交发布申请；
- 请求新下载凭证；
- 新增配置写入。

实现上，每个 Application Service 开头调用：

```ts
offlinePolicy.assertAllowed(operation, localState, networkState)
```

## 7.12 本地事件队列

事件必须先本地持久化，再尝试同步。

```ts
interface LocalEvent {
  id: string;
  idempotencyKey: string;
  requestId?: string;
  extensionId?: string;
  version?: string;
  type: LocalEventType;
  result: 'SUCCESS' | 'FAILURE' | 'PARTIAL_SUCCESS';
  errorCode?: string;
  payloadSummary: Record<string, unknown>;
  occurredAt: string;
  syncStatus: 'PENDING' | 'SYNCED' | 'IGNORED' | 'REJECTED';
}
```

幂等键建议：

```text
<deviceId>:<operation>:<extensionId>:<version>:<targetHash>:<timestampBucketOrUuid>
```

同一次本地执行生成固定幂等键，重试同步不变。

## 7.13 客户端日志

日志分级：

- `app.log`：应用启动、服务端连接、同步；
- `executor.log`：执行计划、步骤结果、回滚；
- `adapter.log`：工具扫描和适配器能力；
- `update.log`：客户端下载、Hash、签名、安装程序启动。

所有日志统一经过 `Redactor`：

```ts
redactor.redact({
  token: "...",
  apiKey: "...",
  downloadTicket: "...",
  mcpSecretValue: "..."
})
```

输出：

```json
{
  "token": "***REDACTED***",
  "apiKey": "***REDACTED***"
}
```

## 7.14 安全设置

Electron 安全建议：

- Renderer 禁用 Node Integration。
- 启用 Context Isolation。
- 只通过 preload 暴露最小 API。
- IPC handler 白名单。
- 禁止 Renderer 传入任意本地路径执行写入，路径需由 Tool Adapter 或用户本地配置产生，并经过规范化。
- 打开外部链接前做协议白名单。
- 下载文件必须从服务端受控 URL 或登记的企业内网 URL 获取，并校验 SHA-256。

## 7.15 客户端后端测试边界

每个本地服务都应可在无 Electron Renderer 环境下单元测试：

- 使用临时目录；
- Mock HubApiClient；
- Mock ToolAdapter；
- Mock SecureVariableStore；
- 验证执行计划和回滚；
- 验证本地事件队列；
- 验证日志脱敏。

Renderer 端测试不在本设计范围内。
