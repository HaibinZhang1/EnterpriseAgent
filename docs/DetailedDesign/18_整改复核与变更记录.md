## 18.5 后续开发注意事项

1. 编码前应生成 OpenAPI 3.0 文件，并用契约测试锁定 Java DTO 与 TypeScript API client。
2. Tool Adapter 默认路径为 MVP 设计，实际实现必须通过 detect、dry-run 和用户配置验证，不得硬编码绕过 manifest。
3. 每个写接口必须明确审计动作、错误码、幂等键、权限判断和事务边界。
4. 每个本地文件操作必须经 ExecutionPlan，不允许 Renderer 或业务页面直接写文件。
5. 指标聚合必须从事件和下载凭证计算，不得从审计日志反推。
