# 需求 2：页面与资源管理

> 目标：定义重构后“本地”页的页面结构、字段、筛选、详情、操作和页面级验收标准。

---

## 1. 范围

本需求覆盖：

```text
概览页
智能体页
扩展页
项目页
工具集页
通用列表
通用详情抽屉
跨页面跳转与数据一致性
```

审计、事件、离线、授权、事务、备份、回滚见 `03_需求_审计事件离线授权与事务.md`。

## 2. 页面公共结构

### 2.1 左侧导航

```text
本地
├─ 概览
├─ 智能体
├─ 扩展
├─ 项目
├─ 工具集
└─ 审计与事件
```

### 2.2 顶部公共区域

所有本地子页面保留 EnterpriseAgent 当前视觉风格，顶部公共区域包含：

```text
页面标题
作用域选择：全局 / 全部项目 / 当前项目 / 指定智能体 / 指定智能体项目作用域
主要操作按钮：重新扫描 / 检查更新 / 运行审计 / 导入 / 新建
搜索框
筛选标签
在线 / 离线状态
待同步事件数量
最近扫描时间
最近审计时间
```

### 2.3 通用列表字段

```text
名称 | 类型 | 智能体/项目 | 权限 | 审计 | 状态
```

字段说明：

| 字段 | 说明 |
|---|---|
| 名称 | 主文本为资源名称，副文本展示路径、版本、来源、更新时间、Hash 摘要。 |
| 类型 | Agent、Settings、Rule、Memory、Subagent、Ignore、Skill、MCP、Plugin、Hook、CLI、Project、Kit、AuditFinding、LocalEvent。 |
| 智能体/项目 | 展示资源作用范围，如 `Codex / 全局`、`Claude Code / EnterpriseAgent`、`Cursor / 3 项目`。 |
| 权限 | 文件、网络、命令、环境变量、配置写入、数据库、Secret 等摘要。 |
| 审计 | 未审计、正常、低风险、需复核、高风险、安全风险，以及风险数和最近审计时间。 |
| 状态 | 按 `01` 中状态优先级聚合展示。 |

### 2.4 通用详情抽屉

所有列表行点击后打开右侧详情抽屉；智能体、项目、工具集可进入完整详情页。

详情抽屉统一包含：

```text
基本信息
作用域
路径 / 来源 / 版本 / Hash
绑定分布
权限摘要
静态审计结果
关联智能体
关联项目
关联工具集
最近本地事件
离线可用操作
联网受限操作
可用操作
```

敏感值不得明文展示。预览失败不得展示空白成功。

## 3. 概览页

### 3.1 目标

概览页回答：

```text
本地环境是否正常？
9 个内置智能体和自定义目录是否可用？
本机资源总量是多少？
哪些资源有更新、异常、授权收缩、安全风险、配置漂移？
哪些事件待同步？
当前是否离线？离线时哪些操作可用？
```

### 3.2 页面结构

```text
概览

┌──────────────────────────────────────────────┐
│ 本地环境状态                                  │
│ 智能体 9+N | 扩展 128 | 项目 12 | 工具集 8     │
│ 有更新 9 | 异常 3 | 安全风险 2 | 授权收缩 1   │
│ 在线/离线 | 待同步事件 6 | 最近扫描 16:03     │
│ [重新扫描] [运行审计] [检查更新] [查看异常]   │
└──────────────────────────────────────────────┘

智能体状态
风险与异常
最近事件
离线可用能力提示
```

### 3.3 操作

| 操作 | 离线可用 | 说明 |
|---|---:|---|
| 重新扫描 | 是 | 扫描本机路径、配置文件、本地数据库。 |
| 运行审计 | 是 | 使用本地静态审计规则。 |
| 检查路径 | 是 | 检查路径存在性、可读写性、符号链接能力。 |
| 检查更新 | 否 | 需要服务端最新版本和授权状态。 |
| 查看异常 | 是 | 跳转过滤后的审计与事件页。 |

### 3.4 验收标准

- [ ] 进入本地页默认显示概览页。
- [ ] 概览页展示 9 个内置智能体和自定义目录。
- [ ] 统计数量与列表过滤结果一致。
- [ ] 点击异常、安全风险、授权收缩、待同步事件能跳转到对应过滤列表。
- [ ] 最近事件与审计与事件页读取同一事件源。
- [ ] 离线时仍可重新扫描、路径检查、运行静态审计。
- [ ] 离线时检查更新、新安装、新接入、新下载等按钮置灰且说明原因。
- [ ] 不改变当前软件整体视觉风格。

## 4. 智能体页

### 4.1 目标

智能体页是完整 Agent Dashboard，不再只是工具检测列表。

智能体页回答：

```text
本机有哪些智能体？
每个智能体的全局配置在哪里？
每个智能体有哪些项目级配置？
有哪些 Settings、Rules、Subagents、Memory、Ignore Files、Skill、MCP、Plugin、Hook、CLI？
这些资源作用于全局还是项目？
是否存在权限风险、配置漂移、外部修改、路径异常、授权收缩或待同步事件？
```

### 4.2 智能体列表

筛选项：

```text
全部
已检测
未检测
未配置路径
异常
支持扩展
有项目配置
有审计风险
有配置漂移
macOS
Windows
自定义目录
```

列表字段固定为：

```text
名称 | 类型 | 智能体/项目 | 权限 | 审计 | 状态
```

### 4.3 智能体详情页

点击智能体进入详情页。

```text
Codex

状态：已检测
平台：macOS / Windows
作用域：全局 / 全部项目 / 当前项目
配置路径：...
项目配置：5 个
扩展：Skill 28 / MCP 3 / Plugin 1 / Hook 2 / CLI 4
规则：12
记忆：3
子智能体：5
Ignore Files：2
审计：正常 / 2 个需复核
事件：12 / 待同步 3

[总览] [设置] [规则] [子智能体] [记忆] [扩展] [Hook] [CLI] [文件] [审计] [事件]
```

### 4.4 Tab 要求

| Tab | 必须展示 | 必须支持操作 |
|---|---|---|
| 总览 | 基础信息、平台支持、路径、能力矩阵、资源摘要、权限摘要、审计摘要、离线能力、最近事件 | 重新扫描、运行审计、查看异常、查看事件 |
| 设置 | Settings / Configs 文件、Hash、漂移、外部修改 | 预览、差异、恢复上一版本、重新扫描、运行审计 |
| 规则 | 全局和项目 Rules、来源、作用域、关联 Kit | 启用、禁用、复制到项目、提升为全局、查看差异、恢复、审计 |
| 子智能体 | Subagent、引用 Skill/MCP/Plugin/Hook/CLI、权限摘要 | 启用、禁用、校验引用、查看配置事件、审计 |
| 记忆 | Memory 文件、敏感信息提示、关联项目、历史版本 | 预览、差异、恢复、脱敏审计 |
| 扩展 | 当前智能体下 Skill/MCP/Plugin/Hook/CLI | 启用、停用、接入、更新、卸载、审计，受授权和离线规则限制 |
| Hook | Hook 配置、触发时机配置、命令路径、环境变量名 | 启用、禁用、检查命令路径、静态审计、查看配置事件 |
| CLI | CLI 路径、版本来源、可执行命令摘要、关联关系 | 注册、启用、禁用、检查路径、读取版本信息、静态审计、查看配置事件 |
| 文件 | 该智能体相关原生文件聚合 | 预览、差异、审计、恢复 |
| 审计 | 当前智能体相关风险 | 查看 finding、跳转资源、标记已处理 |
| 事件 | 当前智能体配置事件 | 筛选、查看详情、同步状态 |

禁止：

```text
执行 Hook
执行 CLI
展示 Hook 运行时触发记录
展示 CLI 运行时调用记录
```

### 4.5 智能体页验收标准

- [ ] 智能体列表展示 9 个内置智能体和自定义目录。
- [ ] 未检测智能体显示“未检测”，不得显示为异常。
- [ ] 自定义目录完成 Profile 配置后，功能行为与内置智能体一致。
- [ ] 每个智能体有独立详情页。
- [ ] 智能体详情页至少包含总览、设置、规则、子智能体、记忆、扩展、Hook、CLI、文件、审计、事件。
- [ ] 列表字段统一为名称、类型、智能体/项目、权限、审计、状态。
- [ ] 支持全局和项目作用域切换。
- [ ] 支持外部修改检测和配置漂移提示。
- [ ] 支持资源详情预览和事件追踪。
- [ ] Hook 和 CLI 只展示配置事件，不展示运行时事件。
- [ ] macOS 和 Windows 均可完成扫描、预览、审计和受控写入。
- [ ] 不改变现有软件整体视觉风格。

## 6. 项目页

### 6.1 目标

项目页管理项目作用域下的完整智能体资源，不再只展示项目路径和 Skill 数量。

项目页回答：

```text
本机有哪些项目？
每个项目启用了哪些智能体？
每个项目下有哪些 Settings、Rules、Memory、Subagents、Ignore Files、Skill、MCP、Plugin、Hook、CLI？
项目路径是否正常？
项目级资源是否有风险、授权收缩或漂移？
项目级操作是否离线可用？
```

### 6.2 项目详情页

```text
EnterpriseAgent

项目路径：...
路径状态：正常
关联智能体：Codex / Cursor / OpenCode
项目级设置：3
项目级规则：12
项目级记忆：2
项目级子智能体：4
项目级 Ignore：1
项目级扩展：Skill 18 / MCP 3 / Plugin 1 / Hook 2 / CLI 4
审计风险：2
最近事件：12
待同步事件：3

[总览] [智能体] [设置] [规则] [记忆] [子智能体] [Ignore] [扩展] [Hook] [CLI] [审计] [事件]
```

### 6.3 项目路径状态

| 状态 | 说明 |
|---|---|
| 正常 | 路径存在且可写。 |
| 不存在 | 路径已移动或删除。 |
| 不可写 | 当前用户无写权限。 |
| 无效 | 路径为空、不是目录或无法访问。 |
| 配置漂移 | 项目配置被外部修改，与管理记录不一致。 |
| 权限不足 | 当前用户无法完成所选写入或扫描。 |

### 6.4 项目操作

```text
添加项目
修改项目路径
重新扫描项目
检查路径
启用智能体
启用本地资源
启用已安装扩展
应用工具集
运行项目审计
查看项目事件
删除项目管理记录
```

删除规则：

1. 如果项目下仍有已启用 Skill、MCP、Plugin、Hook、CLI、Rule、Memory、Subagent、Ignore、Kit 应用记录，不允许直接删除项目管理记录。
2. 用户必须先停用、卸载、解除接入、移除 Kit 应用或清理本地记录。
3. 删除项目只删除本地管理记录，不删除用户真实项目目录。
4. 删除项目必须生成执行计划和本地事件。
5. 路径不存在时，仍需展示关联资源清理指引。

### 6.5 项目页验收标准

- [ ] 项目页能展示项目级智能体资源总览。
- [ ] 项目详情包含智能体、设置、规则、记忆、子智能体、Ignore、扩展、Hook、CLI、审计、事件。
- [ ] 项目路径异常能在列表中直接看到。
- [ ] 项目级资源与智能体页、扩展页、工具集页数据一致。
- [ ] 删除项目时必须校验所有关联资源。
- [ ] 删除项目不得误删用户真实项目目录。
- [ ] 项目重新扫描、静态审计、路径检查、本地清理必须离线可用。
- [ ] 依赖服务端授权、下载凭证或最新版本的操作离线不可用。

## 7. 工具集页

### 7.1 目标

工具集 Kit 是可复用的本地资源组合包。Kit 可以包含：

```text
Skill
MCP Server
Plugin
Hook
CLI / 本地命令
Rule
Memory
Subagent
Settings 片段
Ignore 配置
```

Kit 可以应用到：

```text
智能体全局作用域
某个项目
某个智能体在某个项目下的作用域
自定义目录智能体作用域
```

### 7.2 Kit Manifest

```ts
interface KitManifest {
  kitId: string;
  name: string;
  version: string;
  description?: string;
  sourceType: 'local' | 'central-store' | 'imported';
  createdAt: string;

  supportedAgents: string[];
  supportedPlatforms: Array<'macos' | 'windows'>;

  resources: KitResourceRef[];
  permissionSummary: PermissionSummary;
  auditSummary: AuditSummary;

  requiredAuthorizations: AuthorizationRequirement[];
  resourceHashes: Record<string, string>;
  dependencies: KitDependency[];
  conflictPolicy: 'skip' | 'overwrite-managed' | 'prompt';
  rollbackPolicy: 'best-effort' | 'all-or-nothing-for-managed-overwrites';

  metadata: Record<string, unknown>;
}
```

### 7.3 工具集详情

详情必须展示：

```text
基本信息
包含资源
权限汇总
审计结果
授权收缩资源
Hash 异常资源
适用智能体
适用平台
部署分布
最近事件
离线可用操作
```

### 7.4 Kit 规则

1. Kit 不能绕过授权校验。
2. Kit 不能绕过 Hash 校验。
3. Kit 不能绕过配置备份和回滚。
4. Kit 内任一资源高风险时，Kit 整体审计状态不得显示为正常。
5. Kit 应用后必须记录应用目标、应用资源、版本、时间、结果和执行计划。
6. 移除 Kit 应用时，只移除 Kit 托管的资源，不破坏用户原有配置。
7. Kit 导入后必须先进入未应用状态，不能自动写入本地智能体目录。
8. 离线时可以导入本地 Kit、审计 Kit、预览 Kit、应用本机已有且授权缓存有效的资源。
9. Kit 中包含 Central Store 资源但本地无缓存时，必须联网获取授权和下载凭证。
10. Kit 应用必须拆分为多个资源变更结果，部分失败必须展示部分成功。

### 7.5 工具集页验收标准

- [ ] Kit 能包含 Skill、MCP、Plugin、Hook、CLI、Rule、Memory、Subagent、Settings 片段、Ignore 配置。
- [ ] Kit 能展示权限汇总和审计汇总。
- [ ] Kit 能展示应用到哪些智能体和项目。
- [ ] Kit 应用、移除、漂移、失败、回滚均产生本地事件。
- [ ] Kit 详情中的资源能跳转到对应资源详情。
- [ ] Kit 不得绕过授权、Hash、执行计划、备份和回滚。
- [ ] Kit 本地导入和静态审计必须离线可用。

## 8. 跨页面一致性要求

| 数据 | 概览 | 智能体 | 扩展 | 项目 | 工具集 | 审计与事件 |
|---|---:|---:|---:|---:|---:|---:|
| LocalResource | 读取 | 读取/管理 | 读取/管理 | 读取/管理 | 读取/组合 | 读取 |

要求：

- 同一资源在不同页面展示的状态必须一致。
- 页面跳转只改变过滤条件，不创建重复数据。
- 详情抽屉中的关联资源必须能跳转到对应资源详情。
- 空状态必须真实，不允许虚构示例数据作为运行时数据。
