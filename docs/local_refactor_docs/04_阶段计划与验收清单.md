# 本地页重构阶段计划与验收清单

> 说明：本阶段计划是开发拆分，不是产品裁剪。最终上线必须覆盖 `01`、`02`、`03` 三份需求中的全部要求。
> 使用方式：每完成一项验收，把 `- [ ]` 改为 `- [x]`，并在提交说明中附上测试证据。

---

## 1. 阶段总览

| 阶段 | 名称 | 目标 | 主要输出 |
|---:|---|---|---|
| 阶段一 | 本地资源模型、导航与迁移基础 | 建立统一模型、导航、状态、权限、事件基础，迁移旧 Skill/MCP/Plugin 数据 | LocalResource、ResourceBinding、导航、通用列表、现有数据迁移 |
| 阶段二 | 智能体 Dashboard 与 Path Profile | 完成 9 个智能体 + 自定义目录 Dashboard 和 macOS/Windows Path Profile | Agent Adapter、智能体详情 Tabs、扫描、预览、漂移 |
| 阶段三 | 扩展、项目、工具集页面 | 完成跨智能体资源管理、项目作用域和 Kit 组合复用 | 扩展页、项目页、工具集页、跨页面跳转 |
| 阶段四 | 审计、事件、离线、事务收口 | 完成安全审计、授权收缩、离线矩阵、ExecutionPlan、备份回滚、最终验收 | 18 条审计规则、事件源、离线行为、事务可靠性 |

## 2. 阶段一：本地资源模型、导航与迁移基础

### 2.1 目标

建立完整本地资源模型、统一列表字段、左侧导航、作用域、Adapter 能力声明、权限/审计/事件基础结构。

### 2.2 主要任务

```text
调整本地页导航为概览、智能体、扩展、项目、工具集、审计与事件。
移除旧 Skills / MCP / 插件一级入口。
增加 LocalResource、ResourceBinding、FileBackedResource 模型。
增加资源类型、作用域、来源、状态、权限、审计、事件枚举。
扩展 ToolAdapter / AgentAdapter 能力声明。
增加通用列表组件和详情抽屉。
增加作用域选择器和可复用筛选能力；筛选组件作为页面内二级能力复用，不要求在首屏渲染全局统一筛选栏。
迁移现有 Skill、MCP、Plugin 数据到新模型。
迁移现有本地事件到新 LocalEvent 模型。
所有空状态显示真实空态，不使用虚构资源作为运行时数据。
```

### 2.3 验收清单

- [x] 本地页左侧导航显示 6 个入口：概览、智能体、扩展、项目、工具集、审计与事件。
- [x] 旧 Skills / MCP / 插件一级入口已移除或跳转到新扩展页，不存在重复状态源。
- [x] 通用列表字段固定为名称、类型、智能体/项目、权限、审计、状态。
- [x] 已定义 LocalResource、ResourceBinding、FileBackedResource。
- [x] 已定义 LocalResourceType、SourceType、ScopeType、状态、权限、事件枚举。
- [x] 现有 Skill、MCP、Plugin 能映射为 LocalResource + ResourceBinding。
- [x] 未扫描资源显示未扫描或未审计，不得默认正常。
- [x] 列表状态聚合按安全、失败、路径、漂移优先。
- [x] 通用详情抽屉包含基本信息、路径、权限、审计、绑定分布、事件和操作边界。
- [x] 现有 ToolAdapter、Central Store、LocalExecutor、ExecutionPlan、BackupStore、RollbackManager、LocalEvent 已明确复用或扩展。
- [x] 运行时无 mock、demo、placeholder、随机数据。
- [x] 失败不会被吞掉并返回默认成功。

### 2.4 建议测试

- [x] 有旧 Skill/MCP/Plugin 数据时，启动后能在新扩展页看到。
- [x] 无任何本地资源时，页面显示真实空状态。
- [x] 手动制造路径缺失，状态列显示路径异常而不是已启用。
- [x] 手动制造审计高风险，状态列不被生命周期状态覆盖。

## 4. 阶段三：扩展、项目、工具集页面

### 4.1 目标

完成跨智能体资源管理能力：统一扩展页、项目作用域页、工具集组合页。

### 4.2 主要任务

```text
扩展页统一展示 Skill、MCP、Plugin、Hook、CLI。
增加扩展类型、智能体、项目、来源、权限、审计、平台等数据维度与筛选能力；首屏仅展示扩展页必要的搜索、类型、智能体、来源控制，其余维度进入详情、二级筛选或数据列。
展示扩展跨智能体和跨项目分布。
保留并迁移 Skill、MCP、Plugin 原有操作能力。
Hook 和 CLI 进入权限、审计和事件体系。

重构项目列表页。
新增项目详情页。
项目详情包含总览、智能体、设置、规则、记忆、子智能体、Ignore、扩展、Hook、CLI、审计、事件。
项目删除前校验所有关联资源。
项目路径异常生成本地事件。

新增工具集列表页和详情页。
Kit 支持包含 Skill、MCP、Plugin、Hook、CLI、Rule、Memory、Subagent、Settings 片段、Ignore 配置。
支持从当前智能体或当前项目生成 Kit。
支持 Kit 应用、移除、导入、导出、漂移检测、权限汇总、审计汇总。
Kit 应用、移除、漂移、失败、回滚生成本地事件。
```

### 4.3 验收清单

- [x] 扩展页能同时展示 Skill、MCP、Plugin、Hook、CLI。

- [x] 扩展详情能展示跨智能体和跨项目分布。
- [x] Skill、MCP、Plugin 原有安装、接入、启用、停用、卸载、更新能力已迁移。
- [x] Hook 和 CLI 已进入权限、审计和事件体系。
- [x] Hook 和 CLI 不展示运行时事件。
- [x] 项目页能展示项目级完整智能体资源。
- [x] 项目详情包含智能体、设置、规则、记忆、子智能体、Ignore、扩展、Hook、CLI、审计、事件。
- [x] 项目路径异常在列表中可见。
- [x] 删除项目时必须校验所有关联资源。
- [x] 删除项目不会删除真实项目目录。
- [x] 工具集页能展示包含资源、应用分布、权限汇总、审计汇总。
- [x] Kit 能包含 Skill、MCP、Plugin、Hook、CLI、Rule、Memory、Subagent、Settings 片段、Ignore 配置。

## 5. 阶段四：审计、事件、离线、事务收口

### 5.1 目标

完成全资源静态审计、权限透明、本地事件、离线优先、失败可观测、回滚可见、配置漂移收口。

### 5.2 主要任务

```text
审计页覆盖 Settings、Rules、Memory、Subagents、Ignore、Skill、MCP、Plugin、Hook、CLI、Kit、Project。
实现 18 条 HarnessKit 对齐静态审计规则。
实现 EnterpriseAgent 企业阻断规则。
增加 Trust Score 与 EnterpriseAgent 审计状态映射。
增加文件路径和行号定位。
增加敏感信息扫描。
增加 Hook 命令风险扫描。
增加 CLI 命令路径风险扫描。
增加 Subagent 引用资源完整性扫描。
增加 MCP 必填变量缺失扫描。
增加 Plugin 包 Hash 异常扫描。
增加授权收缩状态扫描。
增加配置漂移扫描。

统一事件表或本地事件存储。
增加所有资源类型事件。
删除 Hook 运行时触发事件和 CLI 运行时调用事件。
支持按智能体、项目、资源、类型、状态、同步状态过滤事件。
概览页、智能体页、项目页、工具集页、审计页读取同一事件源。

将本地写入、备份、回滚机制扩展到 Settings、Rules、Memory、Subagents、Ignore Files、Hook、CLI、Kit。
Hook 写入展示命令影响摘要。
CLI 注册展示命令路径、版本来源、权限摘要。
Memory、Rule、Subagent 修改支持旧版本快照。
Kit 应用拆分为多个资源变更结果。
部分成功不得显示为成功。
回滚失败必须展示失败原因。

补齐离线行为矩阵。
离线时本地扫描、静态审计、路径检查、预览、漂移检测、原生文件受控写入、本地清理可用。
离线时依赖服务端的新安装、新接入、新下载、更新、授权刷新不可用。
```

## 8. UX / 信息架构修正验收（设计简化补充）

> 来源：`.omx/plans/prd-local-page-design-simplification-20260617T021726Z.md` 与 `.omx/plans/test-spec-local-page-design-simplification-20260617T021726Z.md`。本节修正既有已勾选条目的解释边界：历史阶段验收中的“统一筛选 / 多维筛选”表示数据字段、详情展示和可复用筛选能力已具备，不表示必须在本地页首屏保留跨 Tab 的大型全局筛选卡片。

> 填写规则：以下条目在 renderer 测试、静态安全检查和至少一项视觉/人工证据完成前不得勾选为 `[x]`。执行证据需记录命令、结果和截图或截图工具阻塞原因。

### 8.1 首屏信息架构

- [x] 本地页不再在活动内容上方渲染大型标题/统计卡片；活动 Tab 内容应直接跟随紧凑标题或页面内工具栏。
  - 证据：`npm --prefix desktop test -- local-page.test.tsx` 通过，18 tests passed；`LocalPage.tsx` 渲染 `local-page-header` + active tab toolbar，不再渲染独立标题统计卡片。截图阻塞原因：renderer 依赖 Electron preload `window.enterpriseAgent`，普通 Vite 浏览器页面不是真实本地页。
- [x] `.local-filter-grid` 不再作为跨 Tab 全局筛选卡片出现在本地页首屏。
  - 证据：`rg -n "local-filter-grid" desktop/src/renderer/pages/LocalPage.tsx desktop/src/renderer/styles/app.css` 无输出；`desktop/tests/local-page.test.tsx` 静态断言 `app.css` 不含 `.local-filter-grid`。
