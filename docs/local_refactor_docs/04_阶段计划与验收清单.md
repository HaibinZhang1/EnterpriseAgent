# 本地页重构阶段计划与验收清单

> 说明：本阶段计划是开发拆分，不是产品裁剪。最终上线必须覆盖 `01`、`02`、`03` 三份需求中的全部要求。
> 使用方式：每完成一项验收，把 `- [ ]` 改为 `- [x]`，并在提交说明中附上测试证据。

---

## 1. 阶段总览

| 阶段 | 名称 | 目标 | 主要输出 |
|---:|---|---|---|
| 阶段一 | 本地资源模型、导航与迁移基础 | 建立统一模型、导航、状态、权限、事件基础，迁移旧 Skill/MCP/Plugin 数据 | LocalResource、ResourceBinding、导航、通用列表、现有数据迁移 |
| 阶段二 | 智能体 Dashboard 与 Path Profile | 完成 9 个智能体 + 自定义目录 Dashboard 和 macOS/Windows Path Profile | Agent Adapter、智能体详情 Tabs、扫描、预览、漂移 |
| 阶段三 | 扩展、项目、工具集页面 | 完成跨智能体资源管理、项目作用域和 Kit 组合复用 | 扩展页、项目页、工具集页、跨页面跳转 |
| 阶段四 | 审计、事件、离线、事务收口 | 完成安全审计、授权收缩、离线矩阵、ExecutionPlan、备份回滚、最终验收 | 18 条审计规则、事件源、离线行为、事务可靠性 |

## 2. 阶段一：本地资源模型、导航与迁移基础

### 2.1 目标

建立完整本地资源模型、统一列表字段、左侧导航、作用域、Adapter 能力声明、权限/审计/事件基础结构。

### 2.2 主要任务

```text
调整本地页导航为概览、智能体、扩展、项目、工具集、审计与事件。
移除旧 Skills / MCP / 插件一级入口。
增加 LocalResource、ResourceBinding、FileBackedResource 模型。
增加资源类型、作用域、来源、状态、权限、审计、事件枚举。
扩展 ToolAdapter / AgentAdapter 能力声明。
增加通用列表组件和详情抽屉。
增加作用域选择器和统一筛选组件。
迁移现有 Skill、MCP、Plugin 数据到新模型。
迁移现有本地事件到新 LocalEvent 模型。
所有空状态显示真实空态，不使用虚构资源作为运行时数据。
```

### 2.3 验收清单

- [x] 本地页左侧导航显示 6 个入口：概览、智能体、扩展、项目、工具集、审计与事件。
- [x] 旧 Skills / MCP / 插件一级入口已移除或跳转到新扩展页，不存在重复状态源。
- [x] 通用列表字段固定为名称、类型、智能体/项目、权限、审计、状态。
- [x] 已定义 LocalResource、ResourceBinding、FileBackedResource。
- [x] 已定义 LocalResourceType、SourceType、ScopeType、状态、权限、事件枚举。
- [x] 现有 Skill、MCP、Plugin 能映射为 LocalResource + ResourceBinding。
- [x] 未扫描资源显示未扫描或未审计，不得默认正常。
- [x] 列表状态聚合按安全、失败、路径、漂移优先。
- [x] 通用详情抽屉包含基本信息、路径、权限、审计、绑定分布、事件和操作边界。
- [x] 现有 ToolAdapter、Central Store、LocalExecutor、ExecutionPlan、BackupStore、RollbackManager、LocalEvent 已明确复用或扩展。
- [x] 运行时无 mock、demo、placeholder、随机数据。
- [x] 失败不会被吞掉并返回默认成功。

### 2.4 建议测试

- [x] 有旧 Skill/MCP/Plugin 数据时，启动后能在新扩展页看到。
- [x] 无任何本地资源时，页面显示真实空状态。
- [x] 手动制造路径缺失，状态列显示路径异常而不是已启用。
- [x] 手动制造审计高风险，状态列不被生命周期状态覆盖。

## 3. 阶段二：智能体 Dashboard 与 Path Profile

### 3.1 目标

完成 9 个内置智能体 + 自定义目录的 Agent Dashboard，覆盖 Settings、Rules、Subagents、Memory、Ignore Files、扩展、Hook、CLI、文件、审计、事件。

### 3.2 主要任务

```text
新增智能体列表页。
支持 9 个内置智能体和自定义目录。
支持 macOS / Windows 双端 Path Profile。
支持用户覆盖 Path Profile，如 CODEX_HOME、CLAUDE_CONFIG_DIR、OPENCODE_CONFIG_DIR。
支持 EA-managed fallback path。
新增智能体详情页。
新增详情页 Tab：总览、设置、规则、子智能体、记忆、扩展、Hook、CLI、文件、审计、事件。
实现 Settings / Configs 扫描与展示。
实现 Rules 扫描与展示。
实现 Memory 扫描与展示。
实现 Subagents 扫描与展示。
实现 Ignore Files 扫描与展示。
实现 Hook 扫描与展示。
实现 CLI 扫描与展示。
实现智能体维度下的 Skill、MCP、Plugin、Hook、CLI 聚合展示。
支持全局作用域和项目作用域切换。
支持文件内容预览、外部修改检测和配置漂移检测。
支持智能体级审计和事件列表。
```

### 3.3 验收清单

- [ ] 智能体列表展示 Claude Code、Codex、Gemini CLI、Cursor、Antigravity、Copilot、Windsurf、OpenCode、Hermes。
- [ ] 智能体列表展示自定义目录入口。
- [ ] macOS Path Profile 已覆盖 9 个内置智能体。
- [ ] Windows Path Profile 已覆盖 9 个内置智能体。
- [ ] 每个 Path Profile 都有真实检测根和 fallback 规则，不存在 placeholder path。
- [ ] 支持用户覆盖路径变量，例如 CODEX_HOME、CLAUDE_CONFIG_DIR、OPENCODE_CONFIG_DIR。
- [ ] 自定义目录配置 Agent Profile 后能力与内置智能体一致。
- [ ] 智能体详情页包含总览、设置、规则、子智能体、记忆、扩展、Hook、CLI、文件、审计、事件。
- [ ] Settings、Rules、Memory、Subagents、Ignore Files 均可扫描和展示。
- [ ] Hook 和 CLI 可扫描、展示权限、展示配置事件。
- [ ] CLI 版本信息读取不执行 CLI 命令。
- [ ] MCP stdio / command 类型只做静态配置检查，不启动进程。
- [ ] 文件预览支持脱敏和失败原因展示。
- [ ] 外部修改和配置漂移可见。
- [ ] 各 Tab 空状态明确，不显示假数据。
- [ ] 不支持或未配置的能力显示明确状态，不隐藏整体结构。

### 3.4 建议测试

- [ ] 在项目中创建 `.claude/`、`.codex/`、`.gemini/`、`.cursor/`、`.opencode/` 等目录，重新扫描后能被对应智能体识别。
- [ ] 在 Windows 路径中使用反斜杠和空格目录，扫描不报错。
- [ ] 删除一个受管文件后重新扫描，显示路径缺失或漂移。
- [ ] 修改受管文件内容后重新扫描，显示外部修改。
- [ ] 添加一个 Hook 命令配置，只进入审计，不执行。
- [ ] 添加一个 CLI path，只检查文件存在性，不执行 `--version`。

## 4. 阶段三：扩展、项目、工具集页面

### 4.1 目标

完成跨智能体资源管理能力：统一扩展页、项目作用域页、工具集组合页。

### 4.2 主要任务

```text
扩展页统一展示 Skill、MCP、Plugin、Hook、CLI。
增加扩展类型、智能体、项目、来源、权限、审计、平台筛选。
展示扩展跨智能体和跨项目分布。
保留并迁移 Skill、MCP、Plugin 原有操作能力。
Hook 和 CLI 进入权限、审计和事件体系。

重构项目列表页。
新增项目详情页。
项目详情包含总览、智能体、设置、规则、记忆、子智能体、Ignore、扩展、Hook、CLI、审计、事件。
项目删除前校验所有关联资源。
项目路径异常生成本地事件。

新增工具集列表页和详情页。
Kit 支持包含 Skill、MCP、Plugin、Hook、CLI、Rule、Memory、Subagent、Settings 片段、Ignore 配置。
支持从当前智能体或当前项目生成 Kit。
支持 Kit 应用、移除、导入、导出、漂移检测、权限汇总、审计汇总。
Kit 应用、移除、漂移、失败、回滚生成本地事件。
```

### 4.3 验收清单

- [ ] 扩展页能同时展示 Skill、MCP、Plugin、Hook、CLI。
- [ ] 扩展页能按类型、智能体、项目、来源、状态、审计、权限、平台筛选。
- [ ] 扩展详情能展示跨智能体和跨项目分布。
- [ ] Skill、MCP、Plugin 原有安装、接入、启用、停用、卸载、更新能力已迁移。
- [ ] Hook 和 CLI 已进入权限、审计和事件体系。
- [ ] Hook 和 CLI 不展示运行时事件。
- [ ] 项目页能展示项目级完整智能体资源。
- [ ] 项目详情包含智能体、设置、规则、记忆、子智能体、Ignore、扩展、Hook、CLI、审计、事件。
- [ ] 项目路径异常在列表中可见。
- [ ] 删除项目时必须校验所有关联资源。
- [ ] 删除项目不会删除真实项目目录。
- [ ] 工具集页能展示包含资源、应用分布、权限汇总、审计汇总。
- [ ] Kit 能包含 Skill、MCP、Plugin、Hook、CLI、Rule、Memory、Subagent、Settings 片段、Ignore 配置。
- [ ] Kit 应用、移除、漂移、失败、回滚均产生本地事件。
- [ ] Kit 不得绕过授权、Hash、执行计划、备份和回滚。
- [ ] 三个页面都使用统一列表字段。
- [ ] 所有详情抽屉能跳转关联资源。

### 4.4 建议测试

- [ ] 一个 Skill 绑定到多个智能体和项目时，扩展页展示“多分布”，点击能进入各绑定详情。
- [ ] 删除仍有关联资源的项目时，操作被阻断并显示清理指引。
- [ ] 导入本地 Kit 后，不自动写入智能体目录。
- [ ] 应用 Kit 时制造一个资源失败，页面显示部分成功而非成功。
- [ ] 移除 Kit 时只移除 Kit 托管资源，不破坏用户原有配置。

## 5. 阶段四：审计、事件、离线、事务收口

### 5.1 目标

完成全资源静态审计、权限透明、本地事件、离线优先、失败可观测、回滚可见、配置漂移收口。

### 5.2 主要任务

```text
审计页覆盖 Settings、Rules、Memory、Subagents、Ignore、Skill、MCP、Plugin、Hook、CLI、Kit、Project。
实现 18 条 HarnessKit 对齐静态审计规则。
实现 EnterpriseAgent 企业阻断规则。
增加 Trust Score 与 EnterpriseAgent 审计状态映射。
增加文件路径和行号定位。
增加敏感信息扫描。
增加 Hook 命令风险扫描。
增加 CLI 命令路径风险扫描。
增加 Subagent 引用资源完整性扫描。
增加 MCP 必填变量缺失扫描。
增加 Plugin 包 Hash 异常扫描。
增加授权收缩状态扫描。
增加配置漂移扫描。

统一事件表或本地事件存储。
增加所有资源类型事件。
删除 Hook 运行时触发事件和 CLI 运行时调用事件。
支持按智能体、项目、资源、类型、状态、同步状态过滤事件。
概览页、智能体页、项目页、工具集页、审计页读取同一事件源。

将本地写入、备份、回滚机制扩展到 Settings、Rules、Memory、Subagents、Ignore Files、Hook、CLI、Kit。
Hook 写入展示命令影响摘要。
CLI 注册展示命令路径、版本来源、权限摘要。
Memory、Rule、Subagent 修改支持旧版本快照。
Kit 应用拆分为多个资源变更结果。
部分成功不得显示为成功。
回滚失败必须展示失败原因。

补齐离线行为矩阵。
离线时本地扫描、静态审计、路径检查、预览、漂移检测、原生文件受控写入、本地清理可用。
离线时依赖服务端的新安装、新接入、新下载、更新、授权刷新不可用。
```

### 5.3 验收清单

- [ ] 审计与事件页可查看所有资源类型。
- [ ] 18 条 HarnessKit 对齐静态审计规则可运行。
- [ ] EnterpriseAgent 企业阻断规则可运行。
- [ ] 风险详情能定位到具体资源、绑定、路径和行号。
- [ ] 敏感信息只展示脱敏片段或 hash，不展示明文值。
- [ ] Hook 命令风险可审计，但不执行 Hook。
- [ ] CLI 命令路径风险可审计，但不执行 CLI。
- [ ] MCP stdio / command 配置可审计，但不启动进程。
- [ ] Plugin lifecycle scripts 可审计，但不执行脚本。
- [ ] Trust Score 与审计状态映射正确。
- [ ] 事件详情能反查资源、智能体、项目、工具集。
- [ ] 失败、部分成功、回滚状态清晰展示。
- [ ] 授权收缩适用于 Skill、MCP Server、Plugin。
- [ ] 授权收缩后禁止新增启用、接入、安装、下载、复制配置、连接检测或更新。
- [ ] 安全下架优先级高于授权收缩。
- [ ] 离线时本地扫描、静态审计、路径检查、预览、漂移检测可用。
- [ ] 离线时依赖服务端的新安装、新接入、新下载、更新、授权刷新不可用。
- [ ] 离线事件入队，联网后同步。
- [ ] 本地写入前有影响摘要。
- [ ] 写入配置前有备份。
- [ ] 所有危险操作必须生成 ExecutionPlan。
- [ ] 失败有明确错误信息。
- [ ] 部分成功不会显示为全部成功。
- [ ] 回滚成功和失败都可见。

- [ ] 外部修改和配置漂移可见。
- [ ] 空状态、异常状态、未检测状态、未审计状态清晰可见。
- [ ] 不存在失败但显示成功的交互。

## 7. Codex 开发约束

- [ ] 开发前先搜索现有 Local、ToolAdapter、Extension、Project、Audit、Event、Central Store、ExecutionPlan、Backup、Rollback、SecureStore 相关模块。
- [ ] 不新建与现有本地扩展管理割裂的数据结构。
- [ ] 不使用 mock、demo、placeholder、随机数据冒充真实扫描结果。
- [ ] 未扫描数据必须显示未扫描或未审计，不得默认正常。
- [ ] 失败不得吞掉，不得返回默认成功。
- [ ] 本地写入不得只改前端状态，必须有真实本地执行结果或明确失败原因。
- [ ] Hook、CLI、Memory、Subagent 必须进入权限和静态审计体系。
- [ ] Hook 和 CLI 不得执行，只做配置管理和静态检查。
- [ ] Kit 不得绕过授权、Hash 校验、备份和回滚机制。
- [ ] 所有操作必须考虑全局作用域、智能体作用域、项目作用域和智能体项目作用域。
- [ ] 所有本地危险操作必须有本地事件。
- [ ] 页面改造不得改变软件整体配色风格。
- [ ] 旧 Skills / MCP / 插件入口迁移后，不得留下重复入口或状态不一致页面。
- [ ] macOS 和 Windows 必须作为第一版验收平台同时完成。
- [ ] 自定义目录不得降级为低配目录，必须通过 Agent Profile 实现与内置智能体一致的行为。
