# 本地页重构索引文档

> 适用范围：`docs/RequirementDocument/09_desktop_local.md` 的拆分版重构文档。
> 目标：把单个超大文档拆成“索引 + 需求 + 阶段计划”，让后续 Codex 开发、评审、验收都能按文档逐项落地。

---

## 1. 需求背景

当前桌面端“本地”页原本偏向 `Skill / MCP Server / Plugin` 分类管理，无法完整承载本机 AI 智能体生态：不同智能体的配置、规则、记忆、子智能体、Ignore、Hook、CLI、本地工具集、项目作用域、授权收缩、安全审计、离线行为和本地事件都散落在不同概念中。

本次重构把“本地”页升级为：

```text
本地页
= 智能体 Dashboard
+ 统一扩展管理
+ 项目作用域管理
+ 工具集组合复用
+ 权限透明
+ HarnessKit 对齐的 18 条静态审计规则
+ EnterpriseAgent 企业级受控写入、授权、Hash、备份、回滚、本地事件和离线优先
```

重构不是照搬 HarnessKit 视觉风格，而是借鉴其能力组织方式：Agent Dashboard、统一扩展、权限透明、Trust Score、静态审计、Kits、项目作用域。EnterpriseAgent 仍保留服务端授权、Central Store、下载凭证、Hash 校验、ExecutionPlan、BackupStore、RollbackManager、SecureStore、本地事件同步等企业治理机制。

## 5. 共用术语

| 术语 | 说明 |
|---|---|
| Agent | 本机 AI 编程智能体或 IDE/CLI Agent，例如 Claude Code、Codex、Gemini CLI、Cursor、Antigravity、Copilot、Windsurf、OpenCode、Hermes、自定义目录。 |
| Agent Path Profile | 每个智能体在 macOS / Windows 下的真实路径发现规则、写入目标、备份策略、回滚策略、审计输入策略。 |
| LocalResource | 本地资源本体。只表达“资源是什么”，不表达“应用到哪里”。 |
| ResourceBinding | 资源分布关系。表达某个资源绑定到哪个智能体、项目、工具集或自定义路径。 |
| FileBackedResource | 文件型资源的扫描、Hash、mtime、漂移和预览模型。 |
| Central Store | EnterpriseAgent 原有受控包存储目录，用于 Skill 和部分 managed-package Plugin。 |
| Native Agent Directory | 智能体原生目录，例如 `.claude/`、`.codex/`、`.gemini/`、`.cursor/`、`.opencode/` 等。 |
| EA-managed fallback path | 当某智能体没有公开原生路径时，由 EnterpriseAgent 定义的受控补充目录。目录真实存在、可扫描、可备份、可回滚，不是 placeholder。 |
| ExecutionPlan | 本地写入前生成的执行计划，必须包含影响摘要、目标路径、备份、Hash、授权、离线可用性和回滚策略。 |
| Trust Score | 静态审计分数，0–100。用于把风险映射为 SAFE、LOW_RISK、NEEDS_REVIEW、HIGH_RISK、SECURITY_RISK。 |
| 授权收缩 | 服务端对 Skill / MCP Server / Plugin 的授权、可见范围、上架状态或安全下架状态发生变化，导致本地不得继续执行新增主操作。 |
| 配置事件 | 发现、启用、禁用、修改、漂移、授权收缩、路径异常、Hash 异常、备份、回滚、同步等本地配置管理事件。 |
| 运行时事件 | Hook 真实触发、CLI 真实执行、Agent 运行时工具调用等事件。本次本地页明确不做。 |

## 6. 核心边界

### 6.1 删除 Agent-first CLI 概念

文档、枚举、页面、事件、权限和审计中不再出现旧的 Agent-first CLI 概念。统一使用：

```text
CLI / 本地命令
LocalResourceType = CLI_COMMAND
```

CLI 只做本地命令资源管理：发现、注册、路径检查、版本元数据读取、权限摘要、静态审计、配置事件和关联关系展示。

### 6.2 Hook / CLI 不执行

本地页禁止执行：

```text
execute-cli
trigger-hook
shell-command
exec-script
download-and-run
arbitrary-write
```

Hook 和 CLI 只展示配置事件，不展示运行时触发记录或运行时调用结果。

### 6.3 CLI 版本检查边界

“检查版本”不得通过执行 `xxx --version`、`xxx -v`、PowerShell、cmd、bash、sh 或任意 shell 获取。允许方式只有：

1. 读取资源 manifest、package.json、extension manifest、安装清单、Central Store 元数据。
2. 读取文件属性、签名、Hash、mtime、OS 注册表或包管理器本地缓存，但不得启动目标命令。
3. 读取上次服务端同步或用户手动记录的版本字段。

页面文案建议使用“读取版本信息”，版本未知时显示：

```text
版本：未知
原因：禁止执行 CLI 获取版本输出
```

### 6.4 MCP 连接检测边界

MCP 检测分三类：

| 类型 | 本地页允许行为 | 禁止行为 |
|---|---|---|
| 纯配置检查 | 校验 JSON/TOML/YAML 结构、必填字段、变量引用、路径格式 | 无 |
| HTTP / SSE / Streamable HTTP | 用户触发、授权有效、联网可用时可做网络连通检测 | 离线时伪造成功；绕过授权检测 |
| stdio / command | 只做静态配置检查和命令路径风险审计 | 启动本地 server 进程；执行 command/args |

## 7. EnterpriseAgent 必须保留的企业治理能力

1. 服务端仍是扩展、版本、授权、可见范围、安全下架、普通下架、审计同步、下载凭证的权威来源。
2. 桌面客户端负责本机扫描、缓存、离线事件、受控写入、执行计划、备份、回滚和失败反馈。
3. Skill 和部分 Plugin 包仍进入 Central Store 或受控安装目录。
4. MCP Server 通过托管配置块写入智能体原生配置文件，敏感变量保存到 SecureStore。
5. Settings、Rules、Memory、Subagents、Ignore Files、Hook、CLI 使用智能体原生目录或 EA-managed fallback path 受控写入，不进入 Central Store。
6. 所有危险写操作都必须生成 ExecutionPlan。
7. Hash 校验失败时不得覆盖旧版本或旧配置。
8. 授权收缩后不得新增启用、接入、安装、下载、复制配置、配置写入、连接检测或更新；但必须允许停用、卸载、解除托管配置、清理敏感变量引用和清理本地记录。

## 9. 开发阅读约束

Codex 开发前必须先阅读：

1. 本索引文档。
2. 现有项目中 Local、ToolAdapter、Extension、Project、Audit、Event、Central Store、ExecutionPlan、Backup、Rollback、SecureStore 相关代码。
3. `01` 中的模型和迁移落点。
4. `04` 中的阶段计划与验收清单。

开发时不得：

- 新建一套与现有本地扩展管理割裂的数据结构。
- 用 mock、demo、placeholder、随机数据冒充真实扫描结果。
- 把失败返回默认成功。
- 只改前端状态而没有真实本地执行结果。
- 执行 Hook、CLI 或任意 shell 命令。
- 绕过服务端授权、Hash、备份、回滚和本地事件。
