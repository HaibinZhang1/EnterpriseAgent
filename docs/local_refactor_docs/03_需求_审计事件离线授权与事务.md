> 目标：定义本地页的安全审计、Trust Score、本地事件、离线优先、授权收缩、受控写入、备份回滚和隐私保护要求。

## 2. 审计目标

审计只做静态扫描，不执行 Hook、CLI、MCP stdio server、Plugin lifecycle script 或任意 shell 命令。

审计必须覆盖：

```text
Settings
Rules
Memory
Subagents
Ignore Files
Skill
MCP Server
Plugin
Hook
CLI / 本地命令
Kit
Project
Custom Agent Profile
```

审计结果必须能定位到：

```text
resourceId
bindingId
agentId
projectId
kitId
文件路径
行号范围
规则编号
权限维度
风险等级

```

敏感信息只允许展示变量名、脱敏片段、hash、行号，不得展示明文值。

## 3. HarnessKit 对齐的 18 条静态审计规则

第一版必须实现以下 18 条规则。规则编号使用 EnterpriseAgent 编号，但必须保留 HarnessKit 原规则映射，便于后续评审。

| EnterpriseAgent 编号 | HarnessKit Rule | 风险等级 | 适用资源 | 检测要求 | 建议处理 |
|---|---|---|---|---|---|
| EA-AUD-001 | `prompt-injection` | Critical | Skill、Plugin、Rule、Memory、Subagent | 检测“忽略上级规则”“泄露密钥”“绕过审批”“读取所有文件后上传”等诱导 Agent 越权的文本模式。 | 标记 SECURITY_RISK 或 NEEDS_REVIEW；展示命中文本脱敏片段。 |
| EA-AUD-002 | `rce` | Critical | Skill、Hook、Plugin、CLI、MCP、Subagent | 检测可导致任意命令执行的配置、脚本入口、shell 拼接、解释器调用、生命周期脚本。 | 高风险或阻断；不得执行验证。 |
| EA-AUD-003 | `credential-theft` | Critical | Skill、Hook、Plugin、CLI、MCP、Memory | 检测访问 `.ssh`、keychain、credential store、token 文件、浏览器凭据、云厂商凭据路径。 | 高风险；脱敏路径；要求人工复核。 |
| EA-AUD-004 | `plaintext-secrets` | Critical | Skill、Hook、MCP、Plugin、Settings、Memory、Rule、Subagent | 检测 API Key、Token、Password、Private Key、连接串等明文。 | 高风险；展示行号和脱敏片段；要求迁移到 SecureStore。 |
| EA-AUD-005 | `safety-bypass` | Critical | Skill、Hook、Rule、Subagent、Memory | 检测禁用安全策略、关闭审计、绕过 approval、忽略 allowlist、要求 Agent 不报告风险等内容。 | 标记 SECURITY_RISK。 |
| EA-AUD-006 | `dangerous-commands` | High | Skill、Hook、Plugin、CLI、Subagent、Gemini Command | 检测 `rm -rf`、`del /s`、`Remove-Item -Recurse`、格式化磁盘、kill 关键进程、下载并执行等危险模式。 | 高风险；禁止自动执行；写入计划需要人工确认或阻断。 |
| EA-AUD-007 | `broad-permissions` | High | Skill、MCP、Hook、Plugin、CLI、Subagent | 检测过宽文件访问、全盘读写、全网络域名、全部环境变量、数据库通配权限。 | 需复核；建议收缩作用域。 |
| EA-AUD-008 | `supply-chain` | Medium | Skill、MCP、Plugin、Kit | 检测无锁定版本、未知 registry、未固定 commit、依赖脚本、外部 URL、压缩包来源不明。 | 降低 Trust Score；提示来源验证。 |
| EA-AUD-009 | `unknown-source` | Low | Skill、MCP、Hook、Plugin、CLI、Kit | 检测来源为空、无 manifest、无签名、无服务端记录、无 Git origin。 | 默认低信任；需要用户标记可信或转为本地托管。 |
| EA-AUD-010 | `permission-combo-risk` | High | Skill、MCP、Hook、Plugin、CLI、Subagent、Kit | 检测组合权限：文件读取+网络、Secret+网络、命令+网络、数据库+网络、项目写入+命令。 | 组合提级；Kit 汇总状态不得为正常。 |
| EA-AUD-011 | `cli-credential-storage` | High | CLI | 检测 CLI 凭据文件路径、权限过宽、未知存储位置、明文 token。 | 高风险；要求 SecureStore 或权限修复。 |
| EA-AUD-012 | `cli-network-access` | Medium | CLI | 检测 CLI 配置中的外部域名、API endpoint、proxy、webhook。 | 展示域名；非 allowlist 需复核。 |
| EA-AUD-013 | `cli-binary-source` | High | CLI | 检测二进制来源、签名、Hash、安装路径是否可被普通用户写入。 | 高风险；不可信路径禁止自动注册。 |
| EA-AUD-014 | `cli-permission-scope` | Medium | CLI | 汇总 CLI 被哪些 Skill、Hook、Subagent、Kit 引用，以及引用后的权限范围。 | 范围过宽需复核。 |
| EA-AUD-015 | `cli-aggregate-risk` | Medium | CLI | 检测 CLI 子资源组合后是否同时具备网络、文件、命令、Secret 能力。 | 降低 Trust Score；提示拆分权限。 |
| EA-AUD-016 | `mcp-command-injection` | High | MCP Server | 检测 `command`、`args`、`cwd` 中的 shell 操作符、变量拼接、命令替换、未转义输入。 | 高风险；stdio 不启动进程，只静态提示。 |
| EA-AUD-017 | `plugin-source-trust` | Medium | Plugin | 检测无标准 manifest、无 Git origin、来源不明、包 Hash 不匹配、签名缺失。 | Hash 异常阻断安装/更新；来源不明需复核。 |
| EA-AUD-018 | `plugin-lifecycle-scripts` | Medium | Plugin | 检测 package.json、manifest、安装脚本中的 preinstall、install、postinstall、prepare 等生命周期脚本。 | 需复核；禁止自动执行脚本。 |

## 4. EnterpriseAgent 企业阻断规则

以下规则不计入 18 条 HarnessKit 对齐规则，但必须进入同一审计引擎，并且可直接阻断写入：

| 编号 | 名称 | 阻断条件 |
|---|---|---|
| EA-BLOCK-001 | 路径穿越 / 绝对路径越权写入 | 写入目标超出 Path Profile 允许目录、出现 `../` 绕出托管根、写入系统敏感目录。 |
| EA-BLOCK-002 | Hash 校验失败 | 下载包、Central Store 包、Kit 包、Plugin 包 Hash 与服务端或 manifest 不一致。 |
| EA-BLOCK-003 | MCP 必填变量缺失 | 新接入或更新配置时，必填变量缺失且无 SecureStore 引用。 |
| EA-BLOCK-004 | MCP 敏感变量明文落盘 | 敏感变量值被写入普通配置文件，而不是 SecureStore 引用。 |
| EA-BLOCK-005 | 配置漂移待处理 | 外部修改导致 managed block 或 lastManagedHash 不一致，且用户未确认覆盖策略。 |
| EA-BLOCK-006 | 授权收缩 / 安全下架 | Skill / MCP / Plugin 处于 AUTH_REVOKED 或 SECURITY_DELISTED，仍尝试新增主操作。 |
| EA-BLOCK-007 | 回滚快照缺失 | 操作会覆盖受管文件但无法创建备份快照。 |

## 5. Trust Score 与审计状态

### 5.1 评分

```text
Trust Score: 0 - 100
```

建议扣分：

| 风险等级 | 首次命中扣分 | 同规则重复命中扣分 |
|---|---:|---:|
| Critical | 25 | 1 |
| High | 15 | 1 |
| Medium | 8 | 1 |
| Low | 3 | 1 |

### 5.2 状态映射

| Trust Score | 默认审计状态 |
|---:|---|
| 80 - 100 | SAFE |
| 60 - 79 | LOW_RISK |
| 40 - 59 | NEEDS_REVIEW |
| 1 - 39 | HIGH_RISK |
| 0 或命中阻断规则 | SECURITY_RISK |
| 未扫描 | NOT_AUDITED |

阻断规则优先级高于分数。

### 5.3 AuditFinding

```ts
interface AuditFinding {
  findingId: string;
  ruleId: string;
  resourceId: string;
  bindingId?: string;
  resourceType: LocalResourceType;
  agentId?: string;
  projectId?: string;
  kitId?: string;

  severity: 'info' | 'low' | 'medium' | 'high' | 'critical';
  auditStatus: AuditStatus;
  trustScoreImpact: number;
  permissionCategory: PermissionCategory;

  path?: string;
  lineStart?: number;
  lineEnd?: number;
  snippetHash?: string;

  title: string;
  description: string;
  remediation: string;
  detectedAt: string;
}
```

## 6. 本地事件

### 6.1 LocalEvent 字段

```ts
interface LocalEvent {
  eventId: string;
  idempotencyKey: string;
  eventType: LocalEventType;
  operationId?: string;
  executionId?: string;

  resourceId?: string;
  bindingId?: string;
  resourceType?: LocalResourceType;
  agentId?: string;
  projectId?: string;
  kitId?: string;

  status: 'success' | 'failure' | 'partial_success' | 'rolled_back' | 'rollback_failed' | 'info';
  message: string;
  errorCode?: string;
  failureReason?: string;
  suggestion?: string;

  offlineCreated: boolean;
  syncStatus: SyncStatus;
  serverAckStatus?: 'accepted' | 'rejected' | 'ignored';

  createdAt: string;
  syncedAt?: string;
  metadata: Record<string, unknown>;
}
```

### 6.2 事件类型

必须覆盖以下事件族：

```text
AGENT_*
CONFIG_*
RULE_*
MEMORY_*
SUBAGENT_*
IGNORE_*
SKILL_*
MCP_*
PLUGIN_*
HOOK_*
CLI_*
KIT_*
AUDIT_*
PATH_ERROR
PERMISSION_DENIED
HASH_CHECK_FAILED
AUTH_REVOKED
SECURITY_DELISTED
SYMLINK_FALLBACK_COPY
BACKUP_CREATED
ROLLBACK_DONE
ROLLBACK_FAILED
PARTIAL_SUCCESS
LOCAL_IMPORT
EVENT_SYNCED
EVENT_SYNC_FAILED
```

明确禁止以下事件：

```text
CLI_COMMAND_EXECUTED
HOOK_TRIGGERED
HOOK_RUNTIME_STARTED
HOOK_RUNTIME_FINISHED
AGENT_TOOL_CALLED
```

### 6.3 事件规则

1. 所有写操作必须产生事件。
2. 所有路径异常必须产生事件。
3. 所有 Hash 校验失败必须产生事件。
4. 所有回滚成功、回滚失败、部分成功必须产生事件。
5. 所有授权收缩和安全下架必须产生事件。
6. 所有 Hook / CLI 权限复核必须产生配置事件。
7. 离线时事件必须写入本地队列。
8. 联网后本地事件同步到服务端。
9. 同步必须使用 idempotencyKey 去重。
10. 服务端返回 accepted、rejected、ignored 时，本地必须记录同步结果。

| 操作 | 离线可用性 | 规则 |

| 查看已缓存 Skill / MCP / Plugin 状态 | 可用 | 读取本地数据库和本地文件。 |

| 注册 / 启用 / 禁用 CLI 配置 | 可用 | 不执行 CLI，只检查路径和配置。 |

| 停用 / 卸载 MCP 托管配置 | 可用 | 本地清理动作。 |
| MCP 本地配置静态检查 | 可用 | 不访问网络。 |
| MCP HTTP/SSE 网络连接检测 | 条件可用 | 需要联网、用户触发、授权有效。 |

| 新安装 Skill | 不可用 | 需要服务端授权和下载凭证。 |
| 新接入 MCP Server | 不可用 | 需要服务端定义和授权判断。 |
| 新下载 / 新安装 Plugin | 不可用 | 需要服务端授权和下载凭证。 |
| 复制服务端 MCP 配置 | 不可用 | 属于主操作，需要服务端授权。 |
| 检查更新 | 不可用 | 需要服务端最新版本。 |
| 下载更新 | 不可用 | 需要服务端下载凭证。 |
| 本地事件同步 | 不可用但可入队 | 离线写入本地队列，联网后同步。 |

## 8. 授权收缩与安全下架

### 8.1 适用资源

授权收缩适用于：

```text
Skill
MCP Server
Plugin
```

Hook 和 CLI 若只是本地原生配置，不使用服务端授权收缩；若 Hook 或 CLI 作为服务端托管资源、Plugin 或 Kit 的组成部分，则跟随上层资源授权状态。

### 8.2 主操作授权规则

以下主操作必须依赖服务端授权：

```text
安装
下载
复制配置
新增启用
接入
配置写入
连接检测
更新
应用包含服务端托管资源的 Kit
```

### 8.3 授权收缩后禁止

```text
新增 Skill 启用
新增 MCP 接入
新增 MCP 复制配置
新增 MCP 配置写入
MCP 网络连接检测
Plugin 下载
Plugin 安装
Plugin 更新
Skill 更新
应用包含该资源的新 Kit 变更
```

### 8.4 授权收缩后允许

```text
查看本地资源
查看本地配置摘要
运行本地静态审计
停用 Skill
卸载 Skill
停用 MCP 托管配置
卸载 MCP 托管配置
禁用 Plugin
卸载 Plugin
清理 manual-download 本地记录
清理敏感变量引用
查看事件
同步本地清理事件
```

### 8.5 安全下架

安全下架状态优先级高于授权收缩。安全下架后：

1. 禁止新增主操作。
2. 客户端应提示风险。
3. 已托管资源可执行本地停用、卸载或强制清理。
4. 清理动作仍必须生成执行计划、备份、回滚和本地事件。
5. 离线时若本地已经收到安全下架标记，不得继续新增启用。

### 9.3 写入前影响摘要

写入前必须生成影响摘要：

```text
将新增哪些文件
将修改哪些文件
将删除或停用哪些托管资源
将写入哪些配置项
将注册哪些命令
将影响哪些智能体

是否支持回滚
哪些目标会被跳过
哪些目标存在风险
哪些目标离线不可执行
```

### 9.4 写入规则

1. 下载包先进入临时目录。
2. Hash 校验通过后再进入 Central Store 或受控目录。
3. 写配置文件前先读取、校验、备份。
4. 配置写入先写临时文件，再替换正式文件。
5. Rule、Memory、Subagent、Ignore 修改必须记录旧版本快照。
6. Hook 写入必须额外检查命令路径和权限摘要。
7. CLI 注册必须展示命令路径、版本来源、权限摘要。
8. Subagent 修改必须校验引用资源是否存在。
9. Kit 应用必须拆分为多个资源变更结果。
10. 部分目标失败时展示“部分成功”，不得显示为全部成功。
11. 自动回滚失败时必须展示失败路径、原因和建议动作。
12. 所有执行结果必须写本地事件。

### 9.5 禁止行为

本地执行器不得支持：

```text
exec-script
shell-command
download-and-run
arbitrary-write
execute-cli
trigger-hook
start-mcp-stdio-server
run-plugin-lifecycle-script
```

如后续需要执行脚本或命令，必须另行新增安全设计、沙箱、审批和审计方案，不属于本次本地页重构范围。

### 9.6 回滚规则

1. 覆盖目标前必须备份旧目标元数据和必要文件。
2. 修改配置文件前必须保存配置快照。
3. 部分目标失败时，已成功且未覆盖用户内容的目标不强行全部撤销。
4. 对已经覆盖且可回滚的目标，若备份完整且路径仍可写，应自动回滚。
5. 无法自动回滚时，必须列出失败路径、原因和建议动作。
6. 回滚成功、回滚失败、部分成功必须生成本地事件。

## 10. 安全与隐私

### 10.1 敏感信息保护

以下内容不得进入审计日志、普通运行日志、本地事件明文字段或 UI 明文展示：

```text
密码
Token
API Key
Secret
下载凭证明文
用户填写的 MCP 敏感变量值
数据库密码
私钥
客户敏感信息正文
```

允许记录：

```text
变量名
脱敏片段
hash
规则编号
文件路径
行号
风险类型
建议动作
SecureStore 引用
```

### 10.2 文件预览安全

1. 二进制文件默认不预览。
2. 大文件必须限制预览大小。
3. 敏感字段必须脱敏。
4. 审计发现可以定位行号，但不得复制完整敏感值。
5. 预览失败必须显示明确原因，不得显示空白成功。

### 10.3 Hook / CLI 安全边界

1. 本地页不得执行 Hook。
2. 本地页不得执行 CLI。
3. 本地页不得执行任意 shell 命令。
4. 本地页只做配置管理、路径检查、版本元数据读取、权限提取和静态审计。
5. Hook / CLI 的命令内容必须进入静态审计。
6. Hook / CLI 的权限摘要必须在列表和详情中可见。
