# 14. 管理端：部门与用户

## 14.1 页面目标

部门与用户用于维护组织结构、用户账号和管理员归属。

本模块只回答三个问题：

1. 部门结构是否正确？
2. 用户账号和角色是否正确？
3. 当前管理员能管理哪些下级部门和管理员？

## 14.2 页面结构

`部门与用户` 是 Web 管理端一级入口，内部拆成两个任务型子页：

```text
部门与用户
├─ 组织结构
└─ 用户账号
```

规则：

- 组织结构页以部门树、部门详情、部门增删改查为主。
- 用户账号页以用户搜索、筛选、导入、冻结、重置密码、角色变更为主。
- 两个子页可以通过部门筛选联动，但不得在同一屏堆叠所有部门和用户操作。

## 14.3 组织结构页

### 部门树字段

- 部门名称。
- 部门 ID。
- 上级部门。
- 状态。
- 用户数量。
- 部门管理员数量。
- 下级部门数量。

### 部门操作

- 新增下级部门。
- 修改部门名称。
- 停用部门。
- 恢复启用部门。
- 删除部门。
- 查看部门用户。
- 查看部门管理员。

### 部门停用规则

部门暂不支持移动。停用部门用于临时使部门及其资源失效。

规则：

- 停用部门后，该部门用户不可登录，已有会话必须失效。
- 停用部门后，归属部门为该部门的活跃扩展在社区视为失效，不再展示，不允许新安装、接入、复制配置、下载或更新。
- 停用部门不删除用户、扩展、版本、审核记录和审计日志。
- 停用部门前必须展示影响用户数、部门管理员数、活跃扩展数和待审核申请数。
- 已停用部门可由具备权限的管理员恢复启用，恢复后用户登录和扩展可用性按当前权限重新计算。

### 部门删除规则

删除部门前必须满足：

- 部门下没有用户。
- 部门下没有下级部门。
- 部门下没有活跃扩展归属。
- 部门下没有待审核申请。
- 删除操作人具备该部门管理权限。

不满足时必须提示阻塞原因，并提供跳转到用户账号或扩展管理的入口。

## 14.4 用户账号页

### 用户列表字段

- 姓名。
- 手机号。
- 部门。
- 角色。
- 状态。
- 最近登录时间。
- 创建时间。
- 最近更新时间。

### 筛选项

- 部门。
- 角色。
- 状态。
- 关键词。
- 最近登录时间。

### 用户状态

| 状态 | 说明 |
|---|---|
| 正常 | 可登录 |
| 冻结 | 不可登录，已有会话失效 |
| 删除 | 不可登录，默认不在列表展示 |

### 用户角色

系统只允许三类角色：

- 普通用户。
- 部门管理员。
- 系统管理员。

不得出现审核员、超级管理员、运营管理员等额外角色。

## 14.5 新增用户

新增用户字段：

- 姓名。
- 手机号。
- 初始密码或重置链接策略。
- 所属部门。
- 角色。

规则：

- 手机号必须唯一，但手机号只是用户属性唯一键；系统技术主键必须使用不可变 userID。
- 所属部门必须有效。
- 部门管理员只能在管理范围内新增普通用户或下级部门管理员。
- 部门管理员不得新增系统管理员。
- 系统管理员可新增任意角色，但不得破坏最后一个可用系统管理员规则。

## 14.6 修改用户

可修改：

- 姓名。
- 手机号。
- 所属部门。
- 角色。
- 状态。

规则：

- 修改用户部门后，用户已有会话必须失效。
- 修改用户角色后，用户已有会话必须失效。
- 部门管理员只能把用户调整到自己管理范围内部门。
- 部门管理员只能管理下级部门管理员，不得管理同级或上级部门管理员。
- 上级部门管理员可以管理下级部门管理员，用于完成部门和管理员的增删改查。

## 14.7 批量导入

批量导入用于系统管理员或具备权限的部门管理员创建用户，不用于批量修改既有用户角色。

支持字段：

- 姓名。
- 手机号。
- 部门路径。
- 角色。
- 初始状态。

规则：

- 导入前先预校验，不得直接写入。
- 预校验展示成功数量、失败数量和失败原因。
- 部门不存在时导入失败，不自动创建部门。
- 允许在新增用户时指定角色，包括普通用户和部门管理员；系统管理员还可创建系统管理员，但不得破坏最后一个可用系统管理员规则。
- 批量导入不允许对已存在用户执行批量角色变更；若手机号或 userID 已存在，该行默认失败并提示人工处理。
- 部门管理员导入范围不得超出其管理范围，且不得创建系统管理员。
- 导入成功和失败摘要写审计日志。

## 14.8 冻结与解冻

冻结用户：

- 用户不可再登录。
- 用户已有会话失效。
- 用户发布的扩展不自动下架。
- 审计日志保留。

解冻用户：

- 恢复登录能力。
- 不自动恢复旧会话。

## 14.9 删除用户

删除用户为逻辑删除。

规则：

- 删除后用户不可登录。
- 删除后用户历史发布、审核和审计记录保留用户快照。
- 不允许删除最后一个可用系统管理员。
- 不允许部门管理员删除自己账号。
- 不允许部门管理员删除同级、上级或管理范围外管理员。

## 14.10 重置密码

规则：

- 管理员可为权限范围内用户重置密码。
- 重置后目标用户已有会话失效。
- 不得在审计日志中记录密码明文。
- 可要求用户下次登录修改密码。
- 重置密码链接、临时密码或验证码必须有有效期，且只能使用一次。

## 14.11 账号安全规则

- 密码不得明文存储，必须使用强哈希算法。
- 登录失败必须限频，并按策略锁定账号或要求额外校验。
- 管理端会话必须设置有效期和空闲超时。
- 用户被冻结、删除、部门停用、角色变化、部门变化、密码重置或主动修改密码后，已有会话必须失效。
- 手机号修改必须校验唯一性，并保留变更前后摘要。
- 审计日志不得记录密码、Token、验证码、重置链接或敏感密钥明文。

## 14.12 部门管理员限制

部门管理员可以：

- 管理自己部门及下级部门普通用户。
- 管理下级部门管理员。
- 将下级部门普通用户设置为部门管理员。
- 取消下级部门管理员身份。
- 新增、修改、停用、删除下级部门。

部门管理员不得：

- 管理系统管理员。
- 管理同级或上级部门管理员。
- 管理横向部门管理员。
- 将任何用户设置为系统管理员。
- 冻结、删除或降权自己账号。
- 删除某个部门的最后一个可用部门管理员，除非该部门仍可被上级部门管理员覆盖管理。

## 14.13 系统管理员限制

系统管理员可以全局管理用户和部门，但系统必须保证：

- 不允许删除最后一个可用系统管理员。
- 不允许冻结最后一个可用系统管理员。
- 不允许通过批量导入或批量修改破坏最后一个可用系统管理员规则。

## 14.14 审计要求

以下动作必须写审计日志：

- 新增、修改、停用、恢复启用、删除部门。
- 新增、修改、冻结、解冻、删除用户。
- 重置密码。
- 修改用户角色。
- 修改用户部门。
- 批量导入。
- 设置或取消部门管理员。
- 设置或取消系统管理员。

## 14.15 验收标准

- Web 管理端有独立的“部门与用户”入口。
- “部门与用户”内部拆成“组织结构”和“用户账号”两个任务型子页。
- 角色只允许普通用户、部门管理员、系统管理员。
- 部门管理员可管理下级部门管理员。
- 部门管理员不得管理同级、上级、横向部门管理员或系统管理员。
- 系统不得允许删除或冻结最后一个可用系统管理员。
- 用户部门或角色变化后，已有会话必须失效。
- 部门和用户管理写操作必须写审计日志。
