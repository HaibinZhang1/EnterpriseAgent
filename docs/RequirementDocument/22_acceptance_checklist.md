# 22. 验收清单

## 22.1 产品边界验收

- 桌面客户端只包含 Agent、社区、本地三个一级入口。
- Agent 具体能力不在本需求文档验收范围内，由独立 Agent 设计文档验收。
- 发布入口是社区中的按钮，不是一级页签。
- Web 管理端独立于桌面客户端。
- 桌面客户端不包含部门管理、用户管理、审核、审计、客户端更新发布或系统设置管理页面。
- 系统运行时不依赖互联网。
- 不考虑旧测试版本迁移兼容，按全新系统构建。

## 22.2 角色验收

- 系统只存在普通用户、部门管理员、系统管理员三类角色。
- 普通用户不能进入 Web 管理端。
- 部门管理员只能访问管理范围内数据。
- 系统管理员可访问全局数据。
- 部门管理员可管理下级部门管理员。
- 部门管理员不得管理同级、上级、横向部门管理员或系统管理员。
- 服务端必须校验所有管理端写操作权限。
- 用户使用不可变 userID 作为技术主键，手机号作为唯一用户属性。
- 密码不得明文存储，登录失败必须限频或锁定，会话必须可失效。
- 不允许删除或冻结最后一个可用系统管理员。

## 22.3 社区验收

- 点击社区后首先进入社区首页。
- 社区首页包含搜索框、发布按钮、Skill 榜单区、MCP Server 榜单区、Plugin 榜单区和类型入口。
- 社区首页不展示卡片网格。
- 提交搜索或点击探索入口后进入搜索结果页。
- 搜索结果按 Skill、MCP Server、Plugin 分组，不混排。
- 三类扩展有各自筛选项。
- Skill、MCP Server、Plugin 卡片字段符合类型模板。
- 卡片操作不堆叠复杂按钮。
- 详情抽屉承载完整信息。

## 22.4 可见选项验收

- 发布向导必须有“仅授权范围内展示”勾选项。
- 勾选项说明必须包含：`勾选表示无权限用户不可见、不进入榜单统计`。
- 默认不勾选时，已发布扩展对所有登录用户可浏览。
- 默认不勾选时，未授权用户可查看详情，但安装、下载、复制配置、配置写入、新增启用、更新等主操作不可用。
- 勾选后，未授权用户搜索不到、榜单看不到、直接链接不能打开详情。
- 勾选后，该扩展不进入热榜、Star 榜、下载量榜、安装量榜或使用量榜。
- 展示扩大必须进入审核。
- 展示收缩可直接生效，但必须写审计日志。

## 22.5 扩展展示验收

- Skill 卡片展示作者、部门、Star、下载量、版本和本机安装状态。
- MCP Server 卡片展示作者、部门、Star、使用量（去重成功配置写入用户数）、版本和本机接入状态。
- Plugin 卡片展示作者、部门、Star、下载量或安装量、版本和本机安装状态。
- 详情页展示比卡片更多的信息。
- 未授权但可见的扩展展示不可操作原因。
- 已下架、已安全下架、已归档扩展不在普通社区展示。

## 22.6 发布验收

- 发布第一步必须选择 Skill、MCP Server 或 Plugin。
- 发布向导不超过四步。
- 发布向导自动提取可识别字段。
- 发布向导包含授权范围、可见选项和风险声明。
- Skill 发布必须校验 `SKILL.md`。
- MCP Server 发布必须校验接入方式、合法 transport 组合、连接方式、配置模板、变量 schema、权限声明和数据访问说明。
- Plugin 发布必须校验目标工具、安装模式、安装清单或手动安装说明、权限声明和风险说明。
- 服务端必须先创建 submission 和第一个 revision，状态进入 validating。
- 服务端必须执行最终校验。
- 服务端必须执行规则预审。
- 启用 AI 系统预审时，服务端必须生成预审结果并进入审核详情。
- 发布成功后状态显示为待审核。
- 退回修改后重新提交必须生成新的 submission revision，历史 revision 不得被覆盖。

## 22.7 审核验收

- 审核列表支持按类型、申请类型、部门、提交人、风险等级、AI 预审状态、授权范围、可见选项筛选。
- 审核详情展示顶部摘要、本次申请内容、系统校验、AI 系统预审、类型化预览、授权与可见影响、风险声明、历史记录和操作。
- AI 系统预审结果不得自动通过或拒绝申请。
- Skill、MCP Server、Plugin 审核预览内容不同。
- 展示扩大申请必须可审核。
- 禁止自审。
- 已处理申请不能重复处理。
- 审核通过、退回修改、拒绝都写审计日志。

## 22.8 本地能力验收

- 本地页只包含已安装扩展、工具、项目三个子入口。
- Skill 支持安装、更新、启用、停用、卸载和本地事件同步。
- MCP Server 支持安装到工具、配置写入、连接检测、编辑变量、更新配置、停用、卸载和本地事件同步；MCP 使用量固定为去重成功配置写入用户数。
- Plugin 支持 managed-package、config-plugin、manual-download 三种安装模式。
- managed-package 支持下载、Hash 校验、安装、更新、卸载和回滚。
- config-plugin 支持配置写入、启用、禁用、更新、卸载和回滚。
- manual-download 支持受控下载、查看安装说明、标记已安装和标记已卸载。
- 本地文件和配置写入必须生成执行计划。
- 本地失败必须展示失败原因和下一步动作。
- 离线时不得执行依赖服务端授权的新安装、新接入、新下载、配置写入或更新；若本机已有有效授权缓存且未收到授权收缩或安全下架标记，可启用已安装 Skill；本地停用和卸载始终可执行。

## 22.9 管理端验收

- 管理端默认进入概览。
- 管理端包含概览、审核、扩展管理、部门与用户、审计日志、客户端设备、客户端更新、系统设置。
- 部门与用户内部拆成组织结构和用户账号两个任务型子页。
- 部门暂不支持移动；部门停用后用户不可登录，归属该部门的活跃扩展在社区视为失效。
- 批量导入允许新增部门管理员，但不得对已存在用户批量变更角色。
- 扩展管理支持 Skill、MCP Server、Plugin 类型切换。
- 扩展管理可查看授权范围、可见选项、统计、审核记录、AI 预审历史和审计入口。
- 系统管理员可安全下架扩展。
- 安全下架后存量用户必须收到提示或本地状态标记。
- 客户端设备页面可查看设备列表、版本分布、最近在线和更新事件；部门管理员只能查看管理范围内用户设备。

## 22.10 包安全验收

- 路径穿越、绝对路径、解压逃逸必须被拒绝。
- 压缩炸弹风险必须被识别。
- 包大小、解压后大小、文件数必须校验。
- 含脚本、可执行文件、疑似密钥必须在审核详情中提示风险。
- MCP env 示例不得包含真实密钥。
- MCP accessType 与 transport 必须满足合法组合：remote-http/streamable-http、remote-sse/sse、local-command/stdio。
- local-command 的 command、args、工作目录和环境变量摘要必须在审核详情中展示，高风险执行器必须提示风险。
- Plugin 安装清单必须包含安装、更新、卸载和回滚说明，或明确不可回滚风险。
- 预览不得执行脚本或打开未知二进制。
- AI 系统预审输入必须脱敏。

## 22.11 审计验收

- 管理端可查询审计日志。
- 审计日志包含 requestID、操作人快照、对象、动作、结果、变更前后摘要、设备 ID 和时间。
- 审计日志不得记录密码、Token、API Key、真实密钥、下载凭证明文。
- 发布、系统预审、审核、授权变更、可见选项变更、下架、安全下架、归档、客户端更新必须写审计日志。
- MCP 配置写入失败、连接检测失败、Plugin 安装失败必须可追踪。
- 审计详情可复制 requestID。
- 审计日志可导出 CSV，且按管理员数据范围过滤。

## 22.12 部署验收

- 服务端支持 Docker Compose 离线部署。
- PostgreSQL 数据持久化正常。
- 服务端文件存储卷持久化正常。
- 桌面客户端 Electron + React Windows x64 安装包可离线安装运行。
- Web 管理端 React 静态资源可在内网访问。
- 客户端、Web 管理端、服务端运行时不访问互联网。
- 备份脚本可执行。
- 恢复脚本可执行。
- 离线镜像导入脚本可执行。
- 健康检查通过。

## 22.13 技术架构验收

- 服务端是账号、部门、权限、扩展、版本、发布申请、审核、统计和审计的权威来源。
- 服务端采用模块化单体，业务事务和审核生效必须一致。
- PostgreSQL 表结构支持授权范围、可见选项、MCP 定义、Plugin 安装清单、submission revision、扩展维护人或归属部门转移、客户端设备和更新事件。
- API 具备 requestID、错误码、分页、权限校验和幂等能力。
- 文件存储不静默覆盖历史包。
- 客户端 Central Store、本地执行层和 Tool Adapter 边界明确。
- 本地事件同步可去重、可排障。
- 审计日志和运行日志可通过 requestID 关联。
- AI 系统预审可配置内网服务并支持不可用降级。

## 22.14 文案与体验验收

- 页面标题描述任务，而不是解释系统。
- 首屏不放超过两行的说明文字。
- 卡片描述最多两行。
- 卡片最多保留一个主操作和一个轻操作。
- 按钮文案是动作。
- 错误提示包含动作语义、原因和下一步操作。
- 空状态包含当前状态和推荐下一步动作。
- 危险操作有确认弹窗。
