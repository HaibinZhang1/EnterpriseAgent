# 9. 桌面端：本地

## 9.1 页面目标

本地页用于管理用户本机已经安装、接入或发现的扩展，以及本机工具和项目的启用关系。

本地页只回答四个问题：

1. 我本机有哪些扩展？
2. 哪些 MCP Server 已经接入到工具？
3. 哪些 Plugin 已安装或已记录？
4. 哪些工具和项目配置了扩展？

## 9.2 页面结构

本地页包含三个子入口：

```text
已安装扩展  |  工具  |  项目
```

不得包含社区搜索、发布管理、审核、部门、用户等内容。

## 9.3 已安装扩展

### 范围

已安装扩展覆盖三类：

- Skill：安装到 Central Store，并可启用到工具或项目。
- MCP Server：写入受支持工具的 MCP 配置，或记录授权用户复制配置后的本地接入状态。
- Plugin：按安装模式记录受控安装、受控下载、配置写入或手动安装状态。

### 列表字段

已安装扩展列表展示：

- 类型。
- 名称。
- 本地版本。
- 服务端最新版本。
- 状态。
- 启用目标、接入工具或安装目标数量。
- 最近更新时间。
- 异常摘要。

### 筛选项

- 全部。
- 已安装。
- 已启用。
- 已接入。
- 有更新。
- 异常。
- 授权收缩。
- 安全风险。
- 类型：Skill / MCP Server / Plugin。

### Skill 操作

- 查看详情。
- 更新。
- 启用范围。
- 停用。
- 卸载。
- 重新扫描。

### MCP Server 操作

- 查看详情。
- 安装到工具。
- 查看配置指引。
- 复制配置。
- 编辑本地变量。
- 连接检测。
- 更新配置。
- 停用配置。
- 卸载配置。
- 修复配置。

### Plugin 操作

- 查看详情。
- 安装。
- 下载文件。
- 查看安装说明。
- 更新。
- 启用。
- 禁用。
- 卸载。
- 修复安装。
- manual-download 模式允许用户标记已安装或已卸载。

## 9.4 未托管 Skill 扫描

客户端可扫描已配置工具和项目目录中的本地 Skill。

规则：

- 仅根目录包含 `SKILL.md` 的目录可识别为 Skill。
- 扫描发现项必须标明来源路径。
- 与服务端同 ID 且 Hash 一致时，可提示“纳入管理”。
- 与 Central Store 同名但 Hash 不一致时，显示冲突。
- 纳入管理只发生在本机，不创建服务端发布记录。

## 9.5 工具页

### 页面目标

管理本机 AI 工具及其 Skill、MCP Server、Plugin 的适配能力。

### 支持工具

当前支持：

- Codex。
- Claude。
- Cursor。
- Windsurf。
- opencode。
- 自定义目录。

### 列表字段

- 工具名称。
- 检测状态。
- 配置路径。
- Skill 目标路径。
- MCP 配置路径。
- Plugin 目标路径。
- 已启用 Skill 数量。
- 已接入 MCP Server 数量。
- 已安装 Plugin 数量。
- 最近扫描时间。
- 异常摘要。

### 适配能力

每个工具适配器必须声明：

- 是否支持 Skill 启用。
- 是否支持 MCP 配置写入。
- 是否支持 Plugin 安装。
- 支持 symlink、copy、配置写入还是受控下载。
- 是否支持连接检测。
- 是否支持回滚。

### 操作

- 扫描工具。
- 添加自定义工具。
- 修改路径。
- 禁用工具配置。
- 删除工具配置。
- 查看已启用 Skill。
- 查看已接入 MCP Server。
- 查看已安装 Plugin。

### 删除规则

如果工具下仍有已启用 Skill、已接入 MCP Server 或已安装 Plugin，不允许直接删除工具配置。用户需先停用、卸载或解除相关扩展。

## 9.6 项目页

### 页面目标

管理本机项目及项目级 Skill 启用关系。MCP Server 和 Plugin 是否支持项目级配置由具体 Tool Adapter 声明。

### 列表字段

- 项目名称。
- 项目路径。
- Skill 目标路径。
- 路径状态。
- 已启用 Skill 数量。
- 最近更新时间。

### 路径状态

| 状态 | 说明 |
|---|---|
| 正常 | 路径存在且可写 |
| 不存在 | 路径已移动或删除 |
| 不可写 | 当前用户无写权限 |
| 无效 | 路径为空、不是目录或无法访问 |

### 操作

- 添加项目。
- 修改项目路径。
- 修改 Skill 目标路径。
- 删除项目。
- 查看项目已启用 Skill。

### 删除规则

如果项目下仍有已启用 Skill，不允许直接删除项目。用户需先停用相关 Skill。

## 9.7 Skill 启用范围弹窗

Skill 详情中的“启用范围”打开弹窗。

弹窗展示：

- 可用工具列表。
- 可用项目列表。
- 当前已启用目标。
- 路径异常目标。
- symlink / copy 模式说明。

操作：

- 勾选目标。
- 应用变更。
- 查看失败原因。

## 9.8 MCP Server 接入弹窗

MCP Server 详情中的“安装到工具”打开接入弹窗。

弹窗展示：

- 可写入 MCP 配置的目标工具。
- 当前工具的 MCP 配置路径。
- 接入方式和 transport。
- 必填变量。
- 敏感变量标记。
- 配置摘要预览。
- 连接检测按钮。
- 写入影响摘要。

操作：

- 选择目标工具。
- 填写变量。
- 保存敏感变量到本机安全存储。
- 预览配置摘要。
- 应用配置。
- 执行连接检测。
- 查看失败原因。

规则：

- 用户未在授权范围内时不得打开新增写入流程；若本机已有托管配置，仍允许停用或卸载该托管配置。
- 目标工具不支持 MCP 写入时只允许授权用户复制配置。
- 写入前必须备份原配置。
- 写入后必须记录托管配置项 ID，卸载时只移除托管项，不破坏用户原有配置。

## 9.9 Plugin 安装弹窗

Plugin 详情中的“安装”打开安装弹窗。

弹窗展示：

- 安装模式。
- 目标工具。
- 兼容版本检测结果。
- 文件包摘要或配置清单摘要。
- 将写入的位置摘要。
- 是否可回滚。
- 风险提示。

操作按安装模式变化：

- managed-package：下载、Hash 校验、安装、更新、卸载。
- config-plugin：写入配置、启用、禁用、更新、卸载。
- manual-download：下载、打开安装说明、标记已安装、标记已卸载。

规则：

- 用户未在授权范围内时不得下载、安装或更新；若本机已有托管安装或手动安装记录，仍允许卸载或清理本地记录。
- 目标工具不兼容时不得执行自动安装。
- 安装前必须生成执行计划。
- 安装失败必须展示失败位置、原因和建议动作。

## 9.10 本地事务与回滚

Windows 本地文件和配置操作必须按计划执行，不得只显示乐观成功。

### 能力检测

客户端启动或首次启用时检测：

- symlink 创建能力。
- 目标目录可写性。
- 目标配置文件可读写性。
- MCP 配置路径是否存在。
- Plugin 目标路径是否存在。
- 目标路径是否存在冲突文件。
- 企业策略是否禁用开发者模式或符号链接。

检测结果写入本地环境状态。symlink 不可用时默认使用 copy，并记录降级原因。

### 执行计划

启用、接入、安装、停用、更新、卸载前必须生成执行计划：

- 将要新增的目标。
- 将要移除的目标。
- 将要覆盖的目标。
- 将要修改的配置项。
- 将要跳过的目标。
- 每个目标使用 symlink、copy、配置写入还是受控下载。
- 失败后的回滚动作。

用户确认危险操作时，确认弹窗展示影响摘要，不展示低层文件细节。

### 原子写入

- 下载包先写入临时目录。
- Hash 校验通过后再进入 Central Store 或插件受控目录。
- Central Store 更新使用新版本目录，成功后再切换当前版本指针。
- 向工具或项目分发时先写临时目标，成功后再替换正式目标。
- 写入配置文件时先读取、校验、备份，再写入临时文件，最后替换正式文件。
- 不得在 Hash 未通过时覆盖旧版本。

### 备份与回滚

- 覆盖目标前必须备份旧目标的元数据和必要文件。
- 修改配置文件前必须保存配置快照。
- 部分目标失败时，已成功目标不强行全部撤销；客户端展示“部分成功”。
- 对已经覆盖但后续失败的目标，若备份完整且路径仍可写，应自动回滚。
- 无法自动回滚时，必须列出失败路径、原因和建议动作。
- 回滚、部分成功和降级 copy 都必须生成本地事件。

## 9.11 本地事件

以下动作必须生成本地事件：

- Skill 安装、更新、启用、停用、卸载。
- MCP Server 接入、配置写入、连接检测、更新、停用、卸载、复制配置。
- Plugin 下载、安装、更新、启用、禁用、卸载、标记已安装、标记已卸载。
- symlink 降级 copy。
- 路径异常。
- 权限拒绝。
- Hash 校验失败。
- 配置写入失败。
- 连接检测失败。
- 回滚成功。
- 回滚失败。
- 部分成功。
- 本地导入。

联网后，本地事件同步到服务端。

## 9.12 离线行为

离线时允许：

- 查看已缓存本地扩展。
- 在本机已有有效授权缓存且未收到授权收缩或安全下架标记时，启用已安装 Skill。
- 停用已启用 Skill。
- 查看已接入 MCP Server 配置摘要。
- 停用本机已接入 MCP Server 的托管配置。
- 查看已安装 Plugin 状态。
- 禁用或卸载本机已托管 Plugin。
- 查看工具和项目配置。
- 修改本地工具和项目路径。

离线时不允许：

- 搜索社区。
- 安装新 Skill。
- 接入新 MCP Server。
- 下载或安装新 Plugin。
- 更新扩展。
- 提交发布申请。
- 依赖服务端授权判断的新操作，例如新安装、新接入、新下载、新增启用、配置写入或更新。

## 9.13 验收标准

- 本地页只包含已安装扩展、工具、项目三个子入口。
- Skill 支持安装、更新、启用、停用、卸载。
- MCP Server 支持安装到工具、配置写入、连接检测、更新、卸载和本地事件同步。
- Plugin 支持按安装模式完成安装、下载、更新、卸载或本地状态记录。
- 删除工具前必须校验是否存在已启用 Skill、已接入 MCP Server 或已安装 Plugin。
- 本地写入必须生成执行计划并支持失败反馈和尽量回滚。
- 离线时不得执行依赖服务端授权的新安装、新接入或新下载操作。
