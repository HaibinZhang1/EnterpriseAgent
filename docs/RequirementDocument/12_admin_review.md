# 12. 管理端：审核

## 12.1 页面目标

审核页用于部门管理员和系统管理员处理 Skill、MCP Server、Plugin 的发布申请。

审核页只回答一个问题：

**这个发布申请是否可以通过。**

审核页展示系统规则校验和 AI 系统预审结果，但管理员仍是最终决策人。

## 12.2 审核范围

### 部门管理员

可处理：

- 管理范围内用户提交的部门范围申请。
- 管理范围内扩展的版本更新申请。
- 管理范围内扩展的元信息修改申请。
- 授权范围未跨出自己管理范围的指定部门申请。

不可处理：

- 自己提交的申请。
- 超出管理范围的指定部门申请。
- 授权范围为全员可用的申请。
- 展示扩大到管理范围外可见的申请。
- 因安全风险下架后的重新上架申请。

### 系统管理员

可处理：

- 全部申请。
- 跨部门申请。
- 全员可用申请。
- 展示扩大申请。
- 无部门管理员可处理的申请。
- 异常申请。
- 因安全风险下架后的重新上架申请。

审核详情使用全屏详情页或大抽屉。

内容顺序：

5. 类型化内容预览。

8. 历史记录。

## 12.8 本次申请内容

### 首次发布

展示：

- 新扩展基础信息。
- 目标版本。
- 授权范围。
- 可见选项。
- 分类和标签。
- 风险声明。

### 版本更新

展示：

- 当前版本。
- 目标版本。
- 变更说明。
- 包变化摘要。
- 新增风险点。

### 元信息修改

展示：

- 修改前。
- 修改后。
- 变更原因。

### 授权扩大

展示：

- 当前授权范围。
- 目标授权范围。
- 影响部门。
- 影响用户数量估算。

### 展示扩大

展示：

- 当前可见选项。
- 目标可见选项。
- 是否从“仅授权范围内展示”改为“所有登录用户可展示”。
- 可能外显的标题、描述、README、配置示例或安装说明摘要。
- 是否包含敏感信息预警。

### 重新上架

展示：

- 下架原因。
- 下架操作人。
- 下架时间。
- 重新上架原因。
- 是否涉及安全风险。
- 是否提交新版本或修复说明。

## 12.9 系统校验结果

系统校验展示：

- Extension ID 合法性。
- 版本号合法性。
- 版本递增。
- 包大小。
- 文件数量。
- 必填字段。
- 授权范围合法性。
- 可见选项合法性。
- 用户是否有权提交该授权范围和可见选项。
- 包安全校验结果。
- 类型特定校验。
- 内容外显风险校验。

通过项可折叠。警告项和失败项必须显著展示。


## 12.10 AI 系统预审结果

AI 系统预审用于辅助管理员快速识别风险，不得自动通过或自动拒绝申请。

展示内容：

- 预审状态：未启用、待预审、通过、警告、失败、不可用。
- 预审模型或规则版本。
- 风险摘要。
- 疑似敏感信息摘要，不展示密钥明文。
- 安装、配置写入、数据访问、权限声明是否自洽。
- 建议管理员重点检查项。
- 预审时间。

规则：

- AI 预审只读取脱敏后的元数据、README 摘要、配置摘要、安装清单和风险声明。
- AI 预审结果必须可折叠，避免审核详情变成解释型页面。
- AI 预审不可用时，申请仍进入人工审核，并显示“AI 预审不可用”。
- 管理员的通过、退回、拒绝决定必须独立记录，不得只保存 AI 建议。

## 12.11 类型化内容预览

### Skill

展示：

- 文件清单。
- `SKILL.md` 预览。
- `README.md` 预览。
- 其他可预览文本文件。
- 不可预览文件下载入口。
- 包 Hash。
- 风险文件摘要。

### MCP Server

展示：

- MCP 定义清单。
- 接入方式。
- transport。
- accessType 与 transport 合法组合。
- command 或 endpoint 摘要。
- 配置模板。
- 变量 schema。
- 敏感变量标记。
- 支持工具。
- 连接检测定义。
- 权限声明。
- 数据访问说明。
- 外部或内网端点摘要。
- 系统预审识别出的端点、命令、权限和敏感数据风险。
- local-command 的 command、args、工作目录、环境变量摘要和高风险执行器提示。

### Plugin

展示：

- 插件包文件清单。
- 安装模式。
- 安装清单。
- 安装、更新、卸载步骤摘要。
- 是否支持回滚。
- 目标工具。
- 兼容版本。
- 权限声明。
- 风险说明。
- 下载链接或受控文件摘要。
- 系统预审识别出的安装路径、脚本、二进制和权限风险。

## 12.12 审核权限判定

服务端在管理员打开详情和提交决定时都必须重新判定权限。

| 申请情况 | 部门管理员 | 系统管理员 |
|---|---:|---:|
| 授权范围在自己管理范围内 | 可处理 | 可处理 |
| 指定部门中存在管理范围外部门 | 不可处理 | 可处理 |
| 全员可用 | 不可处理 | 可处理 |
| 展示扩大到管理范围外 | 不可处理 | 可处理 |
| 申请人是当前管理员本人 | 不可处理 | 若申请人不是本人则可处理 |
| 安全风险下架后重新上架 | 不可处理 | 可处理 |
| 无部门管理员可处理 | 不可处理 | 可处理 |

如果前端显示可处理但服务端判定不可处理，以服务端结果为准，并返回明确错误。

## 12.13 审核操作

审核详情提供操作：

- 通过。
- 退回修改。
- 拒绝。

### 通过

要求：

- 可填写审核意见。
- 通过后扩展或变更生效。
- 写入审核记录和审计日志。
- 通知发布者。

### 退回修改

要求：

- 必须填写原因。
- 可选择原因模板。
- 发布者可修改后重新提交。
- 写入审核记录和审计日志。
- 通知发布者。

### 拒绝

要求：

- 必须填写原因。
- 可选择原因模板。
- 本次申请结束。
- 发布者可复制为新申请。
- 写入审核记录和审计日志。
- 通知发布者。

## 12.14 并发处理规则

- 管理员打开审核详情时，服务端返回最新状态。
- 管理员提交决定时，服务端再次校验状态。
- 如果申请已被他人处理，当前操作失败并提示最新结果。
- 不需要复杂强制锁单，但必须避免重复通过或重复拒绝。

## 12.15 审核意见模板

系统提供基础模板：

- 缺少必要说明。
- 包结构不符合规范。
- 版本号不符合要求。
- 授权范围过大。
- 风险声明不足。
- 安装说明不完整。
- MCP 配置模板不完整。
- Plugin 卸载或回滚说明不完整。
- 可见选项与内容敏感性不匹配。
- MCP 配置不清晰。
- Plugin 目标工具不明确。
- 包安全校验存在风险。
- 重新上架原因不足。

审核历史展示：

## 12.17 验收标准

- 部门管理员只能看到可处理范围内申请。
- 系统管理员可看到全局申请。
- Skill、MCP Server、Plugin 审核详情内容不同。
- 审核通过、退回、拒绝都写入审计日志。
- 禁止自审。
- 已处理申请不能被重复处理。
- 服务端必须在提交审核决定时重新校验管理员权限。


补充验收要求：

- 审核详情必须展示可见选项。
- 展示扩大必须进入审核。
- AI 系统预审结果必须展示，但不得替代管理员决策。
- MCP Server 审核必须能看到 accessType 与 transport 合法组合、配置模板、变量 schema、连接检测定义和 local-command 风险摘要。
- Plugin 审核必须能看到安装模式、安装清单、卸载步骤和回滚能力。
