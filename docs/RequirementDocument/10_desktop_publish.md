# 10. 桌面端：发布入口

## 10.1 页面目标

发布入口用于普通用户在桌面客户端提交 Skill、MCP Server、Plugin 的发布申请。

发布入口不作为一级导航。用户从社区首页或社区结果页点击 `+ 发布` 按钮进入发布向导。

## 10.2 入口规则

- 社区首页右上角展示 `+ 发布`。
- 社区结果页工具栏展示 `+ 发布`。
- 扩展详情中作者可看到“发布新版本”“修改授权范围”或“修改可见选项”入口。
- 未登录用户点击发布时先登录。
- 发布向导以模态或全屏覆盖层展示。

## 10.3 发布向导步骤

发布向导分为四步，减少用户理解成本：

```text
1. 选择类型与提供内容
2. 自动识别并补全信息
3. 授权范围、可见选项与风险声明
4. 确认提交
```

原则：

- 系统能自动读取的字段不得要求用户重复填写。
- 分类、标签、适用工具、适用系统、README 摘要、Extension ID、版本号应优先从包或配置中提取。
- 高级字段默认折叠，用户需要时再展开。
- 页面只展示当前步骤必须完成的任务，不放大段解释文字。

## 10.4 第一步：选择类型与提供内容

用户必须选择：

- Skill。
- MCP Server。
- Plugin。

选择类型后，内容入口按类型变化。

### Skill

支持：

- 文件夹上传。
- zip 上传。

要求：

- 必须包含 `SKILL.md`。
- 可自动读取 `SKILL.md` frontmatter。
- 可自动读取 README。
- 不允许把散文件当作 Skill 包提交。

### MCP Server

支持：

- 粘贴 MCP 定义清单。
- 上传 MCP 定义文件。
- 填写连接信息和配置模板。

必须提供：

- 接入方式：remote-http、remote-sse 或 local-command。
- transport 类型：stdio、streamable-http 或 sse；兼容输入 http 时服务端规范化为 streamable-http。
- command 或 endpoint。
- 配置模板。
- 变量 schema。
- 支持工具。
- 连接检测方式。
- 权限声明。
- 数据访问说明。

### Plugin

支持：

- 上传插件包。
- 上传或填写安装清单。
- 填写手动安装说明。
- 填写企业内网受控下载信息。

必须提供：

- 目标工具。
- 安装模式：managed-package、config-plugin 或 manual-download。
- 兼容版本。
- 安装、更新和卸载说明。
- 权限声明。
- 风险说明。

## 10.5 第二步：自动识别并补全信息

系统自动提取后展示可编辑表单。

通用字段：

- Extension ID。
- 显示名称。
- 描述。
- 版本号。
- 变更说明。
- 分类。
- 标签。
- 适用工具。
- 适用系统。

MCP Server 自动识别字段：

- serverName。
- transport。
- 接入方式。
- command 或 endpoint 摘要。
- variables schema。
- supportedTools。
- permissions。
- dataAccess。

Plugin 自动识别字段：

- 目标工具。
- 安装模式。
- 兼容版本。
- 文件清单。
- 安装清单。
- 权限声明。

### 分类

当前固定分类：

```text
开发 / 测试 / 文档 / 设计 / 运维 / 安全 / 集成 / 自动化 / 数据 / 知识 / 其他
```

每个扩展必须选择 1 个分类。

### 标签

当前固定标签：

```text
代码 / 审查 / 改进 / 提示 / 规范 / 清单 / 文档 / 写作 / 测试 / 验收 / 前端 / 可访问 / 设计 / 运维 / 值班 / 事故 / 安全 / 权限 / 集成 / 适配 / 自动化 / 发布 / 数据 / 分析 / 入门 / 培训
```

每个扩展选择 1 到 5 个标签。

### 字段补全规则

- 自动识别成功的字段展示为已填充，用户可修改。
- 自动识别失败的必填字段必须高亮提示。
- 自动识别内容与用户手工修改内容都必须进入提交摘要。
- 识别失败不得阻塞用户手工填写。

## 10.6 第三步：授权范围、可见选项与风险声明

### 授权范围

用户选择谁可以发起新安装、新下载、复制配置、新增启用、配置写入或更新等主操作：

- 本部门可用。
- 本部门及下级部门可用。
- 指定部门可用。
- 全员可用。

说明：授权范围只控制主操作是否可用，不直接等同于社区展示。

### 可见选项

发布向导必须提供一个勾选项：

```text
仅授权范围内展示
说明：勾选表示无权限用户不可见、不进入榜单统计。
```

规则：

- 默认不勾选。
- 未勾选时，已发布扩展对所有登录用户可浏览；无权限用户只能查看详情和 Star，不能安装、下载、复制配置、配置写入或更新。
- 勾选时，无权限用户在社区搜索、榜单和直接链接中均不可见。
- 勾选时，该扩展不进入热榜、Star 榜、下载量榜或使用量榜。
- 从勾选改为不勾选属于展示扩大，必须进入审核。
- 从不勾选改为勾选属于展示收缩，可直接生效但必须审计。

### 内容外显提醒

在可见选项附近展示简短提醒：

```text
未勾选时，扩展名称、描述、README、配置示例和安装说明可能被所有登录用户看到，请勿填写密钥、客户信息或部门敏感内容。
```

### 风险声明

按类型展示必要字段：

- Skill：是否包含脚本、是否包含可执行文件、是否写入本地工具或项目。
- MCP Server：访问的数据类型、是否访问敏感资源、接入方式、环境变量要求、连接端点或命令风险。
- Plugin：目标工具、安装位置、权限要求、是否需要受控下载、是否可回滚。

### 审核归属提示

发布向导需在确认提交前展示审核归属：

- 部门范围：由对应部门管理员处理。
- 跨部门、全员范围或展示扩大：由系统管理员或具备范围权限的管理员处理。

提示必须简短，不得展开复杂流程说明。

## 10.7 第四步：确认提交

确认页展示：

- 扩展类型。
- Extension ID。
- 显示名称。
- 版本。
- 授权范围。
- 可见选项。
- 包大小、配置摘要或安装清单摘要。
- 客户端前置校验结果。
- 风险摘要。
- 预计审核归属。

用户点击 `提交发布申请` 后：

- 客户端提交到服务端。
- 服务端创建发布申请，初始内部状态为 `validating`，保存提交 payload 快照。
- 服务端执行最终校验、规则预审和可选 AI 系统预审，并把结果挂载到该发布申请。
- 预审完成后进入待审核状态并显示申请状态。
- 用户可跳转查看“我的提交”。

## 10.8 我的提交

发布向导完成后，用户可从社区账号相关入口或发布成功结果进入“我的提交”。

展示字段：

- 扩展类型。
- 名称。
- 版本。
- 申请类型。
- 状态。
- 授权范围。
- 可见选项。
- 提交时间。
- 最近处理时间。
- 审核意见摘要。

可执行操作：

- 查看详情。
- 撤回待审核申请。
- 修改后重新提交。
- 复制为新申请。
- 发布新版本。
- 修改授权范围。
- 修改可见选项。
- 下架。
- 申请重新上架。
- 申请归档。

## 10.9 校验规则

提交前客户端必须校验：

- Extension ID 格式。
- 显示名称已填写。
- 描述已填写。
- 版本号符合 SemVer。
- 变更说明已填写。
- 分类已选择。
- 标签数量为 1 到 5。
- 授权范围已选择。
- 可见选项已明确。
- Skill 包包含 `SKILL.md`。
- Skill 包大小不超过 5MB。
- Skill 包文件数不超过 100。
- MCP Server 已声明接入方式、合法 transport 组合、连接方式、配置模板、变量 schema、权限声明和数据访问说明。
- Plugin 已声明目标工具、安装模式、兼容版本、安装或下载方式、权限声明和风险说明。

服务端必须执行同等或更严格校验。

## 10.10 发布申请类型

系统支持：

| 类型 | 说明 | 是否需要审核 |
|---|---|---:|
| 首次发布 | 新扩展第一次提交 | 是 |
| 版本更新 | 已发布扩展提交新版本 | 是 |
| 元信息修改 | 修改名称、描述、分类、标签等信息 | 是 |
| 授权扩大 | 扩大可发起主操作的人群 | 是 |
| 授权收缩 | 缩小可发起主操作的人群 | 否，但必须审计 |
| 展示扩大 | 从“仅授权范围内展示”改为“所有登录用户可展示” | 是 |
| 展示收缩 | 从“所有登录用户可展示”改为“仅授权范围内展示” | 否，但必须审计 |
| 重新上架 | 已下架扩展恢复发布状态 | 视下架原因 |
| 归档 | 扩展进入终态 | 否，但必须审计 |

授权收缩和展示收缩可直接生效，但必须记录审计日志。授权扩大和展示扩大必须审核。

## 10.11 退回修改与重新提交

退回修改后，系统必须保留每次提交的 revision。

规则：

- 发布者在原申请中修改后重新提交，不覆盖历史 revision。
- 每个 revision 保存提交 payload 快照、包或配置快照、系统规则校验结果、AI 系统预审结果和提交时间。
- 审核详情必须能查看当前待审 revision，并能查看历史 revision 摘要和管理员退回意见。
- 重新提交后必须重新执行规则校验和 AI 系统预审。
- 最终通过时必须记录生效的是哪一个 revision。

## 10.12 成功反馈

提交成功后显示：

- `发布申请已提交`。
- 当前状态：`待审核`。
- 审核归属：部门管理员或系统管理员。
- 系统预审状态：等待、通过、警告或不可用。
- 操作：查看我的提交、返回社区。

## 10.13 错误反馈

常见错误：

- Extension ID 已存在。
- 包缺少 `SKILL.md`。
- 包过大。
- 文件数量超限。
- 版本号未递增。
- 用户无权选择该授权范围。
- 用户无权执行展示扩大。
- 包含禁止路径或不安全文件。
- MCP endpoint 格式、协议或必填字段不合法。
- MCP 配置模板无法渲染。
- MCP 变量 schema 不合法。
- Plugin 安装清单缺少卸载步骤。
- Plugin 包 Hash 计算失败。
- 服务端不可用。
- 会话失效。

错误必须明确给出修正动作。

## 10.14 验收标准

- 发布入口是按钮，不是一级页签。
- 发布第一步必须选择 Skill、MCP Server 或 Plugin。
- 不同类型表单字段不同。
- 发布向导不超过四步。
- 发布向导自动提取可识别字段。
- 发布向导包含授权范围和可见选项。
- 可见选项文案必须包含“勾选表示无权限用户不可见、不进入榜单统计”。
- 发布成功后状态显示为待审核。
- 部门范围申请提示部门管理员处理。
- MCP Server 发布必须能提供完整配置模板和连接检测定义。
- Plugin 发布必须能提供安装模式、安装清单或手动安装说明。
