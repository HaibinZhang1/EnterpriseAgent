# 16. 核心业务流程

## 16.1 登录流程

1. 用户打开桌面客户端或 Web 管理端。
2. 输入手机号和密码。
3. 服务端校验账号状态、密码、角色、所属部门状态和登录失败锁定状态。
4. 登录成功后返回会话、用户摘要和权限摘要。
5. 桌面客户端进入 Agent 首页；Agent 具体能力由独立设计文档承接。
6. Web 管理端根据角色展示可访问菜单。
7. 登录成功或失败都写审计日志。
8. 桌面客户端登录成功后上报设备心跳和客户端版本。

## 16.2 社区发现流程

1. 用户点击桌面客户端 `社区`。
2. 系统进入社区首页。
3. 用户查看 Skill、MCP Server、Plugin 三个区域的榜单。
4. 榜单只统计未勾选“仅授权范围内展示”的扩展。
5. 用户提交搜索或点击类型入口。
6. 服务端按可见选项过滤结果。
7. 系统进入搜索结果页。
8. 结果按类型分组或单类型展示。
9. 用户点击卡片打开详情。
10. 用户根据类型执行安装、接入、复制配置、下载、查看安装说明或 Star。
11. 若用户不在授权范围内但扩展可见，详情内容仍可查看，但安装、接入、复制配置、下载或自动写入操作不可用。
12. 若扩展勾选“仅授权范围内展示”且用户不在授权范围内，搜索和详情均不可见。

## 16.3 Skill 安装流程

1. 用户在 Skill 详情点击安装。
2. 客户端请求服务端下载凭证。
3. 服务端校验用户身份、扩展状态、版本状态、授权范围和可见选项。
4. 服务端发放短期下载凭证，并记录下载事件。
5. 客户端下载包到临时目录。
6. 客户端校验 SHA-256。
7. 校验通过后写入 Central Store 的新版本目录。
8. 客户端切换当前版本指针。
9. 客户端刷新本地状态。
10. 生成本地安装事件。
11. 联网时同步本地事件。

## 16.4 Skill 启用流程

1. 用户在 Skill 详情或本地页点击启用范围。
2. 客户端展示可用工具和项目。
3. 用户选择目标。
4. 客户端校验目标路径。
5. 客户端生成启用执行计划。
6. 客户端执行格式转换。
7. 客户端优先 symlink 到目标路径。
8. symlink 失败时降级 copy。
9. 客户端记录实际模式和降级原因。
10. 若部分目标失败，客户端按执行计划回滚可回滚目标。
11. 客户端刷新本地状态。
12. 联网时同步本地事件。

## 16.5 Skill 更新流程

1. 客户端发现服务端有新版本。
2. 本地页或详情页展示有更新。
3. 用户点击更新。
4. 服务端校验用户仍具备更新权限。
5. 客户端获取下载凭证。
6. 客户端下载新包到临时目录并校验 Hash。
7. Hash 通过后写入 Central Store 新版本目录。
8. 客户端备份当前版本指针和已启用目标元数据。
9. 客户端切换当前版本指针。
10. 客户端重新分发到已启用目标。
11. 若本地内容已变更，更新前提示覆盖风险。
12. 若分发失败，客户端回滚可回滚目标，并展示部分成功或失败原因。
13. 客户端记录更新事件。

## 16.6 Skill 卸载流程

1. 用户点击卸载。
2. 客户端展示当前启用位置。
3. 用户确认。
4. 客户端生成卸载执行计划。
5. 客户端移除目标位置。
6. 客户端删除 Central Store 副本。
7. 若部分目标失败，显示部分成功。
8. 客户端记录卸载事件。

## 16.7 MCP Server 安装接入流程

1. 用户在社区搜索或榜单中打开 MCP Server 详情。
2. 服务端返回详情、授权标记和可见标记。
3. 用户点击 `安装到工具`。
4. 客户端请求 MCP 定义清单和配置模板。
5. 服务端校验用户身份、扩展状态、版本状态、授权范围和可见选项。
6. 客户端展示可写入 MCP 配置的目标工具。
7. 用户选择目标工具。
8. 客户端检测目标工具配置路径和写入能力。
9. 用户填写变量；敏感变量只保存在本机安全存储。
10. 客户端渲染配置摘要，展示写入影响。
11. 客户端生成配置写入执行计划。
12. 用户确认。
13. 客户端备份原配置。
14. 客户端写入托管 MCP 配置项。
15. 客户端执行连接检测。
16. 写入或检测失败时，客户端按计划回滚可回滚目标并展示原因。
17. 客户端记录 MCP 接入事件、连接检测事件和异常事件。
18. 联网时同步本地事件。

## 16.8 MCP Server 更新与卸载流程

### 更新

1. 客户端发现 MCP Server 有新版本。
2. 本地页展示有更新。
3. 用户点击更新配置。
4. 服务端校验用户仍具备更新权限。
5. 客户端获取新配置模板和变更摘要。
6. 客户端保留用户本机变量值，提示新增或删除变量。
7. 客户端生成配置更新执行计划。
8. 客户端备份旧配置并写入新配置。
9. 客户端执行连接检测。
10. 失败时按计划回滚。
11. 客户端记录更新事件。

### 卸载

1. 用户点击卸载配置。
2. 客户端展示已写入的目标工具和托管配置项。
3. 用户确认。
4. 客户端生成卸载执行计划。
5. 客户端备份当前配置。
6. 客户端仅移除 Enterprise Agent Hub 托管的 MCP 配置项。
7. 客户端不删除发布者部署的远端服务，也不删除用户手动创建的配置。
8. 客户端记录卸载事件。

## 16.9 Plugin 使用流程

### managed-package

1. 用户打开 Plugin 详情。
2. 用户点击安装。
3. 服务端校验身份、扩展状态、版本状态、授权范围和可见选项。
4. 服务端发放下载凭证。
5. 客户端下载插件包并校验 Hash。
6. 客户端检测目标工具和兼容版本。
7. 客户端读取安装清单并生成执行计划。
8. 用户确认。
9. 客户端安装到受控位置或目标工具位置。
10. 失败时按计划回滚。
11. 客户端记录安装事件。

### config-plugin

1. 用户打开 Plugin 详情。
2. 用户点击安装。
3. 服务端校验身份、扩展状态、版本状态、授权范围和可见选项。
4. 客户端获取配置清单。
5. 客户端检测目标工具配置能力。
6. 客户端根据配置清单生成写入执行计划。
7. 用户确认。
8. 客户端备份原配置并写入托管配置项。
9. 失败时按计划回滚。
10. 客户端记录安装或启用事件。

### manual-download

1. 用户打开 Plugin 详情。
2. 用户查看安装说明。
3. 若用户在授权范围内，可请求下载凭证并下载 Hub 托管文件；若使用企业内部下载地址，客户端记录下载来源并校验登记的 SHA-256。
4. 用户按说明手动安装。
5. 用户可在本地页标记已安装或已卸载。
6. 客户端记录受控下载和本地标记事件。

## 16.10 Plugin 更新与卸载流程

### 更新

1. 客户端发现 Plugin 有新版本。
2. 本地页展示有更新。
3. 用户点击更新。
4. 服务端校验用户仍具备更新权限。
5. 客户端按安装模式获取新包或新配置清单。
6. 客户端生成更新执行计划。
7. 用户确认后执行。
8. 失败时尽量回滚并展示原因。
9. 客户端记录更新事件。

### 卸载

1. 用户点击卸载。
2. 客户端展示已安装位置、托管配置或手动安装说明。
3. managed-package 和 config-plugin 按安装清单卸载托管内容。
4. manual-download 只清除本地状态记录，并提示用户按说明手动卸载。
5. 客户端记录卸载事件。

## 16.11 发布申请流程

1. 用户在社区点击 `+ 发布`。
2. 用户选择类型并提供内容：Skill、MCP Server 或 Plugin。
3. 系统自动识别包、配置或安装说明中的可提取字段。
4. 用户补全基础信息。
5. 用户选择授权范围和可见选项，并填写风险声明。
6. 客户端执行前置校验。
7. 用户确认提交。
8. 服务端创建发布申请，内部状态为 `validating`，保存 submission 和第一个 revision 快照。
9. 服务端执行最终校验和规则预审，并把结果写入该 revision 的 system_precheck。
10. 若启用 AI 系统预审，服务端调用企业内网 AI 服务生成预审结果，并写入该 revision 的 system_precheck。
11. 系统计算审核归属。
12. 预审完成后申请进入 `pending_review`。
13. 发布者看到状态：待审核。
14. 管理员收到待审核通知。
15. 提交动作、规则预审和 AI 预审结果写审计日志。

## 16.12 审核流程

1. 管理员登录 Web 管理端。
2. 进入审核页。
3. 查看待处理申请。
4. 打开审核详情。
5. 查看申请内容、系统校验结果、AI 系统预审结果、文件或配置预览、授权影响、可见选项、风险说明。
6. 管理员选择通过、退回修改或拒绝。
7. 服务端校验管理员权限和申请最新状态。
8. 服务端生效审核决定。
9. 服务端写审核历史和审计日志。
10. 发布者收到通知。

## 16.13 退回修改流程

1. 管理员在审核详情点击退回修改。
2. 管理员填写原因。
3. 申请状态变为需修改。
4. 发布者收到通知。
5. 发布者打开我的提交。
6. 发布者修改内容后在原申请中重新提交。
7. 服务端创建新的 submission revision，保留原 revision、退回意见和校验结果。
8. 新 revision 重新进入系统校验、AI 预审和管理员审核流程。
9. 最终通过时记录生效的 revision。

## 16.14 拒绝流程

1. 管理员在审核详情点击拒绝。
2. 管理员填写原因。
3. 申请状态变为已拒绝。
4. 发布者收到通知。
5. 发布者可查看原因或复制为新申请。

## 16.15 授权收缩流程

1. 发布者或管理员提交授权收缩。
2. 服务端判断为授权收缩。
3. 服务端直接生效。
4. 服务端写审计日志。
5. 超出新授权范围的用户不可新增安装、接入、复制配置、新增启用、配置写入、下载或更新。
6. 存量已安装 Skill、已接入 MCP Server、已安装 Plugin 可继续保留当前本地状态，除非扩展被安全下架并被安全策略要求处理。
7. 存量用户不可更新到新版本，除非重新进入授权范围。
8. 授权收缩不禁止存量用户停用、卸载、移除托管配置、清除本地敏感变量或清除 manual-download 本地记录。
9. 客户端下一次同步后刷新授权收缩提示；服务端不承诺实时阻断离线或目标工具直接加载的本地内容。

## 16.16 可见选项变更流程

### 展示收缩

1. 作者或管理员将扩展改为“仅授权范围内展示”。
2. 服务端判断为展示收缩。
3. 变更直接生效。
4. 扩展从社区榜单统计中移除。
5. 无权限用户不可再搜索或打开详情。
6. 服务端写审计日志。

### 展示扩大

1. 作者或管理员将扩展从“仅授权范围内展示”改为“所有登录用户可展示”。
2. 服务端判断为展示扩大。
3. 系统创建发布申请并进入系统校验、AI 预审和管理员审核流程。
4. 审核通过后，扩展恢复对所有登录用户展示并可进入榜单统计。
5. 服务端写审计日志。

## 16.17 下架流程

### 普通下架

1. 发布者、部门管理员或系统管理员点击下架。
2. 系统展示影响说明。
3. 操作人选择普通下架原因并填写说明。
4. 操作人确认。
5. 服务端校验权限。
6. 扩展状态变为已下架。
7. 社区不再展示。
8. 不再允许新安装、接入、复制配置、下载或更新。
9. 服务端写审计日志。
10. 已安装用户不自动删除本地副本或配置。

### 安全下架

1. 系统管理员点击安全下架。
2. 系统展示安全影响说明。
3. 系统管理员填写安全原因和处置建议。
4. 服务端校验权限。
5. 扩展状态变为已安全下架。
6. 社区不再展示。
7. 不再允许新安装、接入、复制配置、下载或更新。
8. 已安装、已接入或已下载用户收到强提醒。
9. 本地页标记安全风险。
10. 客户端按系统安全策略执行存量处置：仅提醒、阻止新增启用/接入或强制停用托管项。
11. 服务端写审计日志。

## 16.18 重新上架流程

1. 作者或管理员发起重新上架。
2. 服务端读取原下架原因。
3. 若为维护、误操作、过期等非安全原因，且版本、包、元信息、授权范围、可见选项未变化，具备权限的管理员可直接重新上架。
4. 若涉及新版本、元信息修改、授权扩大或展示扩大，系统创建发布申请并进入审核。
5. 若原下架原因为安全风险，必须由系统管理员处理。
6. 安全风险重新上架必须提供修复说明；包内容、MCP 配置、Plugin 安装清单或权限声明变化时必须提交审核。
7. 服务端写审计日志。
8. 重新上架后扩展恢复社区展示；安装、下载、复制配置、配置写入、新增启用和更新仍按授权范围与可见选项控制，停用、卸载和本地清理不受当前授权范围限制。

## 16.19 归档流程

1. 作者提交归档申请，或管理员在扩展管理中点击归档。
2. 系统展示终态影响说明。
3. 操作人填写原因并二次确认。
4. 服务端校验权限。
5. 扩展状态变为已归档。
6. 服务端写审计日志。
7. 已归档扩展不得重新上架、不得发布新版本、不得变更授权范围或可见选项。

## 16.20 扩展维护人或归属部门转移流程

1. 部门管理员或系统管理员在扩展管理详情中点击转移。
2. 系统展示当前维护人、当前归属部门、目标维护人、目标归属部门和影响说明。
3. 操作人填写转移原因。
4. 服务端校验操作人权限、目标用户状态、目标部门状态和管理范围。
5. 服务端更新扩展维护人或归属部门。
6. 历史作者、历史所属部门、历史审核和审计快照不变。
7. 后续治理权限、通知接收人和管理端归属过滤按新的维护人或归属部门计算。
8. 服务端写审计日志并通知新维护人。

## 16.21 客户端设备登记流程

1. 桌面客户端首次启动时生成 deviceID。
2. 用户登录成功后，客户端向服务端登记设备。
3. 服务端记录用户、部门、设备 ID、客户端版本、OS、架构和首次登记时间。
4. 客户端定期上报心跳和版本摘要。
5. 服务端更新最后在线时间和版本分布。
6. 更新失败、本地事件异常等关键事件可关联到设备 ID。

## 16.22 客户端更新流程

1. 系统管理员在 Web 管理端创建客户端更新版本。
2. 上传或登记 Windows x64 安装包。
3. 服务端校验版本号、Hash、签名状态和包大小。
4. 系统管理员发布更新。
5. 桌面客户端登录或启动时检查更新，并携带 deviceID 和当前版本。
6. 若有更新，通知用户。
7. 用户申请下载凭证。
8. 客户端下载并校验安装包 Hash。
9. 客户端校验安装包签名。
10. Hash 和签名均通过后，用户确认并启动安装程序。
11. 更新事件上报服务端并关联设备 ID。
12. 更新后首次启动时上报新版本。

## 16.23 离线同步流程

1. 客户端离线期间允许查看和清理本地内容；若本机已有有效授权缓存且未收到授权收缩或安全下架标记，可启用已安装 Skill；始终允许停用已安装 Skill、停用已接入 MCP Server、禁用或卸载本机托管 Plugin。
2. 客户端记录离线本地事件。
3. 网络恢复后客户端同步事件。
4. 服务端返回 accepted、rejected 或 ignored。
5. 服务端返回权限、版本、下架、安全下架和可见选项变化提示。
6. 客户端按服务端最新状态刷新本地提示。
7. 本地事件同步必须具备幂等键，避免重复上报导致统计重复。

## 16.24 审计查询流程

1. 管理员进入审计日志。
2. 输入用户、对象、动作、时间、设备 ID 或 requestID。
