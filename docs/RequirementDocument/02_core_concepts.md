# 2. 核心概念

## 2.1 Extension

Extension 是企业内部 Agent 扩展中心的统一治理对象。Skill、MCP Server、Plugin 都属于 Extension。

Extension 具备以下共同能力：

- 唯一 ID。
- 显示名称。
- 类型。
- 描述。
- 作者。
- 所属部门。
- 当前版本。
- 授权范围。
- 可见选项。
- 生命周期状态。
- Star 数。
- 下载量或使用量。
- 审核记录。
- 审计记录。

浏览原则：

- 已发布且未下架、未归档的 Extension 默认对登录用户展示卡片和详情。
- 发布者可在发布申请中勾选“仅授权范围内展示”。勾选后，无权限用户不可见，该扩展不进入社区首页榜单统计。
- 授权范围决定用户是否可以发起服务端授权的新安装、新下载、复制配置、新增启用、配置写入和更新等主操作。已安装或已接入内容的本地停用、卸载和托管配置清理不受当前授权范围限制。
- 未勾选“仅授权范围内展示”的扩展，即使用户不在授权范围内，也可查看详情，但主操作区必须显示不可操作原因，例如“你当前不在授权范围内，无法安装”。
- 已下架、已归档、审核中、已拒绝的 Extension 不在普通社区中展示，只在管理端、作者提交记录或审计链路中可追踪。

## 2.2 Skill

Skill 是可被 Agent 或 AI 工具识别、加载和使用的文件型能力包。

Skill 的基本特征：

- 以目录或 zip 包形式提交。
- 根目录必须包含 `SKILL.md`。
- 可包含 README、示例、脚本、资源文件、图标等内容。
- 可安装到本机 Central Store。
- 可启用到工具或项目。
- 支持完整安装、更新、启用、停用、卸载闭环。

## 2.3 MCP Server

MCP Server 是通过 Model Context Protocol 向 Agent 或 AI 工具提供上下文、工具调用或数据访问能力的接入型扩展。

MCP Server 的基本特征：

- 以 MCP 定义清单登记，不要求 Enterprise Agent Hub 托管服务进程。
- 服务进程可以是企业内网或企业认可的 Streamable HTTP Endpoint、兼容旧版 HTTP+SSE 的端点，也可以是用户本机已有命令提供的 stdio 进程。
- 必须描述接入方式、transport、启动或连接方式、配置模板、变量 schema、权限声明、数据访问说明、适用工具、风险说明。
- 可发布、审核、展示、Star、授权接入、配置写入、连接检测、更新、卸载、统计和审计。
- 客户端通过 Tool Adapter 将 MCP 配置写入受支持工具；无法写入时必须展示明确原因，并允许授权用户复制配置。
- 服务端不得保存用户填写的真实密钥；本机敏感变量由客户端使用 Windows 安全能力保存。

## 2.4 Plugin

Plugin 是面向指定 AI 工具、内部工具或 Agent 环境的插件型扩展。

Plugin 的基本特征：

- 必须声明目标工具、适用环境、安装模式、权限声明、风险说明。
- 可以提供插件包、配置清单、受控下载文件或手动安装说明。
- 可发布、审核、展示、Star、受控安装或受控下载、更新、卸载、统计和审计。
- 受控安装必须通过插件安装清单和 Tool Adapter 执行，不允许 UI 组件直接写入工具目录。
- 手动安装模式必须明确标注，提供受控下载、本地“已安装”记录和用户可理解的卸载说明，不得伪装成自动安装。

## 2.5 Extension ID

Extension ID 是扩展的唯一标识。

规则：

- 由小写字母、数字、连字符组成。
- 不得以连字符开头或结尾。
- 不得连续出现多个连字符。
- 全局唯一，不允许不同类型复用同一个 Extension ID。
- Extension ID 一经发布不得修改；如需更名，只能修改显示名称。

示例：

```text
code-review-skill
finance-mcp-server
design-export-plugin
```

## 2.6 显示名称

显示名称用于界面展示，可使用中文、英文、数字、空格和常见符号。

要求：

- 允许重名。
- 不用于权限判断。
- 不用于本地目录名。
- 列表、卡片、详情、通知均显示显示名称。

## 2.7 作者与所属部门

作者是提交发布申请的用户。所属部门默认是作者提交时所在部门。扩展维护人是当前负责后续版本、元信息和治理申请的用户；归属部门是扩展当前治理归属的部门。

规则：

- 扩展发布后，历史作者和所属部门以发布时快照展示。
- 用户后续调部门不改变历史扩展归属快照。
- 管理员审核记录保留操作时的管理员快照。
- 管理员可转移扩展维护人或归属部门。部门管理员只能在管理范围内转移；系统管理员可全局转移。
- 转移维护人或归属部门不修改历史发布、审核和审计快照，只影响后续治理权限、通知接收人和管理端归属过滤。

## 2.8 安装与接入

安装与接入是指客户端在授权校验通过后，将扩展制品、配置或安装清单应用到本机受控位置或受支持工具。

类型差异：

- Skill：从服务端获取下载凭证，下载包并写入 Central Store。
- MCP Server：获取 MCP 定义和配置模板，选择目标工具，填写必要变量，写入目标工具 MCP 配置。
- Plugin：按安装模式获取插件包、配置清单或受控下载文件，执行受控安装或记录手动安装状态。

通用规则：

- 安装、接入、新下载、新增配置写入、新增启用和更新必须通过服务端校验身份、用户状态、部门状态、扩展状态、版本状态、授权范围和可见选项。
- 文件下载必须通过服务端下载凭证。
- 客户端下载后必须校验 Hash。
- 本地写入必须生成执行计划，支持失败反馈和尽量回滚。
- 安装或接入不等于启用；是否启用由类型和目标工具能力决定。

## 2.9 启用、配置写入与停用

启用是指客户端将已安装或已接入的扩展配置到指定工具或项目，使其在目标环境中生效。

类型差异：

- Skill：分发到工具或项目目录，优先 symlink，失败时降级 copy。
- MCP Server：写入或移除目标工具的 MCP 配置项，并保留备份和回滚信息。
- Plugin：按插件安装清单启用、禁用或移除目标工具中的插件配置或文件。

通用规则：

- 每次启用、配置写入、停用、更新和卸载必须按事务计划执行。
- 停用、卸载和清理本地托管配置属于本机安全清理动作，不因用户后续失去授权、扩展下架或归属部门停用而被禁止。
- 失败时尽量回滚；无法回滚时标记部分成功并列出影响位置。
- 降级、失败、回滚、部分成功都必须记录本地事件并可同步到服务端。

## 2.10 卸载

卸载指删除本机受控存储中的扩展副本、接入记录或托管配置，并清理所有已托管启用位置。

卸载规则：

- 卸载、停用和清理本地托管配置不受当前授权范围限制；即使用户已不在授权范围内、扩展已下架或扩展归属部门已停用，用户仍可清理本机托管内容。
- 卸载前必须展示当前启用、配置写入或安装位置。
- 用户确认后执行。
- 若部分目标清理失败，显示部分成功，并列出需要处理的位置。
- Skill 卸载后删除 Central Store 副本。
- MCP Server 卸载后删除目标工具中的托管 MCP 配置项，不删除发布者自行部署的远端服务。
- Plugin 卸载后按安装清单删除托管文件或配置；手动安装模式只清除本地记录并提示用户按说明处理。
- 卸载后，如果服务端权限已收缩、扩展已下架、扩展已归档或归属部门已停用，用户不得重新安装、重新接入、重新下载或重新启用该扩展。

## 2.11 授权范围

授权范围决定用户是否可以发起服务端授权的新安装、新下载、复制配置、新增启用、配置写入和更新等主操作。

授权范围不是社区展示范围。社区展示由“可见选项”决定。

范围类型：

| 授权范围 | 说明 | 审核归属 |
|---|---|---|
| 本部门可用 | 仅作者发布时所属部门用户可以发起主操作 | 部门管理员 |
| 本部门及下级部门可用 | 作者部门及下级部门用户可以发起主操作 | 部门管理员，前提是不超出其管理范围 |
| 指定部门可用 | 指定一个或多个部门可以发起主操作 | 未跨出管理范围时由部门管理员审核，跨出后由系统管理员审核 |
| 全员可用 | 企业内所有登录用户可以发起主操作 | 系统管理员 |

界面展示规则：

- 发布向导必须展示“授权范围”字段。
- 详情页主操作区根据当前用户是否在授权范围内展示可操作或不可操作状态。
- 服务端必须在每次安装、下载、复制配置、新增启用、配置写入和更新时重新校验授权范围。
- 服务端不能承诺实时阻断离线客户端或目标工具直接加载的本地内容；存量本地运行状态通过下架、安全下架策略、客户端同步提示和本地清理能力治理。

## 2.12 可见选项

可见选项用于控制未授权用户是否能在社区中看到扩展。它不是角色，不替代授权范围，也不引入复杂的可见范围体系。

发布向导提供一个勾选项：

```text
仅授权范围内展示
说明：勾选表示无权限用户不可见、不进入榜单统计。
```

规则：

- 默认不勾选。默认情况下，已发布且未下架、未归档的扩展可被所有登录用户浏览。
- 勾选后，只有授权范围内用户、作者、具备管理权限的管理员可在相应入口看到该扩展。
- 勾选后，该扩展不进入社区首页热榜、Star 榜、下载量榜或使用量榜统计。
- 勾选后，未授权用户通过搜索、榜单、直接链接访问时，服务端返回不可见结果或简短不可见提示，不暴露扩展详情。
- 从“仅授权范围内展示”改为“所有登录用户可展示”属于展示扩大，必须进入审核。
- 从“所有登录用户可展示”改为“仅授权范围内展示”属于展示收缩，可直接生效，但必须写审计日志。

## 2.13 Star

Star 表示收藏、推荐和搜索权重提升。

规则：

- 同一用户对同一扩展只能 Star 一次。
- 取消 Star 后计数减少。
- 再次 Star 后重新计数。
- Star 统计为实时有效数量。
- 勾选“仅授权范围内展示”的扩展不进入榜单统计，但授权范围内用户仍可 Star。

## 2.14 下载量

下载量指用户以安装或下载目的获取扩展制品的去重累计值。

规则：

- 同一用户同一扩展安装下载只计一次。
- 更新下载不计入下载量。
- 文件预览、审核下载、客户端更新包下载不计入扩展下载量。
- 安装失败不回退已发放下载凭证的统计。
- 勾选“仅授权范围内展示”的扩展不进入榜单统计，但下载量仍可在作者和管理端范围内查看。

## 2.15 使用量

使用量用于 MCP Server 等非典型下载型扩展。

规则：

- MCP Server 使用量固定为“去重成功配置写入用户数”。同一用户对同一 MCP Server 多次写入只计一次。
- Plugin 指标按安装模式区分：managed-package 使用去重成功安装用户数，config-plugin 使用去重成功配置写入用户数，manual-download 分开展示去重受控下载用户数和用户标记已安装数。
- 指标名称必须在界面标明，不得把查看次数、连接检测次数或配置复制次数伪装为真实使用量。
- 同一用户高频重复操作应做去重或限频统计，避免榜单被异常行为放大。
- 勾选“仅授权范围内展示”的扩展不进入榜单统计。

## 2.16 审计日志

审计日志记录关键业务动作。

审计日志必须包含：

- 操作人快照。
- 操作对象类型。
- 操作对象 ID。
- 动作。
- 结果。
- 原因。
- 请求 ID。
- 客户端信息。
- 操作时间。
- 变更前后摘要。

审计日志必须能与服务端运行日志通过 requestID 关联，用于排查发布、系统预审、审核、下载、安装、权限拒绝、MCP 配置写入、Plugin 安装和客户端更新问题。
