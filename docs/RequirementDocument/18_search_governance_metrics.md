# 18. 搜索、榜单与治理指标

## 18.1 搜索目标

搜索用于帮助用户快速发现企业内部可用的 Skill、MCP Server 和 Plugin。

已发布且未下架、未归档的扩展默认可被登录用户搜索和查看详情。勾选“仅授权范围内展示”的扩展只对授权范围内用户、作者和具备管理权限的管理员可见。授权范围只影响新安装、新下载、复制配置、新增启用、配置写入和更新等主操作；停用、卸载和清理本地托管配置不受当前授权范围限制。

## 18.2 搜索范围

搜索字段：

- 扩展名称。
- Extension ID。
- 描述。
- 类型。
- 分类。
- 标签。
- 作者。
- 部门。
- 适用工具。

类型特定字段：

- Skill：README 摘要、适用工具。
- MCP Server：serverName、接入方式、transport、权限声明摘要、数据访问摘要。
- Plugin：目标工具、安装模式、兼容版本。

服务端返回搜索结果前必须按可见选项过滤。

## 18.3 授权标记与可见标记

搜索结果返回当前用户可见的已发布扩展，并标记当前用户是否具备操作权限。

结果对象应包含：

- 当前用户是否可查看详情。
- 当前用户是否可安装或下载。
- 当前用户是否可复制配置。
- 当前用户是否可配置写入、启用或自动安装。
- 不可操作原因摘要。
- 可见选项摘要，仅对有权限用户或管理员展示。

前端不得通过隐藏扩展来表达未授权，而应由服务端依据可见选项过滤；对可见但未授权的扩展，在详情主操作区显示不可操作状态。

## 18.4 搜索结果类型组织

搜索结果必须按类型组织。

### 全部视图

全部视图按类型分组：

```text
Skill 结果
MCP Server 结果
Plugin 结果
```

### 单类型视图

单类型视图只展示当前类型卡片和当前类型筛选器。

## 18.5 排序规则

支持排序：

- 综合排序。
- 相关度。
- 最新发布。
- 最近更新。
- Star 数。
- 下载量、安装量或使用量。

综合排序考虑：

- 搜索相关度。
- Star 数。
- 下载量、安装量或使用量。
- 最近更新时间。
- 风险等级。
- 用户是否具备操作权限。

未授权但可见的扩展可以出现在结果中，但在综合排序中可适度降权。

## 18.6 分类和标签

分类固定为：

```text
开发 / 测试 / 文档 / 设计 / 运维 / 安全 / 集成 / 自动化 / 数据 / 知识 / 其他
```

标签固定为：

```text
代码 / 审查 / 改进 / 提示 / 规范 / 清单 / 文档 / 写作 / 测试 / 验收 / 前端 / 可访问 / 设计 / 运维 / 值班 / 事故 / 安全 / 权限 / 集成 / 适配 / 自动化 / 发布 / 数据 / 分析 / 入门 / 培训
```

规则：

- 每个扩展必须选择 1 个分类。
- 每个扩展选择 1 到 5 个标签。
- 不支持自由标签。
- 标签变更由系统管理员在系统设置中维护。

## 18.7 Star 规则

- 登录用户可 Star 自己可见的扩展。
- 同一用户对同一扩展只能 Star 一次。
- 取消 Star 后数量实时减少。
- Star 可影响综合排序。
- Star 动作写入普通操作日志，可选写审计日志。
- Star 不受授权范围限制，但受可见选项影响。
- 勾选“仅授权范围内展示”的扩展不进入榜单统计。

## 18.8 下载量、安装量与使用量规则

Skill 下载量采用安装下载去重口径。

规则：

- 同一用户同一 Skill 安装下载只计一次。
- 更新下载不计入下载量。
- 审核包下载不计入下载量。
- 文件预览不计入下载量。
- 客户端更新包下载不计入扩展下载量。

Plugin 统计口径：

- managed-package：使用去重成功安装用户数；必要时可另行展示受控下载量，但不得混入安装量。
- config-plugin：使用成功配置写入去重数。
- manual-download：分别展示去重受控下载用户数和用户标记已安装数，不合并为一个安装量。

MCP Server 统计口径：

- MCP Server 使用量固定为去重成功配置写入用户数。
- 连接检测成功次数、连接检测失败次数和配置复制次数可作为管理端辅助指标展示，但不得命名为使用量。
- 不得把详情查看次数伪装为真实运行次数。

勾选“仅授权范围内展示”的扩展不进入社区榜单统计，但管理端可在权限范围内查看其统计。

## 18.9 社区首页榜单

社区首页必须展示类型化榜单。

### Skill 区域

横向排列：

- Skill 热榜。
- Skill Star 榜。
- Skill 下载量榜。

### MCP Server 区域

横向排列：

- MCP Server 热榜。
- MCP Server Star 榜。
- MCP Server 使用量榜。

### Plugin 区域

横向排列：

- Plugin 热榜。
- Plugin Star 榜。
- Plugin 下载量/安装量榜。

### 周期切换

- Star 榜支持总榜、月榜、周榜切换。
- 下载量/使用量榜支持总榜、月榜、周榜切换。
- 热榜使用公式计算，不切换总榜、月榜、周榜。

## 18.10 热榜规则

热榜优先使用近 7 天数据。

建议计算因子：

- 近 7 天下载量、安装量或使用量。
- 近 7 天新增 Star。
- 最近更新时间。
- 风险等级降权。
- 异常事件降权。

建议公式：

```text
热度分 = 近7天下载、安装或使用量 * 0.50
       + 近7天新增Star * 0.35
       + 最近更新时间得分 * 0.15
       - 风险降权
       - 异常降权
```

热榜只在社区首页展示，不在结果页右侧重复堆叠。

## 18.11 周榜、月榜、总榜口径

| 榜单 | 周榜 | 月榜 | 总榜 |
|---|---|---|---|
| Star 榜 | 最近 7 天新增 Star | 最近 30 天新增 Star | 当前有效 Star 总数 |
| 下载量榜 | 最近 7 天去重下载 | 最近 30 天去重下载 | 发布以来去重下载 |
| 安装量榜 | 最近 7 天去重安装 | 最近 30 天去重安装 | 发布以来去重安装 |
| 使用量榜 | 最近 7 天去重成功配置写入用户数 | 最近 30 天去重成功配置写入用户数 | 发布以来去重成功配置写入用户数 |

去重口径以用户为主，必要时可增加设备 ID 限频，避免重复点击刷榜。

## 18.12 治理指标

管理端扩展详情展示：

- Star 数。
- 下载量、安装量或使用量。
- 近 7 天趋势。
- 安装用户数。
- 更新用户数。
- MCP 使用量：去重成功配置写入用户数。
- MCP 连接检测失败数。
- Plugin 安装成功数。
- Plugin 卸载失败数。
- 本地事件失败数。
- 权限拒绝次数。
- 不可见访问拒绝次数。
- 最近审核结果。
- 最近 AI 预审结果。
- 最近审计事件。

## 18.13 指标一致性

同一指标在不同页面必须口径一致。

示例：

- Skill 下载量在社区卡片、详情页、管理端列表中必须一致。
- MCP 使用量在社区榜单、详情页、管理端列表中必须标明同一口径。
- Plugin 安装量和下载量不得混用，页面必须标明。
- Star 数在取消后必须实时更新。
- 更新下载不得使下载量增加。
- 未授权用户查看详情不得增加下载量。
- 勾选“仅授权范围内展示”的扩展不得进入社区榜单统计。

## 18.14 验收标准

- 社区首页按 Skill、MCP Server、Plugin 三个区域展示榜单。
- 每个区域横向排列热榜、Star 榜、下载量/使用量榜。
- Star 榜和下载量/使用量榜支持总榜、月榜、周榜切换。
- 搜索结果按类型分组，不混排。
- Skill、MCP Server、Plugin 使用各自筛选项。
- 搜索结果由服务端按可见选项过滤。
- 勾选“仅授权范围内展示”的扩展对无权限用户不可见且不进入榜单统计。
- 下载量、安装量、使用量和 Star 口径清晰一致。
- 管理端可查看扩展治理指标。
