# 13. 管理端：扩展管理

## 13.1 页面目标

扩展管理用于部门管理员和系统管理员查看、治理和追踪企业内部扩展。

扩展管理只回答三个问题：

1. 当前有哪些扩展？
2. 哪些扩展存在风险或异常？
3. 我能对这些扩展执行什么治理动作？

## 13.2 数据范围

### 部门管理员

可访问：

- 本部门及下级部门发布的扩展。
- 授权范围包含本部门或下级部门的扩展摘要和治理信息。
- 本部门及下级部门用户提交的扩展申请记录。
- 管理范围内扩展相关审计入口。

### 系统管理员

可访问全部扩展。

## 13.10 治理动作权限表

| 动作 | 作者 | 部门管理员 | 系统管理员 | 是否需要审核 | 审计要求 |
|---|---|---|---|---:|---|
| 普通下架 | 可下架自己发布的扩展 | 可下架管理范围内扩展 | 可下架任意扩展 | 否 | 必须写审计 |
| 安全下架 | 不可执行 | 不可执行 | 可执行 | 否 | 必须写审计并通知存量用户 |
| 重新上架 | 提交重新上架申请 | 可处理管理范围内非安全下架扩展 | 可处理任意扩展，安全下架仅系统管理员可处理 | 视下架原因 | 必须写审计 |
| 归档 | 可提交归档申请 | 可归档管理范围内扩展 | 可归档任意扩展 | 否 | 必须写审计 |
| 授权收缩 | 可提交并直接生效 | 可直接处理管理范围内扩展 | 可直接处理任意扩展 | 否 | 必须写审计 |
| 授权扩大 | 提交申请 | 可处理未跨出管理范围的申请 | 可处理跨部门和全员申请 | 是 | 必须写审计 |
| 展示收缩 | 可直接生效 | 可直接处理管理范围内扩展 | 可直接处理任意扩展 | 否 | 必须写审计 |
| 展示扩大 | 提交申请 | 可处理管理范围内申请 | 可处理跨部门和全员影响申请 | 是 | 必须写审计 |
| 转移维护人或归属部门 | 不可执行 | 可在管理范围内执行 | 可全局执行 | 否 | 必须写审计 |

## 13.11 下架规则

### 普通下架

- 发布者可普通下架自己发布的扩展。
- 部门管理员可普通下架管理范围内扩展。
- 系统管理员可普通下架任意扩展。
- 下架必须填写原因。
- 下架原因必须从维护、过期、违规、其他中选择一类，并允许补充说明。
- 下架必须写审计日志。
- 下架后社区不再展示，不再允许新安装、接入、复制配置、下载或更新。
- 存量本地 Skill 不自动删除。
- 存量 MCP 托管配置不自动删除。
- 存量 Plugin 不自动删除。

### 安全下架

- 只有系统管理员可执行。
- 必须填写安全原因、影响范围和处置建议。
- 社区不再展示，不再允许新安装、接入、复制配置、下载或更新。
- 已安装、已接入或已下载用户收到强提醒。
- 本地页标记“安全风险”。
- 是否阻断存量继续使用由系统安全策略控制；默认不强制删除用户本地文件或配置。
- 安全下架后存量策略支持三档：仅提醒、阻止新增启用/接入、强制停用托管项。
- 强制停用托管项只能处理 Enterprise Agent Hub 管理的启用项、MCP 配置项或 Plugin 配置；目标工具直接加载或客户端离线运行的内容不能承诺立即阻断。

## 13.12 重新上架规则

- 已下架扩展可重新上架。
- 重新上架必须填写原因。
- 若下架原因为维护、过期或误操作，且版本、包、元信息、授权范围、可见选项均未变化，具备管理权限的管理员可直接重新上架。
- 若重新上架伴随新版本、元信息修改、授权扩大或展示扩大，必须创建发布申请并进入审核。
- 若下架原因为安全风险，只有系统管理员可以重新上架。
- 因安全风险重新上架必须提供修复说明；涉及包内容、MCP 配置、Plugin 安装清单、权限声明或可见选项变化时必须提交审核。
- 重新上架必须写审计日志。

## 13.13 归档规则

- 归档为终态。
- 归档必须二次确认。
- 归档必须填写原因。
- 归档必须写审计日志。
- 已归档扩展不得重新上架。
- 已归档扩展不得发布新版本。
- 已归档扩展不得扩大或收缩授权范围。
- 已归档扩展不得修改可见选项。
- 已归档扩展的 Extension ID 永久保留，不得被新扩展复用。

## 13.14 可见选项管理规则

- 默认可见：已发布扩展对所有登录用户可浏览，未授权用户不可执行主操作。
- 仅授权范围内展示：无权限用户不可见，不进入榜单统计。
- 展示收缩可直接生效并写审计。
- 展示扩大必须进入审核。
- 管理员在扩展详情中必须能看到当前可见选项和最近一次变更记录。

## 13.15 维护人和归属部门转移

管理员可在扩展详情中转移扩展维护人或归属部门。

规则：

- 部门管理员只能将管理范围内扩展转移给管理范围内用户或部门。
- 系统管理员可全局转移。
- 转移不改变历史作者、历史所属部门、历史审核记录和历史审计快照。
- 转移后，后续版本发布、授权变更、可见选项变更、下架申请和通知接收人按新的维护人或归属部门计算。
- 转移必须填写原因，并写审计日志。

## 13.16 统计信息

扩展详情展示：

- Star 数。
- 下载量、安装量或使用量。
- 近 7 天下载量、安装量或使用量。
- 近 7 天新增 Star。
- 已安装用户数。
- MCP 使用量：去重成功配置写入用户数。
- MCP 连接检测失败数。
- Plugin 成功安装用户数。
- Plugin 安装或卸载失败数。
- 本地事件异常数。
- 权限拒绝次数。

统计用于辅助治理，不作为管理员审核的唯一依据。勾选“仅授权范围内展示”的扩展不进入社区榜单，但管理端仍可在权限范围内查看统计。

## 13.17 验收标准

- 扩展管理必须支持 Skill、MCP Server、Plugin 类型切换。
- 部门管理员只能操作管理范围内扩展。
- 系统管理员可操作全部扩展。
- 下架、重新上架、归档、授权变更、可见选项变更必须写审计日志。
- 安全下架只能由系统管理员处理。
- 安全风险下架后的重新上架只能由系统管理员处理。
- 归档后不可重新上架。
- MCP Server 管理详情必须展示配置写入、连接检测和使用统计。
- Plugin 管理详情必须展示安装模式、下载或安装统计和失败统计。
