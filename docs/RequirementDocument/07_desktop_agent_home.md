# 7. 桌面端：Agent 首页

## 7.1 页面目标

Agent 首页是桌面客户端的默认入口。本需求文档只定义该入口在 Enterprise Agent Hub 中的位置、导航边界和与扩展中心的接口边界。

Agent 的任务编排、执行内核、模型调用、工具调用、上下文管理、会话管理和安全策略由独立 Agent 设计文档承接，本需求文档不展开设计、不作为 Agent 能力验收依据。

## 7.2 页面边界

本需求只要求：

- 桌面客户端一级导航保留 `Agent`。
- 用户登录后默认进入 Agent 首页。
- Agent 首页不得承载社区搜索、发布审核、部门管理、用户管理、审计日志等功能。
- Agent 首页可以读取“本地已安装或已接入扩展摘要”，但不得绕过服务端授权校验。
- Agent 首页与 Skill、MCP Server、Plugin 的深度调用关系由独立 Agent 设计文档定义。

## 7.3 与扩展中心的最小接口

Agent 首页在本需求范围内只允许依赖以下扩展中心能力：

- 获取当前用户摘要。
- 获取本机本地扩展摘要。
- 打开社区详情页。
- 打开本地扩展管理页。
- 打开设置弹窗中的服务连接配置。

Agent 运行时需要的上下文、工具调用、MCP 调用、Skill 调用和 Plugin 调用，不在本需求文档内定义。

## 7.4 不展示内容

Agent 首页不得展示：

- 发布申请状态堆叠列表。
- 审核入口。
- 管理端入口。
- 部门和用户管理。
- 大段说明文字。
- 与 Agent 当前任务无关的社区卡片网格。

## 7.5 验收标准

- 桌面客户端存在 `Agent` 一级入口。
- 登录后默认进入 Agent 首页。
- Agent 首页不包含管理端功能。
- Agent 具体执行能力不在本需求文档验收范围内。
