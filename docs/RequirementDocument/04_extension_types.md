# 4. 扩展类型与边界

## 4.1 类型边界原则

Skill、MCP Server、Plugin 共享以下能力：

- 社区首页曝光。
- 搜索。
- 详情展示。
- Star。
- 发布申请。
- 系统预审。
- 管理员审核。
- 上架、下架、归档。
- 审计日志。
- 本地事件同步。

但三者不得在用户界面中被混为一种对象。系统必须在以下位置明确展示类型：

- 社区首页榜单。
- 搜索结果页。
- 扩展卡片。
- 详情页头部。
- 发布向导第一步。
- 审核列表。
- 扩展管理列表。
- 审计日志对象类型。

授权范围控制新安装、新下载、复制配置、新增启用、配置写入和更新等主操作。停用、卸载和清理本机托管配置不受当前授权范围限制。可见选项控制未授权用户是否可在社区看到扩展；勾选“仅授权范围内展示”后，该扩展不进入榜单统计。

## 4.2 Skill

### 定位

Skill 是文件型能力包，支持完整安装、更新、启用、停用、卸载闭环。

### 必填元数据

- Extension ID。
- 显示名称。
- 描述。
- 分类。
- 标签。
- 版本号。
- 变更说明。
- 适用工具。
- 适用系统。
- 授权范围。
- 可见选项。
- 风险声明。

### 包要求

- 支持文件夹上传。
- 支持 zip 上传。
- 根目录必须包含 `SKILL.md`。
- 推荐包含 `README.md`。
- 单包大小不超过 5MB。
- 包内文件数不超过 100。

### 社区展示

Skill 卡片展示：

- Skill 标识。
- 名称。
- 两行以内描述。
- 分类。
- 最多 3 个标签。
- 作者。
- 部门。
- 当前版本。
- Star 数。
- 下载量。
- 本机安装状态。

### 详情展示

Skill 详情展示：

- 基础信息。
- 适用工具。
- 适用系统。
- 分类和标签。
- 版本历史。
- README。
- 审核与风险信息。
- 已安装和已启用状态。
- 授权范围、可见选项和当前用户操作状态。
- 操作区：安装、更新、启用范围、Star。

未在授权范围内但可见的用户可查看详情，但安装、更新和新增启用操作不可用，并展示简短原因；如果本机已有托管内容，停用和卸载仍可执行。

### 本地能力

Skill 支持：

- 安装。
- 更新。
- 启用。
- 停用。
- 卸载。
- 本地扫描。
- 本地导入。

## 4.3 MCP Server

### 定位

MCP Server 是接入型扩展，用于向 Agent 或 AI 工具提供上下文、工具调用或数据访问能力。Enterprise Agent Hub 负责 MCP Server 的接入定义、配置模板、权限治理、本地配置写入、更新、卸载、统计和审计，不负责自动部署业务 MCP 服务进程。

### 支持的接入方式

MCP Server 必须选择一种接入方式：

| 接入方式 | 说明 | 本地闭环 |
|---|---|---|
| remote-http | MCP Streamable HTTP Endpoint，服务端进程独立运行 | 写入配置、连接检测、更新、卸载 |
| remote-sse | 兼容旧版 HTTP+SSE 的远端端点 | 写入配置、连接检测、更新、卸载 |
| local-command | 用户本机已有命令或工具提供 stdio 服务 | 检测命令、写入配置、连接检测、更新、卸载 |

接入方式与 transport 的合法组合：

| 接入方式 | 合法 transport | 说明 |
|---|---|---|
| remote-http | streamable-http | 当前推荐的远端 HTTP 传输形态，配置中保存 MCP endpoint |
| remote-sse | sse | 兼容旧版 HTTP+SSE 服务，界面必须标记为 legacy |
| local-command | stdio | 客户端启动本机命令，通过 stdin/stdout 交换 MCP 消息 |

规则：

- `http` 可作为 UI 兼容输入，但服务端落库应规范化为 `streamable-http`。
- MVP 暂不做 endpoint 是否属于企业内网地址的阻断校验；服务端仅校验 URL 格式、协议和必填字段，并在审核详情中展示端点风险摘要。内网域名或 CIDR allowlist 作为系统设置预留项。
- local-command 不允许服务端下发真实密钥，不允许静默执行安装脚本。
- local-command 的 command、args、工作目录和环境变量摘要必须在发布、审核和详情中可见。
- local-command 使用 `powershell`、`cmd`、`bash`、`sh`、`curl`、`wget` 等通用执行器，或 args 中出现管道、重定向、下载后执行语义时，必须标记为高风险并交由管理员重点审核。
- connectionTest 不得执行有副作用的命令；只能做命令存在性、版本读取、只读健康检查或 Tool Adapter 支持的安全检测。
- 如果本机缺少命令依赖，客户端显示“依赖缺失”，允许复制配置和查看修复建议，但不得展示写入成功。

### 必填元数据

- Extension ID。
- 显示名称。
- 描述。
- 分类。
- 标签。
- 版本号。
- 变更说明。
- 适用工具。
- 接入方式。
- transport 类型：`stdio`、`streamable-http` 或 `sse`。
- 配置模板。
- 变量 schema。
- 连接检测方式。
- 权限声明。
- 数据访问说明。
- 环境要求。
- 授权范围。
- 可见选项。
- 风险声明。

### 配置要求

MCP Server 发布申请必须提交 MCP 定义清单。清单至少包含：

- serverName。
- transport：`stdio`、`streamable-http` 或 `sse`；兼容输入 `http` 时服务端规范化为 `streamable-http`。
- command 或 endpoint。
- args。
- env 示例。
- variables schema。
- configTemplate。
- supportedTools。
- healthCheck 或 connectionTest。
- permissions。
- dataAccess。
- sensitiveFields。

变量规则：

- env 示例中不得填写真实密钥。
- 变量必须声明名称、类型、是否必填、是否敏感、默认值、说明。
- 敏感变量只允许用户在本机填写，客户端使用 Windows 安全能力保存，服务端只保存变量定义和脱敏摘要。
- 配置模板必须能渲染出目标工具所需的 MCP 配置片段。

### 社区展示

MCP Server 卡片展示：

- MCP 标识。
- 名称。
- 两行以内描述。
- 接入方式。
- 适用工具。
- 作者。
- 部门。
- 当前版本。
- Star 数。
- 使用量，固定为去重成功配置写入用户数。
- 企业状态。

### 详情展示

MCP Server 详情展示：

- 基础信息。
- 接入方式。
- transport。
- 端点或命令摘要。
- 配置示例。
- 变量说明。
- 权限声明。
- 数据访问说明。
- 连接检测说明。
- 版本历史。
- 审核与风险信息。
- 授权范围、可见选项和当前用户操作状态。
- 操作区：安装到工具、连接检测、复制配置、更新、卸载、Star。

未在授权范围内但可见的用户可查看详情和说明，但复制配置、下载配置、安装到工具、连接检测和更新操作不可用；如果本机已写入托管 MCP 配置，卸载配置仍可用。

### 本地能力

MCP Server 必须支持完整本地接入闭环：

- 选择目标工具。
- 检测目标工具是否支持 MCP 配置写入。
- 填写必填变量。
- 敏感变量本机安全保存。
- 预览将写入的配置摘要。
- 生成写入执行计划。
- 备份目标工具原配置。
- 写入托管 MCP 配置项。
- 执行连接检测。
- 记录本地接入状态。
- 更新配置版本。
- 编辑本地变量。
- 停用或卸载托管配置项。
- 失败时尽量回滚。
- 同步本地事件到服务端。

## 4.4 Plugin

### 定位

Plugin 是面向指定 AI 工具、内部工具或 Agent 环境的插件型扩展。Plugin 必须以明确的安装模式交付，避免用户只看到说明却不知道能否安装。

### 安装模式

Plugin 必须选择一种安装模式：

| 安装模式 | 说明 | 本地闭环 |
|---|---|---|
| managed-package | 服务端托管插件包，客户端通过适配器安装到目标工具 | 下载、Hash 校验、安装、更新、卸载、回滚 |
| config-plugin | 通过配置清单写入目标工具配置 | 写入、启用、禁用、更新、卸载、回滚 |
| manual-download | 提供受控下载和安装说明，由用户手动安装 | 受控下载、本地状态记录、手动卸载说明 |

规则：

- managed-package 和 config-plugin 必须提供安装清单。
- manual-download 必须在卡片和详情中明确标注“手动安装”，不得展示自动安装按钮。
- 任何模式都不得在没有适配器或安装清单的情况下直接写入用户工具目录。

### 必填元数据

- Extension ID。
- 显示名称。
- 描述。
- 分类。
- 标签。
- 版本号。
- 变更说明。
- 插件目标工具。
- 安装模式。
- 兼容版本。
- 权限声明。
- 授权范围。
- 可见选项。
- 风险声明。

### 包或说明要求

Plugin 可以是：

- 插件包。
- 安装清单。
- 手动安装说明。
- Hub 受控下载文件。
- 企业内部下载地址。
- 配置清单。

若提交文件包：

- 文件大小必须受限。
- 必须有 SHA-256。
- 必须可供管理员查看文件清单。
- 优先使用 Hub 托管文件，并通过服务端下载凭证下载。
- 如必须使用企业内部下载地址，发布申请必须登记 SHA-256、文件大小、来源系统和有效期；客户端下载后仍必须校验 Hash，下载事件必须同步服务端。

若提交安装清单：

- 必须声明目标工具。
- 必须声明安装位置策略。
- 必须声明启用、禁用、更新、卸载步骤。
- 必须声明回滚能力和不可回滚风险。
- 不得包含真实密钥或用户私有路径。

### 社区展示

Plugin 卡片展示：

- Plugin 标识。
- 名称。
- 两行以内描述。
- 目标工具。
- 安装模式。
- 作者。
- 部门。
- 当前版本。
- Star 数。
- 下载量或安装量。
- 企业状态。

### 详情展示

Plugin 详情展示：

- 基础信息。
- 目标工具。
- 安装模式。
- 兼容版本。
- 安装说明。
- 权限声明。
- 风险信息。
- 文件信息或配置清单摘要。
- 版本历史。
- 审核摘要。
- 授权范围、可见选项和当前用户操作状态。
- 操作区：安装、下载、更新、卸载、查看安装说明、Star。

未在授权范围内但可见的用户可查看详情和安装说明，但受控文件下载、自动安装和更新操作不可用；如果本机已有托管安装或手动安装记录，卸载和清理本地记录仍可用。

### 本地能力

Plugin 必须支持以下能力：

- 识别安装模式。
- 检测目标工具和兼容版本。
- managed-package：下载、Hash 校验、安装、更新、卸载、回滚。
- config-plugin：写入配置、启用、禁用、更新、卸载、回滚。
- manual-download：受控下载、打开安装说明、记录“用户标记已安装”、记录“用户标记已卸载”。
- 生成本地事件并同步服务端。
- 安装失败时展示失败位置、原因和建议动作。

## 4.5 类型筛选规则

社区搜索结果页必须提供类型切换：

- Skill。
- MCP Server。
- Plugin。

允许提供“全部”入口，但“全部”结果必须按类型分组展示，不得把不同类型卡片混排成一个无边界网格。

## 4.6 类型化筛选规则

不同类型使用不同筛选项。

### Skill 筛选

- 分类。
- 标签。
- 适用工具。
- 安装状态。
- 风险等级。
- 排序。

### MCP Server 筛选

- 接入方式。
- transport。
- 适用工具。
- 权限等级。
- 风险等级。
- 本机接入状态。
- 排序。

### Plugin 筛选

- 目标工具。
- 安装模式。
- 兼容版本。
- 风险等级。
- 本机安装状态。
- 排序。

## 4.7 统一操作规则

| 操作 | Skill | MCP Server | Plugin |
|---|---:|---:|---:|
| 查看详情 | 是，受可见选项影响 | 是，受可见选项影响 | 是，受可见选项影响 |
| Star | 是 | 是 | 是 |
| 发布申请 | 是 | 是 | 是 |
| 系统预审 | 是 | 是 | 是 |
| 审核 | 是 | 是 | 是 |
| 下架 | 是 | 是 | 是 |
| 重新上架 | 是 | 是 | 是 |
| 归档 | 是 | 是 | 是 |
| 安装到 Central Store | 是 | 不适用 | managed-package 可用 |
| 写入工具配置 | 启用时可写入 | 是 | config-plugin 可用 |
| 启用到工具或项目 | 是 | 是，表现为 MCP 配置接入 | 是，按安装模式 |
| 停用或卸载 | 是 | 是 | 是 |
| 复制配置 | 不适用 | 是 | 按插件类型 |
| 下载文件 | 是 | 按类型 | 是 |
| 连接检测 | 不适用 | 是 | 按插件类型 |

说明：安装、复制配置、下载文件、配置写入、新增启用和更新受授权范围控制；停用、卸载和清理本机托管配置不受当前授权范围限制；查看详情受可见选项控制；Star 对可见用户开放。
