# 6. 信息架构与导航

## 6.1 总体结构

系统分为：

1. Windows 桌面客户端。
2. Web 管理端。
3. 服务端 API。

桌面客户端和 Web 管理端必须使用不同导航结构，不得把管理功能放入桌面客户端。

## 6.2 桌面客户端一级导航

桌面客户端只有三个一级入口：

```text
Agent  |  社区  |  本地
```

右上角固定入口：

```text
通知  |  账号  |  设置
```

### Agent

默认首页。此处只定义桌面客户端首页入口和扩展中心边界；Agent 任务编排、执行内核、模型调用和工具调用细节由独立 Agent 设计文档承接。

### 社区

进入后首先展示社区首页。社区首页包含搜索框、发布按钮、类型化榜单和类型入口。只有用户提交搜索、点击榜单或进入类型入口后，才进入结果页展示卡片、分类和筛选。

### 本地

管理本机已安装扩展、工具和项目。

## 6.3 桌面客户端页面关系

```text
Agent 首页
  └─ Agent 具体能力由独立 Agent 设计文档承接，本需求仅保留入口和与扩展中心的边界

社区首页
  ├─ 搜索 -> 搜索结果页
  ├─ Skill 榜单区 -> Skill 详情 / Skill 结果页
  ├─ MCP 榜单区 -> MCP 详情 / MCP 结果页
  ├─ Plugin 榜单区 -> Plugin 详情 / Plugin 结果页
  ├─ Skill 探索 -> Skill 结果页
  ├─ MCP 探索 -> MCP 结果页
  ├─ Plugin 探索 -> Plugin 结果页
  └─ + 发布 -> 发布向导

本地
  ├─ 已安装扩展
  ├─ 工具
  └─ 项目
```

## 6.4 社区首页要求

社区首页是一级入口默认落点，必须独立展示，不显示搜索结果卡片网格。

社区首页包含：

- 大搜索框。
- 发布按钮。
- Skill 榜单区。
- MCP Server 榜单区。
- Plugin 榜单区。
- 类型入口卡片。

每个榜单区按横向三列展示：

```text
热榜  |  Star 榜  |  下载量/使用量榜
```

要求：

- Skill 区域展示 Skill 热榜、Skill Star 榜、Skill 下载量榜。
- MCP Server 区域展示 MCP Server 热榜、MCP Server Star 榜、MCP Server 使用量榜。
- Plugin 区域展示 Plugin 热榜、Plugin Star 榜、Plugin 下载量/安装量榜。
- Star 榜和下载量/使用量榜支持总榜、月榜、周榜切换。
- 热榜由统一公式计算，不提供总榜、月榜、周榜切换。
- 每个榜单默认展示前 5 条，避免首页重新变成复杂运营页面。

社区首页不得包含：

- 大量筛选器。
- 扩展卡片网格。
- 长篇说明文字。
- 管理入口。

## 6.5 社区搜索结果页要求

只有发生以下行为后，才进入搜索结果页：

- 用户在社区首页提交搜索。
- 用户点击探索 Skill。
- 用户点击探索 MCP Server。
- 用户点击探索 Plugin。
- 用户点击榜单的“查看更多”。
- 用户从通知或详情返回结果列表。

搜索结果页必须按类型组织。

允许有“全部”结果视图，但必须分组展示：

```text
Skill 结果
MCP Server 结果
Plugin 结果
```

不得把不同类型对象直接混排在同一组卡片中。

## 6.6 Web 管理端一级导航

Web 管理端一级导航：

```text
概览
审核
扩展管理
部门与用户
审计日志
客户端设备
客户端更新
系统设置
```

部门管理员可访问：

- 概览。
- 审核。
- 扩展管理。
- 部门与用户。
- 审计日志。
- 客户端设备，仅限管理范围内用户设备。

系统管理员可访问全部入口。

客户端设备入口仅用于查看设备登记、版本分布、最近在线时间和更新事件，不提供普通用户桌面功能。

## 6.7 Web 管理端页面关系

```text
概览
  ├─ 待审核跳转 -> 审核
  ├─ 异常扩展跳转 -> 扩展管理
  └─ 最近审计跳转 -> 审计日志

审核
  ├─ 待处理
  ├─ 处理中
  ├─ 已处理
  └─ 审核详情

扩展管理
  ├─ Skill
  ├─ MCP Server
  ├─ Plugin
  └─ 扩展详情

部门与用户
  ├─ 组织结构
  │  ├─ 部门树
  │  └─ 部门详情
  └─ 用户账号
     ├─ 用户列表
     └─ 用户详情

审计日志
  ├─ 操作日志列表
  └─ 审计详情

客户端设备
  ├─ 设备列表
  ├─ 版本分布
  └─ 设备事件

客户端更新
  ├─ 版本列表
  ├─ 发布新版本
  └─ 更新事件

系统设置
  ├─ 基础信息
  ├─ 分类标签
  ├─ 服务配置
  ├─ AI 系统预审
  └─ 安全策略
```

## 6.8 模态、抽屉与页面规则

### 桌面客户端

使用模态或抽屉处理：

- 登录。
- 设置。
- 通知。
- 扩展详情。
- MCP 配置接入弹窗。
- Plugin 安装弹窗。
- 发布向导。
- 启用范围选择。
- 危险操作确认。

不得使用独立一级页面处理发布中心、通知中心、设置中心。

### Web 管理端

使用详情页或右侧抽屉处理：

- 审核详情。
- 扩展详情。
- 用户详情。
- 设备详情。
- 审计详情。

复杂审核详情可使用全屏详情页。

## 6.9 导航权限规则

- 普通用户无法进入 Web 管理端。
- 部门管理员登录 Web 管理端后只看到其有权访问的入口。
- 系统管理员登录 Web 管理端后看到全部入口。
- 桌面客户端不因用户是管理员而增加管理页签。
- 桌面客户端账号菜单可显示用户角色，但不显示后台入口。

## 6.10 页面文案规则

- 页面标题必须描述任务，而不是解释系统。
- 首屏不放超过两行的说明文字。
- 说明信息优先放到 tooltip、空状态、错误状态或帮助链接。
- 按钮文案必须是动作，如“安装”“更新”“提交发布”“通过”“退回修改”。
