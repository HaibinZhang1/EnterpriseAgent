# 企业内部 Agent 扩展中心需求说明

## 文档信息

- 文档名称：企业内部 Agent 扩展中心需求说明
- 文档目录：`docs/NewRequirementDocument/`
- 文档版本：V1.2（重构补充版）
- 适用产品：Enterprise Agent Hub
- 适用场景：企业内网、自建服务端、Windows 桌面客户端、Linux Docker 服务端、独立 Web 管理端
- 目标用户规模：几百人级别，分属多个部门
- 交付优先级：桌面普通用户体验、扩展发布与审核、部门级治理、审计排障、离线可运行

## 产品定位

Enterprise Agent Hub 是企业内部的 Agent 扩展中心，用于统一发现、发布、审核、安装和治理面向企业 AI 工具的扩展能力。

系统包含三类核心扩展：

1. **Skill**：面向 AI Agent 的文件型能力包，以 `SKILL.md` 为入口。
2. **MCP Server**：面向 Agent 或 AI 工具的 Model Context Protocol 接入型扩展对象。
3. **Plugin**：面向指定 AI 工具或内部工具的插件型扩展对象。

系统由三部分组成：

1. **Windows 桌面客户端**：普通用户使用 Agent 入口、发现扩展、安装或接入扩展、管理本地启用、提交发布申请。
2. **服务端 API**：保存账号、部门、扩展、发布、审核、安装、通知、审计等数据，是社区、权限和版本的权威来源。
3. **Web 管理端**：部门管理员和系统管理员处理审核、管理用户与部门、治理扩展、查看审计日志、查看客户端设备、发布客户端更新。

## 设计原则

1. **任务优先**：页面必须围绕用户要完成的任务组织，不使用大段解释型说明堆叠页面。
2. **角色简单**：系统只保留普通用户、部门管理员、系统管理员三类角色。
3. **端侧分工清晰**：桌面客户端服务普通用户与发布者；Web 管理端服务管理员。
4. **授权与展示分离**：授权范围控制安装、下载、复制配置、新增启用、配置写入和更新等主操作；停用、卸载和本地清理不因后续授权收缩而被禁止；发布时可选择是否“仅授权范围内展示”，勾选后无权限用户不可见且不进入榜单统计。
5. **扩展类型清晰**：Skill、MCP Server、Plugin 共享发现与治理能力，但在列表、筛选、详情、发布表单中保持类型边界，不混为同一种对象。
6. **部门治理可控**：部门范围的发布内容由对应部门管理员审核；跨部门或全员影响由系统管理员审核。
7. **上级管理下级**：上级部门管理员可以管理下级部门、下级普通用户和下级部门管理员，但不得横向或向上操作管理员。
8. **流程简洁**：普通用户只看到必要状态；系统内部状态不得直接增加用户理解成本。
9. **审计内置**：管理、发布、系统预审、审核、下载、安装同步、客户端设备、客户端更新等关键动作必须可追踪。
10. **内网优先**：不依赖互联网服务；开发环境构建安装包后复制到内网部署运行。
11. **准确优先**：服务端是扩展、版本、权限、可见选项、审核和统计的唯一权威来源；客户端只负责本机执行和缓存。
12. **模板统一**：卡片、详情、发布表单、审核详情、空状态、错误状态使用统一模板。

## 本版修订要点

- 保留统一的“授权范围”作为操作权限控制，不再使用复杂的可见范围体系。
- 发布时增加“可见选项”：默认已发布扩展对登录用户展示；勾选“仅授权范围内展示”后，无权限用户不可见且不进入榜单统计。
- MCP Server 和 Plugin 从“展示说明”补齐为可发布、可审核、可安装或接入、可更新、可卸载、可统计、可审计的完整可用闭环。
- 部门管理员支持管理下级部门管理员，满足部门和管理员的日常增删改查需求。
- 统一称为“管理员”，不引入额外角色名称。
- 社区首页按 Skill、MCP Server、Plugin 三个区域组织，每个区域横向排列热榜、Star 榜、下载量或使用量榜。
- 补充发布、权限变更、下架、重新上架、归档的权限和审核边界。
- 补充包安全校验、Windows 本地事务与回滚、审计日志排障、客户端设备与更新事件、客户端更新签名校验和技术架构边界。
- 明确桌面客户端采用 Electron + React，Web 管理端采用 React，服务端采用 Spring Boot 3 + Java 21 模块化单体架构并支持接入内网 AI 服务进行系统预审，数据库采用 PostgreSQL。
- 明确 Extension ID 全局唯一，归档后 ID 不复用。
- 明确 MCP Server 接入方式与 transport 合法组合：remote-http/streamable-http、remote-sse/sse、local-command/stdio。
- 固定 MCP Server 使用量口径为去重成功配置写入用户数，并细化 Plugin 各安装模式统计口径。
- 调整发布申请流程为先创建 submission/revision，再执行规则预审和 AI 系统预审；退回修改保留 revision 历史。
- 明确授权收缩、下架、安全下架、离线状态下的本地保留、停用、卸载和新增主操作边界。
- 补充用户账号安全策略、不可变 userID、部门停用规则、客户端设备入口权限和扩展维护人/归属部门转移。
