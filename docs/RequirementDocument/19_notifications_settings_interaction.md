# 19. 通知、设置与交互反馈

## 19.1 通知目标

通知用于提醒用户处理与自己相关的发布、审核、更新和本地状态变化。

通知入口固定在桌面客户端和 Web 管理端右上角，不作为一级页面。

## 19.2 桌面通知类型

普通用户可收到：

- 发布申请已提交。
- 发布申请通过。
- 发布申请退回修改。
- 发布申请拒绝。
- 已安装 Skill 有新版本。
- 已接入 MCP Server 有新版本。
- 已安装 Plugin 有新版本。
- 授权收缩提示。
- 可见选项变化提示。
- 安装结果。
- MCP 配置写入结果。
- MCP 连接检测结果。
- Plugin 安装结果。
- 更新结果。
- 启用结果。
- 卸载结果。
- 安全下架提醒。
- 客户端有新版本。
- 企业服务连接恢复或失败。

## 19.3 管理端通知类型

部门管理员可收到：

- 本部门有新的待审核申请。
- 本部门高风险申请。
- 本部门 AI 预审警告申请。
- 审核处理超时提醒。
- 本部门扩展异常。

系统管理员可收到：

- 全员或跨部门申请。
- 客户端设备异常。
- 客户端更新异常。
- AI 系统预审不可用。
- 全局高风险扩展。
- 系统设置变更提醒。

## 19.4 通知面板

通知面板包含：

- 全部 / 未读切换。
- 最近通知列表。
- 全部已读按钮。

通知项展示：

- 标题。
- 摘要。
- 类型。
- 时间。
- 已读状态。

点击通知后跳转到对应对象：

- 发布进度 -> 我的提交。
- 审核任务 -> 管理端审核详情。
- Skill 更新 -> 本地已安装扩展。
- MCP 更新或连接异常 -> 本地已安装扩展。
- Plugin 更新或安装异常 -> 本地已安装扩展。
- 客户端设备异常 -> 管理端客户端设备详情。
- 客户端更新 -> 更新提示弹窗。

## 19.5 设置入口

桌面客户端设置通过账号菜单打开，不作为一级导航。

设置包含：

- 常规偏好。
- Agent 接入。
- 本地环境。
- 同步与更新。
- 关于。

## 19.6 常规偏好

包含：

- 语言。
- 主题。
- 启动默认页。
- 成功提示显示策略。

## 19.7 Agent 接入

本节只保留桌面客户端设置入口。Agent 具体能力和模型调用策略由独立 Agent 设计文档承接。

包含：

- 模型服务类型。
- Base URL。
- API Key。
- 默认模型。

API Key 默认隐藏，不得明文显示在摘要中。

## 19.8 本地环境

展示：

- Central Store 路径。
- MCP 配置写入能力检测。
- Plugin 安装适配器检测。
- 企业服务地址。
- 当前连接状态。
- 本机设备 ID。
- 客户端版本。

## 19.9 同步与更新

包含：

- 是否自动同步本地事件。
- 是否显示安装结果。
- 是否显示启用结果。
- 是否显示 MCP 连接检测结果。
- 是否显示 Plugin 安装结果。
- 客户端更新检查。

## 19.10 空状态

所有列表类页面必须有空状态。

包括：

- 社区搜索无结果。
- 本地已安装为空。
- MCP Server 未接入。
- Plugin 未安装。
- 工具未检测到。
- 项目为空。
- 我的提交为空。
- 审核待办为空。
- 扩展管理为空。
- 用户列表为空。
- 审计日志为空。

空状态必须包含：

- 当前状态一句话。
- 推荐下一步动作。

## 19.11 错误状态

错误提示必须包含：

- 动作语义。
- 错误原因。
- 下一步操作。

示例：

- `安装 Skill 失败：下载包 Hash 校验不通过。请重新安装或联系管理员。`
- `提交发布申请失败：Extension ID 已存在。请更换 ID。`
- `审核失败：该申请已被其他管理员处理。请刷新列表。`
- `MCP 配置写入失败：目标工具配置文件不可写。请检查工具路径或联系管理员。`
- `Plugin 安装失败：目标工具版本不兼容。请升级目标工具或选择手动安装。`
- `查看详情失败：该扩展不可见或不存在。`

## 19.12 成功提示

以下动作必须有成功反馈：

- 登录成功。
- 发布申请提交成功。
- 审核通过。
- 退回修改。
- 拒绝。
- 安装成功。
- MCP 配置写入成功。
- MCP 连接检测成功。
- Plugin 安装成功。
- 更新成功。
- 启用成功。
- 停用成功。
- 卸载成功。
- 用户创建成功。
- 用户冻结成功。
- 扩展下架成功。
- 客户端更新发布成功。

## 19.13 确认弹窗

以下动作必须二次确认：

- 下架扩展。
- 安全下架扩展。
- 归档扩展。
- 撤回发布申请。
- 卸载 Skill。
- 卸载 MCP 配置。
- 卸载 Plugin。
- 修改可见选项。
- 覆盖更新。
- 删除用户。
- 冻结用户。
- 删除部门。
- 发布客户端更新。
- 暂停或撤回客户端更新。

确认弹窗必须展示影响摘要。

## 19.14 页面密度

- 页面不使用大段解释性文字。
- 说明文字优先放在字段帮助、空状态或确认弹窗。
- 卡片描述最多两行。
- 卡片按钮最多保留一个主操作和一个轻操作。
- 详情页承载完整信息。

## 19.15 验收标准

- 通知不作为一级页面。
- 设置不作为一级页面。
- 错误提示包含动作语义。
- 危险操作都有确认弹窗。
- 页面无大段解释型内容堆叠。
- MCP、Plugin、可见选项、AI 预审和客户端设备相关通知都有明确跳转。
