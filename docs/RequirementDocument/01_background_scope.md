# 1. 背景、目标与边界

## 1.1 背景

企业内部会持续沉淀用于不同 AI 工具、项目场景和部门业务的 Agent 扩展。扩展类型包括 Skill、MCP Server、Plugin 等。若缺少统一的企业内部扩展中心，会出现以下问题：

- 扩展散落在个人电脑、部门共享目录或聊天消息中，无法统一发现。
- 不同部门重复建设相似能力，无法复用。
- 扩展发布缺少审核和版本记录，安全与质量不可控。
- 普通用户不知道哪些扩展适合自己，也不知道如何安装到本地工具。
- 管理员缺少审计日志，排查发布、审核、安装、权限问题困难。
- 插件、MCP、Skill 分散管理，后续扩展能力难以统一治理。

## 1.2 产品目标

搭建一个企业内网可部署的 Agent 扩展中心，形成以下闭环：

1. 普通用户可在桌面端进入 Agent 首页；Agent 具体能力由独立 Agent 设计文档承接。
2. 普通用户可在社区发现 Skill、MCP Server、Plugin。
3. 普通用户可安装 Skill，并在本地工具或项目中启用。
4. 普通用户可安装或接入 MCP Server，并将配置写入受支持的本地工具。
5. 普通用户可安装、更新、卸载受支持的 Plugin，或按手动模式完成受控下载和本地记录。
6. 普通用户可提交 Skill、MCP Server、Plugin 的发布申请。
7. 部门管理员可审核本部门及管理范围内用户提交的发布申请。
8. 系统管理员可审核跨部门或全员影响的发布申请，并进行全局治理。
9. 管理员可在 Web 管理端完成用户、部门、扩展、审计、客户端设备、客户端更新等管理任务。
10. 服务端完整记录关键操作日志和审计日志，便于问题排查和责任追踪。
11. 服务端可接入企业内网 AI 服务对发布申请进行系统预审，但最终审核决定仍由管理员作出。
12. 桌面客户端和服务端均支持外网开发后复制安装包到企业内网运行。

## 1.3 部署环境

### 客户端

- 运行环境：Windows 用户电脑。
- 交付方式：Windows x64 安装包。
- 运行方式：桌面应用。
- 网络环境：企业内部网络。
- 外网依赖：运行时不得依赖互联网服务。

### 服务端

- 运行环境：Linux 服务器。
- 交付方式：Docker Compose 或等价容器化部署。
- 数据存储：PostgreSQL + 服务端文件存储卷。
- 访问范围：仅企业内部网络。
- 外网依赖：运行时不得依赖互联网服务。

### Web 管理端

- 运行环境：浏览器访问企业内网地址。
- 用户范围：部门管理员、系统管理员。
- 部署方式：由服务端或独立静态资源服务提供。

## 1.4 当前交付范围

### 桌面客户端

必须包含：

- Agent 首页。
- 社区首页。
- 社区搜索结果页。
- Skill / MCP Server / Plugin 分类发现。
- 扩展卡片和详情展示。
- Skill 安装、更新、卸载。
- Skill 启用到本地工具或项目。
- MCP Server 安装或接入、配置写入、连接检测、更新和卸载。
- Plugin 受控安装、更新、卸载；手动安装模式提供受控下载和本地记录。
- 本地工具扫描和配置。
- 本地项目维护和配置。
- 发布入口按钮和发布向导。
- 通知面板。
- 设置弹窗。
- 登录、退出、会话失效处理。

不得包含：

- 部门管理页面。
- 用户管理页面。
- 审核工作台。
- 审计日志页面。
- 客户端更新包发布页面。
- 系统配置管理页面。

### Web 管理端

必须包含：

- 管理概览。
- 审核工作台。
- 扩展管理。
- 部门管理。
- 用户管理。
- 审计日志。
- 客户端设备。
- 客户端更新。
- 系统设置。

### 服务端

必须包含：

- 账号与会话。
- 部门与角色。
- 扩展主档。
- 扩展版本。
- 发布申请。
- 审核。
- 包存储与下载凭证。
- MCP 配置模板与本地接入记录。
- Plugin 安装包、安装清单与本地安装记录。
- 安装与本地事件同步。
- Star 与下载量统计。
- 通知。
- 审计日志。
- 客户端设备登记。
- 客户端更新。
- AI 系统预审。
- 健康检查。
- 备份与恢复脚本。

## 1.5 扩展类型范围

当前正式支持三类扩展：

| 类型 | 是否可发布 | 是否可审核 | 是否可展示 | 是否可安装或接入 | 是否可本地启用或配置 |
|---|---:|---:|---:|---:|---:|
| Skill | 是 | 是 | 是 | 是 | 是 |
| MCP Server | 是 | 是 | 是 | 是 | 是，写入受支持工具的 MCP 配置 |
| Plugin | 是 | 是 | 是 | 是 | 是，按插件安装模式和适配器能力执行 |

说明：

- Skill 是文件型能力包，完整支持安装、更新、启用、停用、卸载。
- MCP Server 是接入型扩展，完整支持发布、审核、展示、授权用户安装接入、配置写入、连接检测、更新、卸载、事件同步和审计。
- Plugin 是插件型扩展，完整支持发布、审核、展示、受控下载或受控安装、更新、卸载、事件同步和审计。
- Enterprise Agent Hub 不负责在服务端自动部署业务 MCP 服务进程；MCP 的服务进程由发布者部署在企业内网或由用户本机已有命令提供，Hub 负责其接入定义、配置模板、权限治理和本地配置写入。
- 对不具备适配器的目标工具，Plugin 可作为“手动安装模式”发布，但必须明确标注，不得伪装为自动安装。

## 1.6 当前明确不做

以下事项不进入当前正式交付范围：

- 面向互联网的公开市场。
- 第三方在线账号登录。
- 外部 LDAP、AD、企业统一身份源接入。
- macOS 和 Linux 桌面安装包。
- 自动 RAG 检索增强。
- 自动脚本安全沙箱执行。
- 自动部署或托管业务 MCP Server 运行进程。
- 自动修复用户本地工具环境。
- 多维护者协作模型。
- 复杂运营活动页面。
- 自由标签创建和标签审核后台。
- 未经适配器声明和执行计划的一键真实配置写入。
- 批量启用、批量停用、批量安装、批量升级。

## 1.7 成熟产品最低标准

当前版本必须达到以下最低标准：

1. 普通用户可清晰完成发现、查看、安装、启用、更新、卸载 Skill。
2. 普通用户可清晰完成 MCP Server 的查看、授权接入、配置写入、连接检测、更新和卸载。
3. 普通用户可清晰完成 Plugin 的查看、受控安装或受控下载、更新、卸载和本地状态记录。
4. 发布者可通过按钮进入发布向导，完成 Skill、MCP Server、Plugin 的发布申请，并选择授权范围与可见选项。
5. 部门管理员可在 Web 管理端处理本部门发布申请。
6. 系统管理员可处理跨部门和全员范围发布申请。
7. 管理端可查看每个关键操作的审计日志。
8. 管理端可查看客户端设备、版本分布和更新失败事件。
9. 服务端异常、审核异常、安装异常、权限异常均可被定位到 requestID 或审计记录。
10. 桌面端页面不出现大量解释文字；主要信息通过状态、列表、卡片、详情和操作反馈表达。
