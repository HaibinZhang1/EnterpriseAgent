# 15. 管理端：审计、客户端设备、客户端更新与系统设置

## 15.1 审计日志目标

审计日志用于追踪关键业务动作，帮助管理员排查问题、确认责任和满足企业内部治理要求。

审计日志必须作为正式功能交付，不得只依赖服务端控制台日志。

审计日志必须回答三个问题：

1. 谁在什么时候做了什么。
2. 操作对象和变更摘要是什么。
3. 出错时如何通过 requestID 关联运行日志、本地事件和客户端设备事件。

## 15.2 审计日志访问范围

部门管理员可查看：

- 自己管理范围内用户相关审计。
- 自己管理范围内部门相关审计。
- 自己管理范围内扩展相关审计。
- 自己管理范围内客户端设备摘要和设备事件。
- 自己执行的操作审计。

系统管理员可查看全部审计日志。

## 15.3 审计日志字段

审计日志至少包含：

- 审计 ID。
- requestID。
- 操作人 ID。
- 操作人名称快照。
- 操作人角色快照。
- 操作人部门快照。
- 对象类型。
- 对象 ID。
- 对象名称快照。
- 动作。
- 结果。
- 原因。
- 变更前摘要。
- 变更后摘要。
- IP。
- userAgent。
- 客户端版本。
- 设备 ID。
- 发生时间。

字段要求：

- Token、密码、API Key、下载凭证明文、真实密钥不得进入审计日志。
- 变更摘要必须可读，但不得包含敏感配置明文。
- requestID 必须可复制。
- 审计详情应提供跳转到相关用户、部门、扩展、申请、客户端设备、客户端更新记录的入口。

### 审计详情

要求：

- 支持按 requestID 查询关联运行日志摘要。
- 支持跳转到客户端设备详情或更新事件。
- 支持导出当前筛选结果为 CSV。
- 导出必须遵守管理员数据范围，且写审计日志。

## 15.6 审计日志保留、脱敏与防误改

- 审计日志不得通过普通业务接口修改。
- 审计日志删除或保留周期只能由系统管理员在系统设置中配置。
- 修改审计保留天数必须写审计日志。
- 默认审计保留期不得低于 180 天。
- 删除过期审计日志应由定时任务执行，并记录清理摘要。
- 审计日志导出默认脱敏手机号中间位、IP 可按企业策略保留或脱敏。
- 审计日志不得保存密码、Token、API Key、真实密钥、下载凭证明文。

## 15.7 普通操作日志与审计日志边界

- 查看扩展详情。
- Star 和取消 Star。
- 社区搜索。
- 榜单点击。

以下行为必须进入审计日志：

- 下载凭证发放。
- 权限拒绝。
- 不可见访问拒绝。
- 安装失败上报。
- 本地启用失败上报。
- MCP 配置写入失败上报。
- Plugin 安装失败上报。
- 管理端写操作。
- 发布、系统预审和审核操作。

## 15.8 客户端设备目标

客户端设备用于支持版本治理、更新排查和本地事件追踪。

设备登记不用于监控用户隐私行为，只记录企业应用运行和更新所需的最小信息。

## 15.10 客户端设备事件

设备事件包括：

- 首次登记。
- 心跳。
- 版本变化。
- 本地事件同步失败。
- 更新检查。
- 更新包下载成功或失败。
- Hash 校验失败。
- 签名校验失败。
- 安装程序启动。
- 用户取消更新。
- 更新后首次启动。

规则：

- 心跳类事件进入设备事件表，不进入审计日志。
- 更新失败、签名校验失败、Hash 校验失败必须进入审计日志。
- 设备 ID 由客户端首次启动生成并持久化。
- 主机名只保存摘要或按企业策略脱敏保存。

## 15.11 客户端更新目标

客户端更新用于系统管理员发布 Windows 桌面客户端新版本。

更新包由可信构建环境产出并签名，服务端负责登记、发布、下载凭证和更新事件记录。

## 15.12 客户端更新页面

页面包含：

## 15.14 客户端更新状态

| 状态 | 说明 |
|---|---|
| 草稿 | 已创建但未发布 |
| 已发布 | 客户端可检查到 |
| 已暂停 | 暂停签发下载凭证 |
| 已撤回 | 不再提供下载，不再恢复 |

## 15.15 客户端更新安全规则

- 更新包必须有 SHA-256。
- 更新包必须有签名状态。
- 服务端发布前必须校验包大小、Hash、版本号递增和签名状态。
- 客户端下载后必须校验 Hash。
- 客户端启动安装程序前必须校验签名。
- Hash 或签名校验失败时不得启动安装。
- 校验失败必须上报更新失败事件并写审计日志。
- 强制更新只能由系统管理员发布，且必须填写原因。
- 已撤回版本不得重新发布；如需恢复，必须创建新版本记录。

系统设置仅系统管理员可访问。

设置项包括：

- 企业服务名称。
- 服务端地址展示。
- 分类列表。
- 标签列表。
- 密码策略。
- 会话策略。
- 上传限制。
- 审计保留天数。
- 客户端更新策略。
- MCP 配置写入策略。
- Plugin 受控安装策略。
- 安全下架后存量使用策略。
- 登录失败锁定策略。
- 管理端会话超时策略。
- AI 系统预审开关。
- AI 服务地址。
- AI 模型名称。
- AI 预审超时。
- AI 预审失败降级策略。

### AI 系统预审设置

规则：

- AI 服务必须是企业内网地址。
- 不允许运行时调用互联网 AI 服务。
- AI 预审只用于辅助风险识别，不得自动通过或自动拒绝。
- AI 预审配置变更必须写审计日志。
- AI 预审不可用时，系统按设置选择继续人工审核或标记“预审不可用”。

## 15.17 设置变更审计

所有系统设置变更必须写审计日志，记录变更前后摘要。

## 15.18 验收标准

- 管理端可查询审计日志。
- 部门管理员只能查看管理范围内审计。
- 系统管理员可查看全局审计。
- 审计详情可复制 requestID。
- 审计日志可导出 CSV，并按管理员数据范围过滤。
- 审计日志不得记录敏感明文。
- 客户端设备页面可查看设备列表、版本分布、最近在线和更新事件。
- 客户端更新功能只对系统管理员开放。
- 客户端更新发布、暂停、撤回必须写审计日志。
- 客户端更新包必须支持 Hash 和签名校验。
- AI 系统预审配置只允许系统管理员修改。
- 系统设置变更必须写审计日志。
