# 3. 角色与权限

## 3.1 角色原则

系统只保留三类角色：

1. 普通用户。
2. 部门管理员。
3. 系统管理员。

不得在产品中出现额外角色名称。处理发布申请的人员统一称为“管理员”。

## 3.2 普通用户

普通用户是桌面客户端的主要使用者。

### 可执行能力

- 登录桌面客户端。
- 使用 Agent 首页入口。
- 进入社区首页。
- 搜索 Skill、MCP Server、Plugin。
- 查看可见的已发布扩展卡片和详情。
- Star 或取消 Star。
- 安装、更新授权范围内的 Skill。
- 启用、停用和卸载本机已安装 Skill；停用和卸载不因后续授权收缩而被禁止。
- 安装或接入授权范围内的 MCP Server，写入受支持工具配置、连接检测和更新；可卸载本机已托管 MCP 配置。
- 安装、更新授权范围内的 Plugin；手动安装模式可下载受控文件并记录本地状态；可卸载本机已托管 Plugin 或清除手动安装记录。
- 管理本机工具和项目配置。
- 对未授权但可见的扩展，只能查看详情和 Star，不能安装、下载、复制配置或自动写入。
- 提交 Skill、MCP Server、Plugin 发布申请。
- 查看自己提交的发布申请状态。
- 撤回自己尚未被处理完成的发布申请。
- 对需要修改的申请重新提交。
- 下架或申请归档自己已发布且仍具备操作权的扩展。
- 修改自己扩展的可见选项；展示扩大必须进入审核，展示收缩可直接生效并写审计。

### 不可执行能力

- 进入 Web 管理端。
- 处理发布申请。
- 管理部门。
- 管理用户。
- 查看全局审计日志。
- 发布客户端更新包。
- 进行全局扩展治理。

## 3.3 部门管理员

部门管理员负责本部门及其管理范围内的用户、扩展发布审核和内容治理。

### 管理范围

部门管理员的管理范围为：

- 自己所属部门。
- 所属部门的下级部门。

上级部门管理员可以管理下级部门管理员，用于完成下级部门和管理员的增删改查。部门管理员不得管理同级管理员、上级管理员、管理范围外管理员或系统管理员。

### 可执行能力

在普通用户能力基础上，部门管理员还可以：

- 登录 Web 管理端。
- 查看管理概览中的本部门待办和异常。
- 处理管理范围内用户提交的发布申请。
- 查看管理范围内扩展。
- 普通下架、重新上架、归档管理范围内扩展。
- 查看本部门及下级部门相关审计日志。
- 查看管理范围内用户的客户端设备、版本分布和设备事件。
- 新增、修改、冻结、解冻、删除、重置管理范围内普通用户。
- 新增、修改、冻结、解冻、删除、重置下级部门管理员。
- 将下级部门普通用户设置为部门管理员。
- 取消下级部门管理员身份，使其成为普通用户。
- 新增、修改、停用、删除管理范围内下级部门，前提是符合部门停用和删除规则。

### 不可执行能力

- 管理系统管理员。
- 将用户设置为系统管理员。
- 处理授权范围为全员可用的扩展申请。
- 处理跨出自己管理范围的扩展申请。
- 管理同级或上级部门管理员。
- 冻结、删除或降权自己账号。
- 发布客户端更新包。
- 修改系统全局配置。
- 查看全局审计日志。

## 3.4 系统管理员

系统管理员负责全局配置、跨部门治理、安全处置和客户端更新。

### 可执行能力

系统管理员可以：

- 登录 Web 管理端。
- 查看全局管理概览。
- 处理全员可用、跨部门、指定多部门扩展申请。
- 接管任何部门的异常审核任务。
- 管理所有扩展。
- 管理所有部门。
- 管理所有用户。
- 设置或取消部门管理员。
- 设置或取消系统管理员。
- 查看全局审计日志。
- 发布、暂停、撤回客户端更新包。
- 修改系统设置。
- 执行全局安全下架。
- 转移任意扩展维护人或归属部门。

### 约束

- 系统管理员不得在桌面客户端看到额外后台页面。
- 系统管理员在桌面客户端与普通用户体验一致，只在 Web 管理端拥有管理能力。
- 系统管理员的关键操作必须进入审计日志。
- 系统不得允许删除或冻结最后一个可用系统管理员。

## 3.6 审核归属规则

### 部门范围申请

当发布申请的授权范围限定在提交人所属部门或下级部门内，且可见选项未造成管理范围外展示影响时：

- 由提交人所在部门的部门管理员处理。
- 若提交人所在部门没有部门管理员，向上查找最近的上级部门管理员。
- 若组织链路中没有部门管理员，交由系统管理员处理。

### 指定部门申请

当发布申请授权给多个指定部门时：

- 若所有指定部门都在提交人所属部门管理员的管理范围内，由该部门管理员处理。
- 若任一指定部门超出提交人所属部门管理员的管理范围，交由系统管理员处理。

当申请将“仅授权范围内展示”改为“所有登录用户可展示”时：

- 若展示扩大后仍只影响管理范围内内容，由对应部门管理员处理。
- 若展示扩大涉及跨部门、全员可见或管理员无法判断内容外显风险，交由系统管理员处理。

### 全员申请

当发布申请的授权范围为全员可用时：

- 必须由系统管理员处理。

### 禁止自审

- 发布申请提交人不得处理自己的申请。
- 如果唯一部门管理员就是提交人本人，则任务交由上级部门管理员或系统管理员处理。
- 如果上级部门管理员也处于提交人链路且存在利益冲突，系统管理员可以接管。

## 3.7 部门管理员用户管理规则

部门管理员可以：

- 新增管理范围内普通用户。
- 修改管理范围内普通用户所属部门，但目标部门必须在其管理范围内。
- 冻结、解冻、删除管理范围内普通用户。
- 重置管理范围内普通用户密码。
- 新增下级部门管理员。
- 修改下级部门管理员资料和所属部门，但目标部门必须仍在其下级管理范围内。
- 冻结、解冻、删除下级部门管理员。
- 重置下级部门管理员密码。
- 将下级部门普通用户设置为部门管理员。
- 取消下级部门管理员身份。

部门管理员不得：

- 新增系统管理员。
- 删除系统管理员。
- 设置系统管理员。
- 操作管理范围外用户。
- 操作同级或上级部门管理员。
- 将自己降权、冻结或删除。
- 删除某个部门的最后一个可用部门管理员，除非该部门仍可被上级部门管理员覆盖管理。

## 3.8 系统管理员用户管理规则

系统管理员可以：

- 新增普通用户。
- 新增部门管理员。
- 新增系统管理员。
- 调整任意用户部门。
- 冻结、解冻、删除任意用户。
- 重置任意用户密码。

系统管理员不得：

- 删除最后一个可用系统管理员。
- 冻结最后一个可用系统管理员。

## 3.9 权限失效规则

以下动作发生后，目标用户已有会话必须失效：

- 用户被冻结。
- 用户被删除。
- 用户角色变化。
- 用户部门变化。
- 用户密码被管理员重置。
- 用户主动修改密码。

客户端下一次请求必须收到明确错误，并提示重新登录。

## 3.10 账号安全策略

账号安全策略必须由服务端统一执行。

规则：

- 用户使用手机号和密码登录；手机号是用户属性唯一键，技术主键必须使用不可变 userID。
- 密码不得明文存储，必须使用 bcrypt、Argon2 或企业认可的强哈希算法，并保存算法版本。
- 系统必须支持密码复杂度配置，至少包含最小长度、字符类型和弱密码拒绝。
- 登录失败必须记录审计日志，并支持按账号和来源限频或锁定。
- 会话必须有有效期；管理端会话有效期应短于桌面客户端会话。
- 用户被冻结、删除、部门停用、角色变化、部门变化、主动修改密码或管理员重置密码后，既有会话必须失效。
- 管理员重置密码后，可要求目标用户下次登录强制修改密码。
- 密码、Token、验证码、重置链接和下载凭证明文不得写入审计日志、运行日志或前端错误提示。
- 删除、冻结、降权、批量导入、角色变更、部门停用等路径都不得破坏最后一个可用系统管理员保护。

## 3.11 服务端强制校验原则

- 服务端不得信任客户端传入的角色、部门、授权范围和可见选项判断结果。
- 所有管理端写操作必须由服务端按当前用户、当前部门树和目标对象重新计算权限。
- 操作人、对象、结果、失败原因和 requestID 必须写入审计日志。
